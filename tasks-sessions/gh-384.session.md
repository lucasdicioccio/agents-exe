# Design session tracking infrastructure for agent-to-agent recursive calls

Depends-on: 
Base-branch: gh-384-recursive-agent-calls
Final: false

## Overview

Currently, `turnAgentRuntimeIntoIOTool` in `System.Agents.AgentTree.OneShotTool` creates "one-shot" agent calls that don't properly track sessions. When an agent calls another agent (or itself recursively) via `extraAgents`, the sub-agent's conversation is ephemeral - it's not saved to the `SessionStore` and can't be viewed in the TUI.

This issue designs the infrastructure needed to properly capture and track these recursive agent calls.

## Current Behavior

In `src/System/Agents/AgentTree/OneShot.hs`:

```haskell
runSubAgent ctx (PromptOtherAgent query) = do
    -- Creates a fresh session without parent reference
    session0 <- Session [] <$> newSessionId <*> pure Nothing <*> newTurnId
    
    -- Generates new conversation ID without linking to parent
    convId <- newConversationId
    
    -- Uses no-op tracer - traces are lost
    let tracer = Tracer $ const $ pure ()
    
    -- Runs agent but doesn't persist session
    (finalTurnContent, _) <- run convId agentWithQuery session0
    
    -- Only returns response text, session is discarded
    pure $ Text.encodeUtf8 result
```

## Proposed Changes

### 1. Add Session Tracking Callbacks

Modify `turnAgentRuntimeIntoIOTool` to accept session tracking callbacks:

```haskell
data AgentCallCallbacks = AgentCallCallbacks
    { onSessionCreated :: Session -> ConversationId -> IO ()
    -- ^ Called when a sub-agent session is created
    , onSessionUpdated :: Session -> IO ()
    -- ^ Called after each turn completes
    , onSessionCompleted :: Session -> IO ()
    -- ^ Called when sub-agent finishes
    }
```

### 2. Link Parent-Child Conversations

Use `forkedFromSessionId` field in `Session` to establish parent-child relationships:

```haskell
-- In runSubAgent:
parentSessionId <- lookupParentSession ctx parentConversationId
session0 <- Session [] <$> newSessionId <*> pure parentSessionId <*> newTurnId
```

### 3. Capture Proper Traces

Replace no-op tracer with one that captures to parent conversation:

```haskell
let tracer = contramap (SubAgentTrace callerSlug callerId parentConvId) parentTracer
```

## Files to Modify

- `src/System/Agents/AgentTree/OneShotTool.hs` - Core implementation
- `src/System/Agents/AgentTree.hs` - May need to update `Props` type

## Data Types to Consider

```haskell
-- From System.Agents.Session.Types
newtype SessionId = SessionId UUID

    { turns :: [Turn]

    , forkedFromSessionId :: Maybe SessionId  -- Use this for parent linkage

    }

```

## Tradeoffs

1. **Storage vs Performance**: Persisting all sub-agent sessions increases storage usage but provides full audit trails
2. **Synchronous vs Asynchronous**: Sub-agent calls could be async (fire-and-forget) but that complicates error handling
3. **Depth Limits**: Should we limit recursion depth? Current default is 10 levels via `defaultMaxToolCallDepth`

## Summary: Session Tracking Infrastructure for Agent-to-Agent Recursive Calls

### Changes Made

#### 1. **Modified `System.Agents.AgentTree.OneShotTool`**

**New Types:**
- `AgentCallCallbacks` - Callbacks for tracking sub-agent session lifecycle:
  - `onSessionCreated` - Called when sub-agent session is created
  - `onSessionUpdated` - Called after each turn completes  
  - `onSessionCompleted` - Called when sub-agent finishes

- `ParentSessionLookup` - Type alias for `ConversationId -> IO (Maybe SessionId)` to establish parent-child relationships

**Default Values:**
- `defaultAgentCallCallbacks` - No-op callbacks for backward compatibility
- `defaultParentSessionLookup` - Always returns `Nothing` when parent tracking isn't needed

**Updated Function:**
- `turnAgentRuntimeIntoIOTool` now accepts:
  - `AgentCallCallbacks` - for session lifecycle tracking
  - `Tracer IO Trace` - parent tracer for capturing sub-agent traces
  - `ParentSessionLookup` - for establishing parent-child session relationships

**Key Features:**
- Sub-agent sessions are created with `forkedFromSessionId` linked to parent session
- Traces are properly captured via `contramap (SubAgentTrace ...)` and routed to parent tracer
- Callbacks are invoked at session start, after each turn, and at completion

#### 2. **Modified `System.Agents.Runtime.Trace`**

Added `SubAgentTrace` constructor to the `Trace` type:
```haskell
SubAgentTrace
    { subAgentCallerSlug :: AgentSlug
    , subAgentCallerId :: AgentId
    , subAgentParentConvId :: ConversationId
    , subAgentSubConvId :: ConversationId
    , subAgentTrace :: Trace
    }
```

Updated `traceAgentSlug`, `traceAgentId`, and `traceConversationId` functions to handle the new constructor.

#### 3. **Updated CLI Modules**

All CLI modules now use a `makeAgentTool` wrapper that provides default values:
- `System.Agents.CLI.TUI`
- `System.Agents.CLI.OneShot`
- `System.Agents.CLI.Check`
- `System.Agents.CLI.ToolCall`
- `System.Agents.CLI.McpServer`

### Design Decisions

1. **Backward Compatibility**: The design uses default callbacks and lookup functions to maintain backward compatibility while allowing opt-in to full session tracking.

2. **Synchronous Callbacks**: Session callbacks are invoked synchronously for predictable ordering and error handling.

3. **Trace Routing**: Sub-agent traces are wrapped in `SubAgentTrace` and routed through the parent's tracer, enabling complete trace tree visualization.

4. **Parent-Child Linkage**: The `forkedFromSessionId` field in `Session` establishes the relationship for audit trails and TUI visualization.

5. **Error Handling**: The `onSessionCompleted` callback is always invoked (even on failure) to ensure resources are properly cleaned up.
