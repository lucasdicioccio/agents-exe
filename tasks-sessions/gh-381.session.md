## Summary

I have successfully added snapshot mode support to the SQLite toolbox. Here are the key changes:

### 1. New `SqliteSnapshot` Access Mode (Base.hs)

Added a new constructor to `SqliteAccessMode`:
```haskell
data SqliteAccessMode
    = SqliteReadOnly
    | SqliteReadWrite
    | SqliteSnapshot { snapshotSuffix :: Text }
```

The snapshot mode accepts a suffix parameter that will be appended to the original database filename to create the snapshot copy.

### 2. JSON Serialization Support (Base.hs)

- **ToJSON**: Snapshot mode serializes as `{"tag": "snapshot", "suffix": "<suffix>"}`
- **FromJSON**: Supports parsing both string format (`"read-only"`, `"read-write"`) and object format for snapshot mode

### 3. Snapshot Creation Logic (SqliteToolbox.hs)

- Added `Snapshot` to the internal `AccessMode` type
- `initializeToolbox` now detects snapshot mode and:
  1. Creates a copy of the original database with the given suffix
  2. Opens the copy in read-write mode
  3. Performs all subsequent operations on the copy
- Added `SnapshotCreatedTrace` for trace logging

### 4. New Fields in Toolbox (SqliteToolbox.hs)

The `Toolbox` type now tracks both paths:
- `toolboxOriginalPath`: The original database path (as configured)
- `toolboxWorkingPath`: The actual database being used (differs in snapshot mode)

### 5. Write Operations in Snapshot Mode (SqliteToolbox.hs)

Snapshot mode allows write operations (INSERT, UPDATE, DELETE, etc.) just like read-write mode, but all modifications happen on the copy.

### 6. Updated Trace Rendering (TraceUtils.hs)

Added handling for the new `SnapshotCreatedTrace` event.

### 7. New Tests (Main.hs)

Added 4 new tests for snapshot mode:
- Round-trip serialization
- Suffix preservation
- Agent with snapshot toolbox round-trip
- Mixed access modes round-trip

### Usage Example

```json
{
  "tag": "SqliteToolbox",
  "contents": {
    "name": "data-snapshot",
    "description": "Working copy of data",
    "path": "/path/to/data.sqlite",
    "access": {"tag": "snapshot", "suffix": "-working-copy"}
  }
}
```

This will:
1. Copy `/path/to/data.sqlite` to `/path/to/data.sqlite-working-copy`
2. Open the copy in read-write mode
3. Leave the original database completely untouched
4. Allow the agent to make modifications to the snapshot copy
