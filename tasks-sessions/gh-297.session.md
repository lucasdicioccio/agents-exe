# Add mechanism, builtin toolbox, and tools for writing new agents/tools

Depends-on: 
Base-branch: gh-288
Final: false

## Overview

This is the parent issue for adding comprehensive tooling support to make writing new agent configurations and tools easier.

## Goals

1. **Documentation Access**: `agents-exe spec bash-tools` command to embed and display bash-tools specification
2. **Tool Validation**: `agents-exe describe` standalone command to check tool registration without creating agent files
3. **Agent Scaffolding**: `agents-exe new agent <slug> <file> <model>` command
4. **Tool Scaffolding**: `agents-exe new tool <slug> <proglang> <file>` command  
5. **Developer Toolbox**: Builtin toolbox with tools for writing/validating agents and tools

## Feature Branch

All work should be done on branch `gh-288`.

## Architecture Notes

### Current State
- `System.Agents.Tools.Bash` defines the `describe|run` protocol for bash tools
- `System.Agents.CLI.SelfDescribe` provides basic describe functionality
- `System.Agents.CLI.Initialize` creates initial agent configurations
- `BuiltinToolboxDescription` in `System.Agents.Base` supports extensible builtin toolboxes

### Design Decisions Needed

1. **Spec Command**: Should embed full markdown documentation in binary vs read from file system
2. **Describe Command**: Needs to validate tool scripts directly without agent context
3. **New Agent Command**: Should support templates for different LLM providers (OpenAI, Mistral, Ollama)
4. **New Tool Command**: Need templates for different languages (bash, python, haskell, etc.)
5. **Developer Toolbox**: Should provide runtime tools for validation and scaffolding

## Summary

I have implemented the comprehensive tooling support feature for the agents-exe framework as specified in issue #288. Here's what was added:

### 1. `agents-exe spec bash-tools` Command (Spec.hs)
- Embedded the `docs/binary-tool.md` documentation into the binary using TemplateHaskell
- New command `agents-exe spec bash-tools` displays the full bash-tools specification

### 2. `agents-exe describe-tool` Command (DescribeTool.hs)
- Standalone command for validating tool scripts without requiring an agent configuration
- Supports JSON and pretty-print output formats
- `--check-only` flag for quick validation checks

### 3. `agents-exe new` Command (New.hs)
- `agents-exe new agent <slug> <file>` - Creates agent scaffolding with templates:
  - `openai` (default)
  - `mistral`
  - `ollama`
- `agents-exe new tool <slug> <file>` - Creates tool scaffolding in:
  - `bash` (default)
  - `python`
  - `haskell`
- `--force` flag to overwrite existing files

### 4. Developer Toolbox (DeveloperToolbox.hs)
New builtin toolbox with capabilities:
- `validate-tool` - Validate bash tool scripts
- `scaffold-agent` - Generate agent scaffolding from templates
- `scaffold-tool` - Generate tool scaffolding in various languages
- `show-spec` - Display embedded specification documentation

### 5. Updated Type System (Base.hs)
Added:
- `DeveloperToolboxDescription` type with `DeveloperToolbox` constructor in `BuiltinToolboxDescription`
- `DeveloperToolCapability` enum with four capabilities
- JSON serialization/deserialization support

### 6. Updated Runtime (Runtime/Runtime.hs, Runtime/Trace.hs)
- Integrated DeveloperToolbox initialization into the runtime
- Added `DeveloperToolboxTrace` to the trace system

### 7. Updated Tool Registration (ToolRegistration.hs)
- Added `registerDeveloperTool` and `registerDeveloperTools` functions
- Tool naming: `developer_{toolboxName}_developer_tools`
- Full LLM parameter schema for all developer tool operations

### 8. Updated Main.hs
- Integrated all new CLI commands into the command parser
- Added help documentation for all new commands

The implementation follows the existing codebase patterns and integrates seamlessly with the existing tool system, allowing agents to use the developer toolbox for self-improvement and validation tasks.
