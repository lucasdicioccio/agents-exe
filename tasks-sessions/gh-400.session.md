# Implement tools module for LuaToolbox to expose ToolPortal to Lua scripts

Depends-on: #379
Base-branch: gh-379
Final: false

## Overview

This issue implements the feature requested in #379: exposing tools to the LUA sandbox through the `ToolCallContext`'s `ToolPortal`.

This enables Lua scripts to call other tools (bash, sqlite, http, etc.) directly from within the sandbox, allowing for powerful tool orchestration and symbolic prompting patterns.

## Background

The infrastructure for this feature already exists:
- `ToolPortal` type and `makeToolPortal` function are implemented in `System.Agents.ToolPortal`
- `ctxToolPortal :: Maybe ToolPortal` field exists in `ToolExecutionContext`
- `executeScriptWithPortal` function receives the portal in `System.Agents.Tools.LuaToolbox`
- Test stubs exist in `test/LuaToolboxTests.hs` (commented out `portalIntegrationTests`)

**What's missing:** A `tools` Lua module that binds Haskell functions to Lua, allowing scripts to call `tools.call("tool-name", payload)`.

## Implementation Requirements

### 1. Create New Module: `System.Agents.Tools.LuaToolbox.Modules.Tools`

Create a new module following the pattern of existing Lua modules (Fs, Http, Json, Text, Time):

```haskell
module System.Agents.Tools.LuaToolbox.Modules.Tools where

data ToolsConfig = ToolsConfig
    { toolsAllowedTools :: [Text]  -- Whitelist of allowed tool names
    }

data ToolsTrace
    = ToolsCallTrace !Text !Aeson.Value  -- toolName, args
    | ToolsResultTrace !Text !Text       -- toolName, status
    deriving (Show, Eq)

registerToolsModule :: 
    Tracer IO ToolsTrace -> 
    Lua.State -> 
    ToolsConfig -> 
    Maybe ToolPortal -> 
    IO ()
```

### 2. Implement `tools.call()` Lua Function

The Lua function should:
- Accept two arguments: `toolName` (string) and `payload` (table)
- Check if the tool is in the allowed whitelist
- Call the portal if available
- Return a Lua table with the structure:
  ```lua
  {
    status = "ok" | "error" | "not-found" | "not-allowed",
    result_txt = "JSON string with result data",
    duration = 0.123,  -- seconds (optional)
    trace_id = "..."   -- string (optional)
  }
  ```

### 3. Integrate with LuaToolbox

Modify `System.Agents.Tools.LuaToolbox`:

- Add `ToolsTrace` to the `LuaModuleTrace` union type
- Update `registerStandardModulesWithTracer` to call `registerToolsModule`
- Pass the portal (from `executeScriptWithPortal`) and allowed tools to the module registration

### 4. Security Considerations

- **Whitelist enforcement**: Empty whitelist means NO tools allowed (secure default)
- **Portal availability**: Return `"not-found"` status if no portal is configured
- **Tool not allowed**: Return `"not-allowed"` status if tool not in whitelist
- **Tracing**: All tool calls should be traced with the inherited tracer

### 5. Enable Tests

Uncomment and fix the `portalIntegrationTests` in `test/LuaToolboxTests.hs`:
- `testPortalToolCall` - Tests calling a tool via mock portal
- `testNoPortal` - Tests graceful failure when no portal
- `testToolWhitelist` - Tests whitelist enforcement

Add new tests for the tools module specifically.

## Lua API Specification

```lua
-- Call a tool through the portal
local result = tools.call("bash", {command = "echo hello"})

-- Check result status
if result.status == "ok" then
    local data = json.decode(result.result_txt)
    print(data.stdout)
elseif result.status == "not-found" then
    print("Tool not found or portal not available")
elseif result.status == "not-allowed" then
    print("Tool not in allowed whitelist")
else
    print("Error: " .. result.result_txt)
end
```

## Example Use Case

```lua
local json = require("json")
local tools = require("tools")

-- Call bash to list files
local result = tools.call("bash", {command = "ls -la /tmp"})
if result.status == "ok" then
    local data = json.decode(result.result_txt)
    print("Files:", data.stdout)
end

-- Call sqlite to query data
local db_result = tools.call("sqlite_memory_query", {sql = "SELECT * FROM logs"})
if db_result.status == "ok" then
    local rows = json.decode(db_result.result_txt)
    for _, row in ipairs(rows) do
        print(row.message)
    end
end
```
