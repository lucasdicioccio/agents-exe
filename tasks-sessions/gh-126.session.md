### User Query

# Define IssueMetadata types for Feature Branch Isolation

## Summary

Define Haskell data types to support Feature Branch Isolation as described in docs/sqq-dag.md. These types will store feature branch metadata in issues.

## Background

From docs/sqq-dag.md, we need to implement a "Feature Branch Isolation" mechanism where:
- All PRs for a feature branch pull from and merge to a specific branch (e.g., `gh-123`)
- Issues carry metadata: dependencies, branch-name, and final|not-final tag
- A final "ultimate PR" issue merges the feature branch into main

## Required Types

Create a new module `System.Agents.Tools.GitHub.IssueMetadata` with the following types:

```haskell
-- | Feature branch name, typically derived from issue number (e.g., "gh-123")
newtype FeatureBranch = FeatureBranch { unFeatureBranch :: Text }
    deriving (Show, Eq, Ord, Generic)

-- | Tag indicating if this is the final merge issue
data FinalTag = FinalMerge | NotFinal
    deriving (Show, Eq, Ord, Generic)

-- | Complete metadata for a GitHub issue in the feature branch workflow
data IssueMetadata = IssueMetadata
    { imDependencies :: [IssueReference]   -- ^ Dependency issue numbers (e.g., [#125, #126])
    , imFeatureBranch :: FeatureBranch     -- ^ The feature branch this issue belongs to
    , imFinalTag :: FinalTag               -- ^ Whether this is the final merge issue
    , imOriginalIssue :: Maybe IssueReference  -- ^ Reference to the parent/original issue
    }
    deriving (Show, Eq, Generic)

-- | Reference to another issue (e.g., #123)
newtype IssueReference = IssueReference { unIssueReference :: Int }
    deriving (Show, Eq, Ord, Generic)

-- | Parse an issue reference from text like "#123" or "125"
parseIssueReference :: Text -> Either String IssueReference

-- | Normalize an issue number to a feature branch name
-- e.g., issue 123 -> "gh-123"
toFeatureBranch :: IssueReference -> FeatureBranch

-- | Convert FeatureBranch to a valid git branch name
featureBranchToText :: FeatureBranch -> Text
```

## JSON Serialization

All types must have ToJSON/FromJSON instances for storage in issue bodies. The JSON format should be:

```json
{
  "agents_metadata_version": "1.0",
  "feature_branch": "gh-123",
  "dependencies": ["#125", "#126"],
  "final_tag": "not-final",
  "original_issue": "#123"
}
```
