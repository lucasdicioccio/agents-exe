# Integrate SessionStore persistence for recursive agent calls

Depends-on: #384
Base-branch: gh-384-recursive-agent-calls
Final: false

## Overview

Building on the session tracking infrastructure from #384, this issue implements the actual persistence of sub-agent sessions to the `SessionStore`. Currently, when agents call other agents via `extraAgents`, the sub-agent's session is created in-memory but never saved to disk.

## Requirements

When an agent calls another agent:
1. The sub-agent's session must be saved to `SessionStore`
2. The session must reference the parent via `forkedFromSessionId`
3. Session updates during the sub-agent's execution should be persisted
4. The parent conversation should be able to list its child sessions

## Implementation Details

### Modify `runSubAgent` in `OneShotTool.hs`

```haskell
runSubAgent :: ToolExecutionContext -> PromptOtherAgent -> IO CByteString.ByteString
runSubAgent ctx (PromptOtherAgent query) = do
    -- Get parent session if available
    let parentConvId = ctx.ctxConversationId
    
    -- Generate new IDs
    childSessionId <- newSessionId
    childConvId <- newConversationId
    
    -- Create session with parent reference
    let forkedFrom = Just (sessionIdFromConversationId parentConvId)
    session0 <- Session [] childSessionId forkedFrom <$> newTurnId
    
    -- Save initial session to store
    storeSession store childConvId session0
    
    -- Create progress callback that persists updates
    let onProgress = \progress -> case progress of
            SessionUpdated s -> storeSession store childConvId s
            SessionCompleted s -> storeSession store childConvId s
            _ -> pure ()
    
    -- Run with persistence callback
    (finalTurnContent, finalSession) <- runWithProgress childConvId agentWithQuery session0 onProgress
    
    -- Return result
    pure $ Text.encodeUtf8 (extractResponseText finalTurnContent.llmResponse)
```

### Key Data Types

```haskell
-- From System.Agents.SessionStore
storeSession :: SessionStore -> ConversationId -> Session -> IO ()
readSession :: SessionStore -> ConversationId -> IO (Maybe Session)

-- From System.Agents.Session.Types
data Session = Session
    { turns :: [Turn]
    , sessionId :: SessionId
    , forkedFromSessionId :: Maybe SessionId  -- Parent reference
    , turnId :: TurnId
    }

-- From System.Agents.Session.Base
data SessionProgress
    = SessionStarted Session
    | SessionUpdated Session
    | SessionCompleted Session
    | SessionFailed Session Text
```

## Error Handling

- If session store is unavailable (default session store), log warning but continue
- If session persistence fails mid-conversation, log error but don't fail the agent call
- Handle race conditions where parent conversation is closed while child is running

## Testing Considerations

1. Test that sub-agent sessions are saved with correct `forkedFromSessionId`
2. Test that session updates are persisted during sub-agent execution
3. Test that child sessions can be retrieved by parent
4. Test error handling when store is unavailable

## Tradeoffs

1. **Synchronous persistence**: Blocks agent execution on disk I/O. Alternative is async with potential data loss on crash.
2. **Storage growth**: Each sub-agent call creates a new session file. Mitigation: implement session cleanup/rotation.
3. **Transaction boundaries**: Each turn is persisted individually vs. batching at end.
