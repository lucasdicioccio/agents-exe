### User Query

# Add extra-agents configuration field to Agent type for explicit cycle declaration

## Summary

Add an `extra-agents` field to the `Agent` type to allow explicit declaration of cyclic agent references. This enables agents to reference other agents outside their immediate `toolDirectory` hierarchy, supporting self-references and mutual recursion.

## Motivation

Currently, the `AgentTree` only discovers child agents from the `toolDirectory`. This prevents:
- Self-references (an agent calling itself)
- Mutual recursion (Agent A calling Agent B, and B calling A)
- Cross-references between different branches of the agent tree

By adding an explicit `extra-agents` configuration field, users can declare additional agent references that break the tree structure.

## Proposed Configuration Format

```json
{
  "tag": "OpenAIAgentDescription",
  "contents": {
    "slug": "boss-kimi",
    "apiKeyId": "openrouter",
    "flavor": "openai",
    "modelUrl": "https://openrouter.ai/api/v1",
    "modelName": "moonshot-ai/kimi-k2",
    "toolDirectory": "tools",
    "extra-agents": [
       {"slug": "superb-agent", "path": "../superb/agent.json"},
       {"slug": "helper-bot", "path": "./helpers/bot.json"}
    ],
    "announce": "The agent.",
    "systemPrompt": ["You are a helpful software agent"]
  }
}
```

## Changes Required

### 1. New Types in `src/System/Agents/Base.hs`

Add a new type for extra agent references:

```haskell
-- | Reference to an additional agent outside the toolDirectory hierarchy
data ExtraAgentRef
    = ExtraAgentRef
    { extraAgentSlug :: AgentSlug
    -- ^ The slug to use when referring to this agent
    , extraAgentPath :: FilePath
    -- ^ Path to the agent's JSON configuration file
    }
    deriving (Show, Ord, Eq, Generic)

instance ToJSON ExtraAgentRef
instance FromJSON ExtraAgentRef
```

### 2. Update `Agent` type in `src/System/Agents/Base.hs`

Add the optional field:

```haskell
data Agent = Agent
    { slug :: Text
    , apiKeyId :: Text
    , flavor :: Text
    , modelUrl :: Text
    , modelName :: Text
    , announce :: Text
    , systemPrompt :: [Text]
    , toolDirectory :: FilePath
    , mcpServers :: Maybe [McpServerDescription]
    , extraAgents :: Maybe [ExtraAgentRef]  -- NEW FIELD
    }
    deriving (Show, Ord, Eq, Generic)
```

### 3. Update JSON instances

Ensure `extraAgents` field is optional in JSON (defaulting to `Nothing` or `[]` when absent).

## Acceptance Criteria

- [ ] `ExtraAgentRef` type defined with JSON serialization
- [ ] `Agent` type updated with `extraAgents :: Maybe [ExtraAgentRef]` field
- [ ] Backward compatibility: existing agent.json files without `extra-agents` continue to work
- [ ] JSON round-trip tests pass for agents with and without `extra-agents`

## Tradeoffs

| Approach | Pros | Cons |
|----------|------|------|
| `Maybe [ExtraAgentRef]` | Explicit absence vs empty list | Slightly more complex pattern matching |
| `[ExtraAgentRef]` with default `[]` | Simpler usage | Can't distinguish "not set" from "empty" |

**Recommended**: `Maybe [ExtraAgentRef]` for explicitness.
