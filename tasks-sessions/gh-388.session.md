# Final integration: Merge recursive agent call tracking to main

Depends-on: #387

Final: true

## Overview

This is the final integration task that merges the complete recursive agent call tracking feature into the main branch. This includes all work from the feature branch `gh-384-recursive-agent-calls`.

## Summary of Changes

This feature implements proper capture and tracking of LLM-to-LLM recursive conversation calls:

### 1. Session Tracking Infrastructure (#384)
- Added `AgentCallCallbacks` for session lifecycle events
- Linked parent-child conversations via `forkedFromSessionId`
- Modified `turnAgentRuntimeIntoIOTool` signature to accept tracking callbacks

### 2. SessionStore Persistence (#385)
- Sub-agent sessions are now saved to `SessionStore`
- Sessions correctly reference parent via `forkedFromSessionId`
- Session updates during execution are persisted
- Added `findChildSessions` for retrieving child conversations

### 3. Enhanced Tracing (#386)
- Added `AgentTrace_SubAgentCall` and `SubAgentTrace` types
- Sub-agent LLM and tool calls are traced with parent context
- Traces flow through to TUI for debugging

### 4. TUI Updates (#387)
- Conversations display in hierarchical tree view
- Sub-agent calls visible inline in conversation
- Navigation support for parent-child relationships
- Visual indicators for conversations with children

## Files Modified

### Core Implementation
- `src/System/Agents/AgentTree/OneShotTool.hs` - Main implementation
- `src/System/Agents/AgentTree.hs` - Updated Props type

### Tracing
- `src/System/Agents/Runtime/Trace.hs` - New trace types

### Session Store
- `src/System/Agents/SessionStore.hs` - Child session queries

### TUI
- `src/System/Agents/TUI/Types.hs` - Tree state types
- `src/System/Agents/TUI/Render.hs` - Hierarchical rendering
- `src/System/Agents/TUI/Event.hs` - Tree navigation
- `src/System/Agents/TUI/Core.hs` - Tree loading

### CLI Entry Points
- `src/System/Agents/CLI/TUI.hs`

- `src/System/Agents/CLI/ToolCall.hs`

## Migration Notes

### For Users
- Existing sessions are unaffected (no migration needed)
- New sub-agent calls will be automatically tracked
- TUI will show existing conversations flat (no parent info for old sessions)

### For Developers
- `turnAgentRuntimeIntoIOTool` signature has changed - requires tracer parameter
- Custom agent trees need to pass tracer through Props
