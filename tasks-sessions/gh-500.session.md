### 💬 User Query

# System Toolbox File Sandbox

Depends-on: 
Base-branch: gh-497
Final: false

## Overview
Restrict file read/write operations in System Toolbox to allowed/sandboxed directories, preventing unauthorized file system access.

## Motivation
Security and safety requirement: tools should only access files within explicitly allowed directories. This prevents:
- Accidental deletion of system files
- Unauthorized access to sensitive data
- Cross-contamination between projects

## Proposed Solution
Implement a file sandbox that validates all file operations against an allow-list of directories.

### Key Features
1. **Allow-list Configuration**: Define permitted directories/paths
2. **Path Validation**: Check all file operations against allow-list
3. **Deny by Default**: Operations outside scope are rejected
4. **Pattern Support**: Glob patterns for flexible configuration

### Implementation Sketch
```haskell
-- Sandbox configuration
data FileSandbox = FileSandbox {
    allowedPaths :: [FilePath],
    allowSymlinks :: Bool,
    allowSubdirectories :: Bool
}

-- Validate and execute file operation
sandboxedOperation :: FileSandbox -> FileOperation -> IO (Either SandboxError Result)
```

## Acceptance Criteria
- [ ] System Toolbox file operations respect sandbox boundaries
- [ ] Configurable allow-list via configuration file
- [ ] Clear error messages when access is denied
- [ ] Pattern matching for directory globbing (e.g., `/project/**`)
- [ ] Path traversal attack prevention
- [ ] Tests for sandbox enforcement

## Files Likely to be Modified

- Configuration parsing modules
