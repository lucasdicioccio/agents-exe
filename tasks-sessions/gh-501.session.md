# CLI Argument Pinning for Tools

Depends-on: 
Base-branch: gh-498
Final: false

## Overview
Implement `--pin-arg` syntax to pre-configure and lock tool arguments, allowing users to create customized tool variants with fixed parameters.

## Motivation
Users often need to call tools with the same arguments repeatedly. Argument pinning allows:
- **Convenience**: Pre-configure common tool usage patterns
- **Consistency**: Ensure specific arguments are always applied
- **Simplification**: Reduce repetitive typing for complex tool calls

## Proposed Solution
Add argument pinning syntax that creates tool variants with pre-set arguments.

### Key Features
1. **Pin Syntax**: `--pin-arg name=value` or similar
2. **Tool Variants**: Create named tool configurations
3. **Override Protection**: Pinned args cannot be overridden (unless explicitly allowed)
4. **Composition**: Multiple pinned arguments can be combined

### Usage Examples
```bash
# Create a pinned variant
agents-exe tool pin bash --pin-arg "cwd=/project/src" --name "bash-in-src"

# Use pinned variant - automatically applies cwd=/project/src
agents-exe tool call bash-in-src --cmd "ls -la"

# Configuration format
pinned_tools:
  - name: bash-in-src
    tool: bash
    args:
      cwd: /project/src
```

- [ ] Pinned arguments are applied automatically on tool invocation
- [ ] Clear listing of available pinned tools

- [ ] Documentation with examples

   - `parsePinnedArg` function supporting various value types (strings, booleans, numbers, JSON objects/arrays)
   - `applyPinnedArgs` for merging pinned arguments with user-provided ones
   - Name validation for pinned tools

2. **`src/System/Agents/CLI/ToolCommands.hs`** - CLI command handlers:
   - `tool pin` - Create a pinned tool with `--pin-arg name=value`
   - `tool call` - Call a tool or pinned tool with optional JSON arguments
   - `tool list` - List all pinned tools (human or JSON format)
   - `tool unpin` - Remove a pinned tool configuration

### Usage Examples:

```bash
# Create a pinned variant
agents-exe tool pin bash --pin-arg "cwd=/project/src" --name "bash-in-src"

# Call a pinned tool (automatically applies cwd=/project/src)
agents-exe tool call bash-in-src --json '{"cmd": "ls -la"}'

# List pinned tools
agents-exe tool list --details

# Remove a pinned tool
agents-exe tool unpin bash-in-src
```

### Configuration Format:

Pinned tools are stored in `~/.config/agents-exe/pinned-tools.json`:

```json
{
  "pinnedTools": [
    {
      "name": "bash-in-src",
      "tool": "bash",
      "args": {
        "cwd": "/project/src"
      },
      "description": "Bash tool configured for /project/src"
    }
  ]
}
```
