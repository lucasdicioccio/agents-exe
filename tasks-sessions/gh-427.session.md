# Unify MCP, OneShot, TUI into OS Model

Depends-on: #426
Base-branch: gh-os-unify
Final: false

## Summary

Unify the three execution modes (MCP, OneShot, TUI) into a single OS-native implementation. Remove the legacy compatibility layers and have all modes use the ECS-based OS core (`System.Agents.OS.*`).

## Background

The codebase currently has:
1. **Legacy implementations**: `System.Agents.OneShot`, `System.Agents.TUI.*`
2. **OS Interface stubs**: `System.Agents.OS.Interfaces.OneShot`, `System.Agents.OS.Interfaces.TUI`
3. **Missing**: OS-native MCP implementation

The OS model provides:
- `System.Agents.OS.Core` - ECS world and components
- `System.Agents.OS.Agents` - Agent creation and management
- `System.Agents.OS.Conversation` - Turn/lineage management
- `System.Agents.OS.Persistence` - Session storage
- `System.Agents.OS.Resources` - Resource lifecycle
- `System.Agents.OS.Concurrent` - Locking and access patterns

## Goals

1. **Complete OS interface implementations** - Replace stub implementations with working code
2. **Create OS-native MCP interface** - New `System.Agents.OS.Interfaces.MCP`
3. **Remove legacy code** - Delete `System.Agents.OneShot`, `System.Agents.TUI.*`, `System.Agents.Runtime.*`
4. **Single entry point** - All modes use `System.Agents.OS` primitives
5. **Single PR transition** - No compatibility layer

## Detailed Changes

### 1. Complete OS.OneShot Interface (`System.Agents.OS.Interfaces.OneShot`)

**Current State:** Mostly stubs, delegates to legacy code

**Required Implementation:**

```haskell
-- | Execute a one-shot conversation with full OS integration
executeOneShot ::
    OneShotInterfaceHandle ->
    AgentId ->              -- ^ Pre-registered agent entity ID
    Text ->                 -- ^ User query
    IO OneShotResult
executeOneShot handle agentId query = do
    -- 1. Get agent from OS world
    agent <- OSWorld.getComponent (ihOS $ oscBaseHandle handle) agentId
    
    -- 2. Create conversation entity
    convId <- OSConversation.createConversation (ihOS $ oscBaseHandle handle) agentId
    
    -- 3. Add user message as turn
    OSConversation.addUserTurn (ihOS $ oscBaseHandle handle) convId query
    
    -- 4. Execute agent step loop using OS-native agent
    result <- runOneShotLoop handle agentId convId
    
    -- 5. Handle persistence based on config
    handlePersistence handle convId result
    
    pure result

-- | Run the agent step loop using OS-native primitives
runOneShotLoop ::
    OneShotInterfaceHandle ->
    AgentId ->
    ConversationId ->
    IO OneShotResult
runOneShotLoop handle agentId convId = do
    -- Get agent's toolbox bindings
    toolboxes <- OSAgents.getAgentToolboxes (ihOS $ oscBaseHandle handle) agentId
    
    -- Build tool registrations from toolbox bindings
    tools <- loadToolsFromToolboxes toolboxes
    
    -- Apply progressive disclosure
    let skillToolgroups = extractSkillToolgroups tools
    agentWithPD <- agentEvaluateActiveTools tracer tools skillToolgroups baseAgent
    
    -- Run conversation to completion
    loopUntilComplete agentWithPD convId
```

**Key components:**
- Use `System.Agents.OS.Agents` for agent management
- Use `System.Agents.OS.Conversation` for turn management
- Use `System.Agents.OS.Persistence` for session storage
- Integrate with `agentEvaluateActiveTools` for progressive disclosure

### 2. Complete OS.TUI Interface (`System.Agents.OS.Interfaces.TUI`)

**Current State:** Stubs with fake event queues

**Required Implementation:**

```haskell
-- | TUI event loop using OS-native events
tuiEventLoop :: TUIInterfaceHandle -> IO ()
tuiEventLoop handle = do
    let os = ihOS $ tuiBaseHandle handle
    
    -- Subscribe to OS events
    eventSub <- OSSubscription.subscribeToEvents os
    
    loop $ do
        event <- OSSubscription.waitForEvent eventSub
        case event of
            UserInputEvent text -> handleUserInput handle text
            AgentResponseEvent convId content -> handleAgentResponse handle convId content
            ToolCallEvent convId toolCall -> handleToolCall handle convId toolCall
            ShutdownEvent -> pure ()  -- Exit loop

-- | Handle tool execution with OS resource management
handleToolCall ::
    TUIInterfaceHandle ->
    ConversationId ->
    ToolCall ->
    IO ()
handleToolCall handle convId toolCall = do
    let os = ihOS $ tuiBaseHandle handle
    
    -- Acquire resources for tool execution
    resources <- OSResources.acquireResources os (requiredResources toolCall)
    
    -- Execute tool with resource context
    result <- executeToolWithResources resources toolCall
    
    -- Release resources
    OSResources.releaseResources os resources
    
    -- Continue conversation with result
    OSConversation.addToolResult os convId result
```

**Key components:**
- STM-based event queue (real implementation)
- Multi-agent coordination via `System.Agents.OS.Concurrent`
- Resource lifecycle management
- Layout mode state in OS world

### 3. Create OS.MCP Interface (`System.Agents.OS.Interfaces.MCP`)

**New Module:** `System.Agents.OS.Interfaces.MCP`

```haskell
module System.Agents.OS.Interfaces.MCP (
    -- * MCP Interface Handle
    MCPInterfaceHandle(..),
    MCPInterfaceConfig(..),
    
    -- * Initialization
    initMCPInterface,
    runMCPInterface,
    shutdownMCPInterface,
    
    -- * Server Operations
    handleInitialize,
    handleToolsList,
    handleToolsCall,
    handlePromptsList,
    handlePromptsGet,
    
    -- * Configuration
    defaultMCPInterfaceConfig,
) where

import System.Agents.MCP.Base (MCPRequest, MCPResponse, MCPError)
import qualified System.Agents.MCP.Server as MCPServer

-- | MCP interface configuration
data MCPInterfaceConfig = MCPInterfaceConfig
    { mcpBaseConfig :: InterfaceConfig
    , mcpServerConfig :: MCPServer.MCPConfig
    , mcpTransport :: MCPTransport
    }

-- | MCP transport options
data MCPTransport
    = TransportStdIO
    | TransportHTTP { mcpHttpPort :: Int, mcpHttpHost :: String }
    | TransportWebSocket { mcpWsPort :: Int }

-- | MCP interface handle
data MCPInterfaceHandle = MCPInterfaceHandle
    { mcpBaseHandle :: InterfaceHandle
    , mcpServerHandle :: MCPServer.MCPServerHandle
    , mcpActiveSessions :: TVar (Map SessionId ConversationId)
    -- ^ Map MCP sessions to OS conversations
    }

-- | Initialize MCP interface with OS backend
initMCPInterface :: MCPInterfaceConfig -> IO MCPInterfaceHandle
initMCPInterface config = do
    -- Initialize OS
    os <- initializeOS
    
    -- Create MCP server with OS-backed handlers
    server <- MCPServer.createServer (mcpServerConfig config) (makeOSHandlers os)
    
    -- Create session tracking
    sessionsVar <- newTVarIO Map.empty
    
    pure $ MCPInterfaceHandle
        { mcpBaseHandle = baseHandle
        , mcpServerHandle = server
        , mcpActiveSessions = sessionsVar
        }

-- | Make MCP handlers that use OS primitives
makeOSHandlers :: OSWorld -> MCPServer.MCPHandlers
makeOSHandlers os = MCPServer.MCPHandlers
    { onInitialize = handleInitializeOS os
    , onToolsList = handleToolsListOS os
    , onToolCall = handleToolCallOS os
    , onPromptsList = handlePromptsListOS os
    , onPromptGet = handlePromptGetOS os
    }

-- | Handle tools/list - returns tools from OS agent's toolboxes
handleToolsListOS :: OSWorld -> SessionId -> IO (Either MCPError [MCPTool])
handleToolsListOS os sessionId = do
    -- Get or create conversation for this session
    convId <- getOrCreateConversation os sessionId
    
    -- Get conversation's agent
    agentId <- OSConversation.getConversationAgent os convId
    
    -- Get agent's toolbox bindings
    toolboxes <- OSAgents.getAgentToolboxes os agentId
    
    -- Convert to MCP tools
    tools <- concat <$> mapM (toolboxToMCPTools os) toolboxes
    
    -- Apply progressive disclosure filtering
    session <- OSConversation.getSession os convId
    let activeTools = filterToolsByActivation session tools
    
    pure $ Right activeTools

-- | Handle tools/call - execute tool via OS
handleToolCallOS ::
    OSWorld ->
    SessionId ->
    Text ->           -- ^ Tool name
    Aeson.Value ->    -- ^ Arguments
    IO (Either MCPError MCPToolResult)
handleToolCallOS os sessionId toolName args = do
    convId <- getOrCreateConversation os sessionId
    
    -- Create tool call context
    ctx <- makeToolExecutionContext os convId
    
    -- Execute via OS tool registry
    result <- OSTools.executeTool os ctx toolName args
    
    -- Update conversation with tool result
    OSConversation.addToolResult os convId result
    
    pure $ Right (resultToMCP result)
```

```haskell
-- | Example: SQLite resource lifecycle
withSQLiteResource ::
    OSWorld ->
    SqliteConfig ->
    (SQLiteResource -> IO a) ->
    IO a
withSQLiteResource os config action = do
    -- Register resource with OS
    resId <- OSResources.registerResource os (SqliteResourceType config)
    
    -- Acquire exclusive access
    OSConcurrent.withExclusiveAccess os resId $ \res -> do
        result <- action res
        pure result
    
    -- Resource automatically released by OS
```

## Interface Comparison

| Feature | OneShot | TUI | MCP |
|---------|---------|-----|-----|
| **Input** | Single query | Interactive | JSON-RPC |
| **Output** | Single response | Streamed UI | JSON-RPC |
| **Session** | Optional file | Persistent store | Per-session |
| **Multi-agent** | No | Yes | Via sessions |
| **Resources** | Scoped to execution | Scoped to conversation | Scoped to session |
| **Concurrency** | Sequential | Multi-conversation | Concurrent sessions |

## Common Implementation Patterns

### Pattern 1: Agent Execution
```haskell
-- All interfaces use the same agent execution core
runAgent ::
    OSWorld ->
    AgentId ->
    ConversationId ->
    IO AgentResult
runAgent os agentId convId = do
    -- Load agent configuration from OS
    config <- OSWorld.getComponent os agentId :: IO AgentConfig
    
    -- Load toolboxes
    toolboxes <- OSAgents.getAgentToolboxes os agentId
    tools <- loadTools toolboxes
    
    -- Get conversation state
    session <- OSConversation.getSession os convId
    
    -- Apply progressive disclosure
    agent <- agentEvaluateActiveTools tracer tools (extractToolgroups tools) 
             $ makeBaseAgent config
    
    -- Run agent loop
    runAgentLoop agent session
```

### Pattern 2: Tool Execution
```haskell
-- All interfaces use the same tool execution
executeToolCall ::
    OSWorld ->
    ConversationId ->
    ToolCall ->
    IO ToolResult
executeToolCall os convId call = do
    -- Acquire necessary resources
    resources <- identifyRequiredResources call
    OSConcurrent.withResources os resources $ \res -> do
        -- Execute tool with resource context
        executeLlmToolCall tracer tools parsers call
```

### Pattern 3: Session Persistence
```haskell
-- All interfaces use same persistence
persistSession ::
    OSWorld ->

    IO ()

    session <- OSConversation.getSession os convId
```
