# Implement configurable keymap for TUI event bindings

Depends-on: 
Base-branch: gh-470-keymap
Final: false

## Context and Need

Currently, TUI events are bound using hardcoded pattern matches in `System.Agents.TUI.Event`. Users cannot customize keyboard shortcuts, which is problematic for:
- Accessibility needs (users who need different key combinations)
- Personal preferences (familiar keybindings from other tools)
- Platform differences (some key combinations may conflict with OS shortcuts)

## Goal

Implement a configurable keymap system that allows users to override default keyboard bindings via a JSON configuration file.

## Proposed Implementation

### 1. Create new module: `System.Agents.TUI.KeyMapping`

Create a new module that defines the keymap data types and JSON serialization.

**Data Types:**

```haskell
-- | Name of a TUI event/action
data EventName
    = EventQuit
    | EventSendMessage
    | EventNewConversation
    | EventContinueSession
    | EventTogglePause
    | EventAttachFile
    | EventClearAttachments
    | EventPasteClipboard
    | EventExportSession
    | EventViewSessionChronological
    | EventViewSessionReverse
    | EventClearQueuedMessages
    | EventCycleTabForward
    | EventCycleTabBackward
    | EventCycleFocusForward
    | EventCycleFocusBackward
    | EventToggleZoom
    | EventRefreshTools
    | EventEnterTurnNavigation
    | EventForkAtTurn
    | EventExitTurnNavigation
    | EventNavigateUp
    | EventNavigateDown
    | EventDeleteItem
    | EventOpenConversation
    deriving (Show, Eq, Ord, Enum, Bounded)

-- | Virtual key name (can be character, special key, or function key)
data KeyName
    = KeyChar Char
    | KeyEnter
    | KeyEscape
    | KeySpace
    | KeyTab
    | KeyBackTab
    | KeyBackspace
    | KeyDelete
    | KeyUp
    | KeyDown
    | KeyLeft
    | KeyRight
    | KeyPageUp
    | KeyPageDown
    | KeyHome
    | KeyEnd
    | KeyFun Int  -- Function key F1-F12
    deriving (Show, Eq, Ord)

-- | Modifier keys
data Modifiers = Modifiers
    { modCtrl :: Bool
    , modMeta :: Bool
    , modShift :: Bool
    } deriving (Show, Eq, Ord)

-- | A single key binding
data KeyBinding = KeyBinding
    { bindingKey :: KeyName
    , bindingModifiers :: Modifiers
    } deriving (Show, Eq, Ord)

-- | Complete key mapping from events to key bindings
newtype KeyMapping = KeyMapping
    { keyMappingBindings :: Map EventName [KeyBinding]
    -- ^ Multiple bindings per event (e.g., Esc and Ctrl+Q for quit)
    } deriving (Show, Eq)

instance Default KeyMapping where
    def = defaultKeyMapping
```

## Design Decisions & Tradeoffs

1. **Multiple bindings per event**: Allows both `Ctrl+Q` and `Esc` for quit, for example
2. **JSON format**: Consistent with other agent-exe configurations, human-readable
3. **Partial overrides**: Users can specify only the bindings they want to change; unspecified events use defaults
4. **Fail-soft**: If keymap file is missing or invalid, fall back to default bindings with a warning
5. **No dynamic reload**: Keymap is loaded at startup; changing it requires restart (simpler implementation)
