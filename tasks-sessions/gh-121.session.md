### 4. `qa-agent/tools/run-agents-exe`

Tool to run various `agents-exe` commands:

```bash
#!/bin/bash

if [ "$1" == "describe" ]; then
    cat <<'EOF'
{
  "slug": "run_agents_exe",
  "description": "Runs agents-exe with specified arguments. Useful for testing CLI commands like init, check, run, describe, etc.",
  "args": [
    {
      "name": "args",
      "description": "Arguments to pass to agents-exe (e.g., 'check', 'describe', 'run --prompt hello')",
      "type": "string",
      "backing_type": "string",
      "arity": "single",
      "mode": "dashdashspace"
    }
  ],
  "empty-result": { "tag": "AddMessage", "contents": "agents-exe command completed with no output" }
}
EOF
    exit 0
fi

# Parse arguments
ARGS=""
while [[ $# -gt 0 ]]; do
    case $1 in
        --args)
            ARGS="$2"
            shift 2
            ;;
        *)
            shift
            ;;
    esac
done

if [ -z "$ARGS" ]; then
    echo "Error: No arguments provided. Use --args to specify agents-exe arguments."
    exit 1
fi

echo "Running: cabal run -- agents-exe $ARGS"
eval "cabal run -- agents-exe $ARGS" 2>&1
EXIT_CODE=$?

echo ""
echo "Exit code: $EXIT_CODE"
exit $EXIT_CODE
```

## QA Categories Supported

### Code Updates to agents-exe

The agent can verify code changes by:
1. Running `cabal_build` to ensure the project compiles
2. Running `cabal_test` to verify all tests pass
3. Using `run_agents_exe` with various subcommands to test CLI functionality

### Prompt/Tool Updates

The agent can test agent configurations by:
1. Using `run_agents_exe` with the `run` subcommand to test specific agents
2. Testing `init` to verify project initialization works
3. Testing `check` to validate agent configurations
4. Testing `describe` to ensure proper tool discovery

## Testing the QA Agent

```bash
# Build the project
cd qa-agent
agents-exe run --agent-file kimi-qa.json -p "Build the project and run tests"

# Test a specific command
agents-exe run --agent-file kimi-qa.json -p "Run agents-exe describe to check tool discovery"

# Test an agent task
agents-exe run --agent-file kimi-qa.json -p "Test if agents-exe init creates valid agent files"
```

## Tradeoffs

1. **Coverage vs. Speed**: Running full cabal builds and tests can be slow. The agent provides comprehensive validation but may take time.

2. **Isolation**: The agent runs commands in the current directory context. For truly isolated testing, containerized runs would be needed.

3. **State Management**: Some commands (like `init`) create files. The agent doesn't automatically clean up, so users should run in temporary directories when testing stateful commands.

4. **API Keys**: Testing agents that require API keys will need valid keys configured in `~/.config/agents-exe/secret-keys`.
