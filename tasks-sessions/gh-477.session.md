### 💬 User Query

# Add Asynchronous/Resumable Session Execution Mode

Depends-on: 
Base-branch: gh-async-session
Final: false

## Summary

Implement an asynchronous execution mode for agents-exe that enables resumable session execution. This allows sessions to be paused after partial tool execution, persisted, and resumed later (potentially on a different machine). Additionally, support decorated tool calls that can check caches and yield control when results aren't immediately available.

## Background

Currently, `runStepM` executes all tool calls at each turn using `traverse`:

```haskell
AskUserPrompt missing -> do
    -- ALL tool calls execute immediately
    toolResponses <- traverse (agent0.toolCall ctx) missing.missingToolCalls
    let uToolResponses = zip missing.missingToolCalls toolResponses
```

This synchronous design doesn't support:
1. **Partial execution**: Running only some tool calls, saving state, and resuming later
2. **Distributed execution**: Starting on machine A, pausing, continuing on machine B
3. **Long-running tools**: Tools that need external input or take significant time
4. **Cached/Yielding tools**: Tools that can check caches and yield when results aren't ready

## Design Decisions

Based on requirements discussion, this implementation will use:

| Aspect | Decision |
|--------|----------|
| **Execution Model** | Continuation-based: Agent stores continuation, resumes with remaining calls |
| **Pending Storage** | New `PartialUserTurn` constructor alongside existing `UserTurn`/`LlmTurn` |
| **Yield Semantics** | Special `Yield` response type that signals continuation need |
| **Cache Storage** | SQLite-based with table for tool call hash → result mapping |
| **Cache Integration** | First-class concept in Agent type |
| **Backward Compatibility** | Session versioning for migrations (current: `Just 1`, new: `Just 2`) |
| **Implementation** | Single comprehensive change |

### 5. Async-Aware runStepM

```haskell
-- src/System/Agents/Session/Step.hs

-- | Enhanced step function supporting async execution
runStepMAsync :: forall r. ConversationId -> Agent r -> Session -> IO (Agent r, Either r Session)
runStepMAsync convId agent sess =
    case agent.ctxExecutionMode of
        Synchronous -> runStepMSync convId agent sess  -- Existing behavior
        Asynchronous -> runStepMAsyncMode convId agent sess

runStepMAsyncMode :: forall r. ConversationId -> Agent r -> Session -> IO (Agent r, Either r Session)
runStepMAsyncMode convId agent sess = do
    -- Check if there's a partial turn to continue
    case getPartialTurn sess of
        Just partial -> continuePartialTurn convId agent sess partial
        Nothing -> startNewAsyncStep convId agent sess

-- | Continue execution of a partial turn
continuePartialTurn :: ConversationId -> Agent r -> Session -> PartialUserTurnContent 
                    -> IO (Agent r, Either r Session)
continuePartialTurn convId agent sess partial = do
    -- Execute next pending tool call
    case pPendingCalls partial of
        [] -> finalizePartialTurn convId agent sess partial  -- All done
        (nextCall:remaining) -> do
            result <- executeToolCallAsync agent nextCall
            case result of
                ToolComplete response -> do
                    -- Update partial turn with completed call
                    let updatedPartial = partial
                            { pCompletedResponses = pCompletedResponses partial ++ [(nextCall, response)]
                            , pPendingCalls = remaining
                            }
                    sess' <- updatePartialTurn sess updatedPartial
                    pure (agent, Right sess')
                ToolYield cacheKey token -> do
                    -- Store continuation and pause
                    storeContinuation token nextCall
                    sess' <- markAsYielded sess token cacheKey
                    pure (agent, Right sess')  -- Session paused, will resume later

-- | Execute a single tool call with cache check and potential yield
executeToolCallAsync :: Agent r -> LlmToolCall -> IO AsyncToolResponse
executeToolCallAsync agent call = do
    -- Check cache first if available
    case agent.ctxToolCache of
        Just cache -> do
            let cacheKey = computeCacheKey call
            mCached <- cacheLookup cache cacheKey
            case mCached of
                Just cached -> pure $ ToolComplete (crResult cached)
                Nothing -> executeWithPotentialYield agent call cacheKey
        Nothing -> do
            -- No cache, execute normally
            response <- agent.toolCall ctx call
            pure $ ToolComplete response
```

### 6. Tool Decoration API

```haskell
-- src/System/Agents/Tools/Decoration.hs (NEW MODULE)

-- | Configuration for tool decoration
data ToolDecorationConfig = ToolDecorationConfig
    { tdcCache :: Maybe ToolCache
    , tdcYieldWhenMissing :: Bool  -- If True, yield when cache miss
    , tdcCacheTTL :: Maybe NominalDiffTime
    }

-- | Decorator that adds caching and yielding behavior
decorateTool :: ToolDecorationConfig -> Tool -> Tool
decorateTool config tool = tool
    { toolRun = \tracer ctx args -> do
        case tdcCache config of
            Nothing -> tool.toolRun tracer ctx args  -- No cache, passthrough
            Just cache -> do
                let key = computeToolCacheKey tool.toolName args
                mCached <- cacheLookup cache key
                case mCached of
                    Just cached -> pure $ crResult cached
                    Nothing 
                        | tdcYieldWhenMissing config -> do
                            -- Yield instead of executing
                            token <- newContinuationToken
                            pure $ YieldResponse token key
                        | otherwise -> do
                            -- Execute and cache
                            result <- tool.toolRun tracer ctx args
                            now <- getCurrentTime
                            let expires = addUTCTime <$> tdcCacheTTL config <*> pure now
                            cacheStore cache key $ CachedResult result now expires
                            pure result
    }

-- | Create cache-aware agent from base agent
withToolCache :: Agent r -> ToolCache -> Agent r
withToolCache agent cache = agent { ctxToolCache = Just cache }

-- | Enable async/yielding mode for agent
withAsyncMode :: Agent r -> Agent r
withAsyncMode agent = agent 
    { ctxExecutionMode = Asynchronous 
    , ctxAsyncToolCall = Just defaultAsyncExecutor
    }
```

## Usage Example

```haskell
-- Create async-capable agent
asyncAgent :: Agent Result
asyncAgent = withAsyncMode $ withToolCache baseAgent cache

-- Run session that may pause for external input
runSession :: IO ()
runSession = do
    cache <- mkSqliteToolCache ".agents-cache.db"
    let agent = withToolCache (withAsyncMode myAgent) cache
    
    -- Start or resume session
    mSession <- loadSession sessionId
    case mSession of
        Just session -> do
            -- Check if there are pending continuations
            case getPendingContinuation session of
                Just (token, _) -> do
                    -- Get external result (e.g., from user, API, etc.)
                    externalResult <- waitForExternalResult token
                    resumeToolCall cache token externalResult
                    continueSession sessionId agent session
                Nothing -> continueSession sessionId agent session
        Nothing -> startNewSession agent

-- Tool with yielding decoration
expensiveTool :: Tool
expensiveTool = decorateTool config baseTool
  where
    config = ToolDecorationConfig
        { tdcCache = Just cache
        , tdcYieldWhenMissing = True  -- Yield on cache miss
        , tdcCacheTTL = Just $ 3600 * 24  -- 24 hour cache
        }
```

## Migration Strategy

Sessions will be versioned:
- `Nothing` / missing: Legacy (pre-media)
- `Just 1`: Media support
- `Just 2`: Async/resumable execution

Migration from version 1 to 2:
```haskell
migrateSessionV1ToV2 :: Session -> Session
migrateSessionV1ToV2 session = session
    { sessionVersion = Just 2
    , sessionExecutionMode = Just Synchronous  -- Default to sync for existing
    }
```
