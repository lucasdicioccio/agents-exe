### 💬 User Query

# Enhance write-file-range ranges parameter description in tool schema

Depends-on: 
Base-branch: gh-write-file-range-schema-docs
Final: false

## Problem

When using `developer_dev_developer_tools` with `write-file-range` capability to create a new file, it took 4 failed attempts to discover the correct approach:

1. **Attempt 1**: Missing `ranges` parameter → failed
2. **Attempt 2**: `ranges: "1-1"` → failed with "skipped-out-of-bounds" (file didn't exist)
3. **Attempt 3**: `ranges: "1-"` → failed  
4. **Attempt 4**: `ranges: "head"` → SUCCESS via "prepend" operation

The LLM had to trial-and-error discover that `ranges: "head"` is the way to create/write to a new file. The error message "skipped-out-of-bounds" doesn't suggest this alternative.

## Root Cause

The tool schema's `ranges` parameter description (in `src/System/Agents/ToolRegistration.hs`) only covers reading use cases:

```haskell
propertyDescription = "For read-file-range: Line ranges (e.g., '1-10', '5', 'head', 'tail'). Omit to read entire file."
```

It doesn't explain:
- How to create new files (use `ranges: "head"`)
- The different range syntax options and their behaviors
- Common pitfalls (like trying to write to line 1-1 on a non-existent file)

## Solution

Enhance the `ranges` parameter description in the tool schema to include comprehensive guidance. This approach (Option 3 from analysis) was chosen because:
- Works across all installations and configurations
- LLMs see it directly when the tool is registered
- No dependency on external documentation files
- Persistently available in every session

### Proposed Change

Update the `ranges` parameter `propertyDescription` in `buildDeveloperToolParams` (around line 1206) to include:

```haskell
propertyDescription = 
    "Line ranges for file operations. "
    <> "Range formats: 'N' (single line), 'N-M' (range), 'N+' (after line N), 'head' (prepend), 'tail' (append), 'whole' (entire file). "
    <> "IMPORTANT: To CREATE a new file, use ranges='head'. "
    <> "For read-file-range: Omit ranges to read entire file. "
    <> "For write-file-range: Use contentBlocks array with one block per range."
```
