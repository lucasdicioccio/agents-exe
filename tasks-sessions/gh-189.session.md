### 💬 User Query

# aspirational: improve on experience when running outside a project

Today agents-exe main program looks-up configuration file starting from the current directory and upwards; when it finds no configuration, it assumes a default configuration in the `~/.config/agents-exe` directory.

Although useful in theory, I find it falls short in practice and I could use it more.

What would be nice is to have a nice features like:
- [ ] a `agents-exe paths` command returning all the important paths used while loading
- [ ] an `--agent <agent-slug>` command line argument like `--agent-file`  that picks up agents by slug based on what has been loaded
- [ ] aliases for well-known prompts (probably better handled outside but we could have `agents-exe run --alias translate -f /dev/stdin` , and 'omega' is mapped to some other well-known configurable file)

Some unexpected behaviors/bugs:
-  [ ] the conversation json files seem empty when running outside a project
