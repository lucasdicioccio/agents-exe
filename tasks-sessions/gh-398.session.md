### 💬 User Query

# Dynamic preset list in 'new agent' help text instead of hardcoded values

Depends-on: 
Base-branch: main
Final: false

## Issue

The `agents-exe new agent` command has a `-p PRESET` option, but the list of available presets is **hardcoded** in the help text rather than being dynamically generated from the internal preset definitions.

## Current State

In `app/Main.hs` (around line 793):

```haskell
parseNewAgentOptions :: Parser NewCmd.NewAgentOptions
parseNewAgentOptions =
    NewCmd.NewAgentOptions
        <$> ...
        <*> ...
        <*> ...
        <*> strOption
            ( long "preset"
                <> short 'p'
                <> metavar "PRESET"
                <> help "Provider preset (openai, mistral, ollama)"  -- <-- HARDCODED
                <> value "openai"
                <> showDefault
            )
```

The presets are actually defined in `src/System/Agents/CLI/New.hs`:

```haskell
defaultPresets :: Map Text ModelPreset
defaultPresets =
    Map.fromList
        [ ( "openai", ModelPreset { ... } )
        , ( "mistral", ModelPreset { ... } )
        , ( "ollama", ModelPreset { ... } )
        ]
```

## Problem

1. **Maintenance burden**: Adding a new preset requires editing both `New.hs` AND `Main.hs`
2. **Risk of inconsistency**: The help text can become out of sync with actual available presets
3. **Poor extensibility**: External contributors may not realize they need to update two places

## Proposed Solution

Add a helper function in `New.hs` that formats the preset list for help display, then use it in `Main.hs`.

### Changes Required

#### 1. `src/System/Agents/CLI/New.hs`

Add a new exported function:

```haskell
-- | Get a formatted list of available presets for help text
-- Returns: "openai, mistral, ollama"
formatPresetListHelp :: Text
formatPresetListHelp = Text.intercalate ", " (Map.keys defaultPresets)

-- Alternative: Export the keys for flexible formatting
listPresetNames :: [Text]
listPresetNames = Map.keys defaultPresets
```

Update module exports:
```haskell
module System.Agents.CLI.New (
    ...
    defaultPresets,
    formatPresetListHelp,  -- NEW
    ...
) where
```

#### 2. `app/Main.hs`

Update the parser to use the dynamic list:

```haskell
import qualified System.Agents.CLI.New as NewCmd

parseNewAgentOptions :: Parser NewCmd.NewAgentOptions
parseNewAgentOptions =
    NewCmd.NewAgentOptions
        <$> ...
        <*> ...
        <*> ...
        <*> strOption
            ( long "preset"
                <> short 'p'
                <> metavar "PRESET"
                <> help (Text.unpack $ "Provider preset (" <> NewCmd.formatPresetListHelp <> ")")
                <> value "openai"
                <> showDefault
            )
```

### Alternative Approaches

1. **Use `Map.keys` directly in Main.hs** (simpler, but less abstraction):
   ```haskell
   <> help ("Provider preset (" ++ intercalate ", " (map Text.unpack (Map.keys NewCmd.defaultPresets)) ++ ")")
   ```

2. **Add detailed preset help** (more informative):
   ```haskell
   formatDetailedPresetHelp :: Text
   formatDetailedPresetHelp = Text.unlines $
       ["Available presets:"] ++
       ["  " <> name <> " - " <> presetModelName preset
       | (name, preset) <- Map.toList defaultPresets]
   ```
   
   This would produce:
   ```
   Available presets:
     openai - gpt-4-turbo-preview
     mistral - mistral-large-latest
     ollama - llama3.2
   ```

## Acceptance Criteria

- [ ] Help text for `-p PRESET` option is generated from `defaultPresets` map
- [ ] Adding a new preset to `defaultPresets` automatically updates the help
- [ ] No hardcoded preset names remain in `Main.hs`
- [ ] The help format remains user-friendly (comma-separated list)

## Files to Modify

## Testing
