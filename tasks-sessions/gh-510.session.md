### 💬 User Query

# Triple-Newline Message Send

Depends-on: 
Base-branch: gh-510
Final: false

## Overview
Add option to send messages by typing triple newlines (`\n\n\n`) instead of requiring a keymap binding, providing an alternative input method for message submission.

## Motivation
Different users have different preferences for sending messages:
- Some prefer explicit key combinations (Enter, Ctrl+Enter)
- Others prefer text-based triggers (like Slack/Discord)
- Triple-newline is intuitive for multi-paragraph messages

## Proposed Solution
Support triple-newline as an alternative message send trigger, configurable by user preference.

### Key Features
1. **Triple-Newline Trigger**: `\n\n\n` sends the message
2. **Double-Newline for Paragraph**: `\n\n` creates a paragraph break
3. **Configurable**: Enable/disable in settings
4. **Visual Indicator**: Show when message will be sent

### Usage Example
```
This is paragraph one.

This is paragraph two, separated by double newline.



# The above triple newline sends the message
```

### Configuration
```yaml
ui:
  message_input:
    send_trigger: triple_newline  # or keymap
    show_send_indicator: true
```

### Implementation Sketch
```haskell
-- Message input configuration
data InputConfig = InputConfig {
    sendTrigger :: SendTrigger,  -- Keymap | TripleNewline
    showIndicator :: Bool
}

-- Check if content should trigger send
shouldSendMessage :: InputConfig -> Text -> Bool
shouldSendMessage config text = 
    case sendTrigger config of
        TripleNewline -> "\n\n\n" `T.isSuffixOf` text
        Keymap -> False  -- Handled by key event
```
