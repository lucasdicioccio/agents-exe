## Summary

**Problem:** When a Lua toolbox script calls a sub-agent via `tools.call("prompt_agent_xxx", {...})`, the nested conversation is **not captured and displayed in the TUI**.

## Root Cause

The issue lies in the `ToolPortal.executeTool` function in `src/System/Agents/ToolPortal.hs`. When a tool is invoked through the portal:

1. It creates a **fresh minimal context** using `mkMinimalContext` with **no `World` or `EventQueue`**
2. The oneshot agent tool (`AgentTree/OneShotTool.hs`) relies on these fields to emit `OSEvent_SubcallStarted`, `OSEvent_SubcallProgress`, etc.
3. Since the fields are `Nothing`, no events are emitted вЖТ TUI never learns about the subcall

## The Code Path

```
Lua script вЖТ tools.call() вЖТ ToolPortal вЖТ executeTool
                                     вЖУ
                         Creates minimalCtx with:
                         - ctxWorld = Nothing
                         - ctxEventQueue = Nothing
                         вЖУ
                         OneShotTool runs but can't emit events
                         вЖУ
                         TUI never sees the subcall
```

## Key Tradeoffs

1. **Context propagation:** The portal intentionally creates fresh contexts for isolation, but we need to selectively propagate OS integration fields
2. **Recursion safety:** The minimal context prevents infinite loops - we must preserve this while enabling TUI visibility
3. **Session ID management:** Portal generates fresh IDs; we need to maintain parent-child relationships for proper nesting

## Proposed Fix

Modify `executeTool` to accept and propagate the parent context's `World` and `EventQueue`:

```haskell
executeTool tracer portal parentCtx tool args = do
    -- Generate new IDs for this tool execution
    sessId <- newSessionId
    convId <- newConversationId
    turnId <- newTurnId
    
    -- Create context with OS fields from parent
    let minimalCtx = (mkMinimalContext sessId convId turnId portal)
            { ctxWorld = ctxWorld parentCtx
            , ctxEventQueue = ctxEventQueue parentCtx
            , ctxParentConversation = Just (ctxConversationId parentCtx)
            }
    ...
```

This ensures nested sub-agent calls from Lua scripts are properly captured and displayed as nested conversations in the TUI, matching the behavior of direct agent subcalls.
