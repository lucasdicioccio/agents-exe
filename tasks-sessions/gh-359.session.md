# Add callback mechanism to turnAgentRuntimeIntoIOTool for session reporting

Depends-on: #358
Base-branch: gh-347-recursive-conversations
Final: false

## Background

This issue is part of implementing proper LLM-to-LLM recursive conversation capture (#347).

Depends on: #348 (Session parent-child tracking)

Currently, `turnAgentRuntimeIntoIOTool` creates and runs a sub-agent session, but there's no way for the parent agent to track the sub-agent's session progress. The sub-agent session is saved to disk (if a SessionStore is provided), but the parent doesn't receive callbacks about:
- When the sub-agent session starts
- Progress updates during sub-agent execution
- When the sub-agent session completes

## Task

Add a callback mechanism to `turnAgentRuntimeIntoIOTool` that allows the parent agent to receive session progress updates from the sub-agent.

## Tradeoffs

1. **API Complexity**: Adding callback configuration increases API surface area. The `SubAgentSessionConfig` pattern with a default config mitigates this.

2. **Performance**: Callbacks are called synchronously during execution. For high-frequency updates, this could impact performance. Consider async callbacks if needed.

3. **Error Handling**: If the callback throws an exception, it will propagate and potentially abort the sub-agent. Document this behavior.

## Summary

I successfully implemented the callback mechanism for `turnAgentRuntimeIntoIOTool` to enable session reporting from sub-agents to parent agents. Here are the changes made:

### Changes to `src/System/Agents/AgentTree/OneShotTool.hs`

1. **New `SubAgentSessionConfig` type** - A configuration type that encapsulates:
   - `subAgentOnProgress :: Maybe OnSessionProgress` - Optional callback for receiving session progress updates
   - `subAgentStore :: Maybe SessionStore` - Optional session store for persisting sessions

2. **`defaultSubAgentConfig`** - A default configuration with no callbacks or storage for easy use

3. **`turnAgentRuntimeIntoIOToolWithCallbacks`** - New function that accepts `SubAgentSessionConfig` and:
   - Creates child sessions linked to parent via `mkChildSession`
   - Notifies parent of `SessionStarted` when sub-agent begins
   - Wraps the agent's step function to emit `SessionUpdated` events during execution
   - Notifies parent of `SessionCompleted` when sub-agent finishes
   - Persists to store if configured

4. **Updated `runtimeToAgentForToolInIOScriptExecution`** - Now accepts optional callback and store parameters

5. **Backward-compatible `turnAgentRuntimeIntoIOTool`** - Preserved the original API by wrapping the new callback-based function

6. **Helper functions** (duplicated from `OneShot.hs` to avoid circular dependencies):
   - `agentWithSessionProgress` - Wraps agent to emit progress events
   - `agentStoreSession` - Combines session storage with progress callbacks
   - `sessionStoreCallback` and `filepathStoreCallback` - Callback helpers

### Changes to `src/System/Agents/AgentTree.hs`

1. Re-exports the new types from `OneShotTool`:
   - `SubAgentSessionConfig(..)`
   - `defaultSubAgentConfig`
   - `turnAgentRuntimeIntoIOTool`
   - `turnAgentRuntimeIntoIOToolWithCallbacks`

### Key Design Decisions

1. **Avoided circular dependencies** - By not importing `System.Agents.OneShot` in `OneShotTool`, we broke the import cycle that would have occurred

2. **Optional callbacks** - Both the progress callback and session store are optional (`Maybe`), providing flexibility

3. **Proper parent-child linking** - Sub-agent sessions are created with `mkChildSession` to properly track the conversation hierarchy

4. **Synchronous callbacks** - Callbacks are called synchronously during execution as specified in the issue (with documented tradeoff about error propagation)
