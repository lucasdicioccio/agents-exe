# Add clipboard paste and drag-and-drop support for file attachments in TUI

Depends-on: #449
Base-branch: main
Final: false

## Overview
Extend the TUI file attachment system (implemented in #449) with advanced input methods:
1. **Clipboard paste support** - Paste images directly from screenshot tools or file copies
2. **File drag-and-drop** - Drop files from the system file manager into the terminal

These features provide a more natural workflow for users who want to quickly attach content without navigating through file browsers.

## Features

### 1. Clipboard Paste Support

Allow users to paste content directly into the TUI message editor.

#### Supported Content Types
- **Image data** - Screenshots, copied images (PNG, JPEG, etc.)
- **File paths** - Copied file paths (text/plain containing a valid path)
- **File objects** - Files copied in file manager (application-specific clipboard data)

#### Proposed Shortcuts
| Shortcut | Action |
|----------|--------|
| `Ctrl+V` | Paste from clipboard (standard terminal paste) |
| `Ctrl+Shift+V` | Enhanced paste - detect images/files in clipboard and attach |
| `Shift+Insert` | Alternative paste method |

#### Implementation Approach

**Option A: OSC 52 Sequence (Terminal-Integrated)**
- Use OSC 52 escape sequences to request clipboard content from the terminal
- Terminal emulator must support clipboard integration
- Works over SSH if terminal supports it

**Option B: External Clipboard Tool**
- Use platform-specific clipboard tools:
  - Linux: `xclip`, `wl-clipboard` (wl-copy/wl-paste), `xsel`
  - macOS: `pbcopy`/`pbpaste`
  - Windows: PowerShell `Get-Clipboard`
- Spawn subprocess to read clipboard content
- Detect MIME type from content or file extension

**Option C: Bracketed Paste Mode + Detection**
- Handle bracketed paste sequences in the TUI
- Analyze pasted content:
  - If it starts with `/` or `~` and points to a file → attach that file
  - If it's image data (detect magic bytes) → save to temp file and attach
  - Otherwise → treat as text paste

#### Technical Considerations

```haskell
-- New clipboard handling module
data ClipboardContent
    = ClipboardImage ImageData MimeType
    | ClipboardFilePath FilePath
    | ClipboardText Text
    | ClipboardUnknown

detectClipboardContent :: IO ClipboardContent
-- Uses platform-specific detection logic
```

**Image handling:**
- Detect image format from magic bytes (PNG: `\x89PNG`, JPEG: `\xFF\xD8\xFF`)
- Save to temporary file with appropriate extension
- Create `MediaAttachment` with detected MIME type
- Clean up temp files after successful send or on exit

**Multi-item paste:**
- If clipboard contains multiple files → attach all
- If mixed content → prompt user or attach files + paste text

### 2. File Drag-and-Drop Support

Allow users to drag files from their system file manager and drop them into the terminal window.

#### Supported Protocols
- **X11 XDND Protocol** - Standard X11 drag-and-drop
- **Wayland Data Control** - Modern Wayland protocol
- **iTerm2/Terminal.app Escape Sequences** - macOS terminal extensions
- **Windows Console VT** - Windows Terminal sequences

#### Implementation Approach

**Option A: Terminal-Specific Escape Sequences**

Many modern terminals support file drop via escape sequences:

```
# Example from iTerm2
\033]1337;File=name=filename.png;size=1234;inline=1:\x89PNG...\x07

# Example from kitty
\033]52;c;base64data\x07
```

**Implementation steps:**
1. Enable specific mode via escape sequences on TUI startup
2. Listen for incoming file drop sequences in event loop
3. Parse filename and content (inline) or path (reference)
4. Create `MediaAttachment` from received data

**Option B: External Drop Handler**
- Register TUI as a drop target using desktop integration
- Receive file paths via IPC/named pipe
- Attach files by path (similar to Phase 1/2)

#### Terminal Support Matrix

| Terminal | Clipboard Images | File Drop | Notes |
|----------|------------------|-----------|-------|
| iTerm2 (macOS) | ✅ | ✅ | Rich escape sequences |
| Kitty | ✅ | ✅ | kittens protocol |
| Alacritty | ✅ | ❌ | OSC 52 only |
| WezTerm | ✅ | ✅ | iTerm2-compatible |
| GNOME Terminal | ✅ | ❌ | Limited DND |
| Konsole | ✅ | ✅ | KDE protocols |
| Windows Terminal | ✅ | ❌ | Limited DND |

### 3. Smart Content Detection

When content is pasted or dropped, intelligently determine what to do:

```haskell
data ContentAction
    = AttachAsMedia MediaAttachment
    | PasteAsText Text
    | PromptUser ClipboardContent  -- When ambiguous
    | IgnoreContent

analyzeContent :: ClipboardContent -> IO ContentAction
analyzeContent content = case content of
    ClipboardImage data mime -> 
        AttachAsMedia <$> createTempAttachment data mime
    ClipboardFilePath path -> 
        ifM (doesFileExist path)
            (AttachAsMedia <$> loadMediaAttachment path)
            (PasteAsText $ Text.pack path)
    ClipboardText text ->
        if isSingleFilePath text
            then analyzeContent (ClipboardFilePath $ Text.unpack text)
            else pure $ PasteAsText text
```

```haskell
module System.Agents.TUI.Clipboard where

import System.Info (os)
import qualified Data.ByteString as BS

data ClipboardBackend
    = X11Backend      -- Uses xclip/xsel
    | WaylandBackend  -- Uses wl-clipboard
    | MacOSBackend    -- Uses pbcopy/pbpaste
    | WindowsBackend  -- Uses PowerShell
    | OSC52Backend    -- Uses terminal escape sequences
    | NoBackend       -- Clipboard unavailable

detectBackend :: IO ClipboardBackend

readClipboard :: ClipboardBackend -> IO (Maybe ClipboardContent)

-- Terminal-specific file drop support
data FileDropEvent = FileDropEvent
    { dropFilename :: FilePath
    , dropContent :: Maybe BS.ByteString  -- Nothing = reference, Just = inline
    }

enableFileDropMode :: IO ()
-- Sends escape sequences to enable DND mode

disableFileDropMode :: IO ()
-- Sends escape sequences to disable DND mode

parseTerminalSequence :: Vty.Event -> Maybe FileDropEvent
-- Parses DND escape sequences from terminal
```

### Event Loop Integration

In `System.Agents.TUI.Event`:

```haskell
-- Add to main event handler
handleNormalEvent tracer ev = case ev of
    -- ... existing handlers ...
    
    -- Clipboard paste (Ctrl+Shift+V)
    VtyEvent (Vty.EvKey (Vty.KChar 'v') [Vty.MCtrl, Vty.MShift]) -> do
        handleClipboardPaste tracer
    
    -- File drop from terminal
    AppEvent (AppEvent_FileDrop fileDrop) -> do
        handleFileDrop tracer fileDrop
    
    -- ... rest ...

handleClipboardPaste :: Tracer IO Trace -> EventM N TuiState ()
handleClipboardPaste tracer = do
    mContent <- liftIO $ detectClipboardContent
    case mContent of
        Just (ClipboardImage img mime) -> do
            tmpPath <- liftIO $ saveTempImage img mime
            attachFile tracer tmpPath
            showStatus StatusInfo "Image attached from clipboard"
        Just (ClipboardFilePath path) -> do
            attachFile tracer path
            showStatus StatusInfo $ "File attached: " <> Text.pack path
        Just (ClipboardText text) -> do
            -- Insert text into editor
            zoom (tuiUI . messageEditor) $ 
                handleEditorEvent (VtyEvent $ Vty.EvKey (Vty.KChar 'v') [Vty.MCtrl])
        Nothing -> 
            showStatus StatusError "Clipboard is empty or inaccessible"
```

## Platform-Specific Notes

### Linux (X11)
- Requires `xclip` or `xsel` for clipboard access
- DND via Xdnd protocol (requires terminal support)
- Alternative: `xdotool` to simulate drag events

### Linux (Wayland)
- Use `wl-clipboard` (wl-copy/wl-paste)
- Data control protocol for clipboard
- Some compositors support DND between apps

### macOS
- `pbcopy`/`pbpaste` built-in
- iTerm2 has excellent file drop support via escape sequences
- Terminal.app limited support

### Windows
- PowerShell `Get-Clipboard`
- Windows Terminal OSC 52 support
- Limited native DND support in terminals

| Clipboard empty | "Nothing to paste" |
| File not found (pasted path) | "File not found, pasted as text" |
| Image format unknown | "Unknown image format, saved as application/octet-stream" |
| Temp directory full | "Cannot save attachment - disk full" |
| Permission denied | "Cannot read file - permission denied" |

## Security Considerations

1. **Temp file cleanup** - Ensure temp files from clipboard are cleaned up
2. **Path validation** - Validate pasted paths don't escape allowed directories (no `../../../etc/passwd`)
3. **Size limits** - Limit clipboard image size (e.g., 50MB max) to prevent memory issues
4. **Binary safety** - Don't execute or interpret clipboard content, only attach or display

## Summary: Add clipboard paste and drag-and-drop support for file attachments in TUI

### New Module: `System.Agents.TUI.Clipboard`

Created a comprehensive clipboard handling module with the following features:

#### Clipboard Content Types
- `ClipboardContent` - Represents different types of clipboard content:
  - `ClipboardImage` - Binary image data with MIME type
  - `ClipboardFilePath` - Single file path
  - `ClipboardText` - Plain text
  - `ClipboardFilePaths` - Multiple file paths
  - `ClipboardUnknown` - Unsupported content

- `ContentAction` - Actions to take based on content analysis:
  - `AttachAsMedia` - Attach as media attachment
  - `PasteAsText` - Insert into message editor
  - `AttachMultipleFiles` - Attach multiple files
  - `IgnoreContent` - Skip unsupported content

- `FileDropEvent` - File drop events from terminal drag-and-drop

#### Platform-Specific Backends
- `X11Backend` - Uses `xclip` or `xsel` for X11 clipboard
- `WaylandBackend` - Uses `wl-paste` for Wayland clipboard  
- `MacOSBackend` - Uses `pbcopy`/`pbpaste` on macOS
- `WindowsBackend` - Uses PowerShell on Windows
- `OSC52Backend` - Uses OSC 52 terminal escape sequences
- `NoBackend` - Graceful fallback when no clipboard available

#### Key Functions
- `detectBackend` - Auto-detects best available clipboard backend
- `readClipboard` - Reads raw clipboard content
- `detectClipboardContent` - Detects content type (image, text, file paths)
- `analyzeContent` - Determines appropriate action for clipboard content
- `createTempAttachment` - Creates temporary file from clipboard image data
- `loadMediaAttachmentFromPath` - Loads file as media attachment
- `isSingleFilePath` - Detects if text is a file path
- `detectImageFormat` - Detects image format from magic bytes

#### File Drop Support
- `enableFileDropMode` - Enables terminal file drop mode (iTerm2 protocol)
- `disableFileDropMode` - Disables file drop mode
- `parseTerminalSequence` - Parses file drop escape sequences

### TUI Event Integration

Added `Ctrl+Shift+V` keyboard shortcut in `System.Agents.TUI.Event`:

- Smart content detection and handling:
  - **Images**: Automatically saves to temp file and attaches
  - **File paths**: Attaches the referenced file
  - **Multiple files**: Attaches all valid files
  - **Text**: Inserts into message editor

- Visual feedback via status messages:
  - "Text pasted from clipboard"
  - "Attached from clipboard: filename"
  - "Attached N files from clipboard"
  - "Clipboard not available - install xclip, wl-clipboard, or pbpaste"

### Updated Help Content


### Security Considerations
- Maximum clipboard image size limit (50MB)
- Path validation for pasted file paths
- Temporary file cleanup support
- No execution or interpretation of clipboard content
