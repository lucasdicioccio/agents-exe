## Summary

Implement a `session-search` command that enables fast fuzzy text search across session files with optional filtering by metadata (date, tool, agent).

## User Requirements

- **Scale**: Hundreds of JSON session files (100-999)
- **Storage**: File-based JSON sessions
- **Search style**: Grep-style with fuzzy matching ("error" matches "errors", "erroring")
- **Index**: Acceptable to create `.agents-search.db` alongside session files

## Proposed CLI Interface

```bash
# Build/update the search index (manual mode - default)
agents-exe session-index --build
agents-exe session-index --update       # incremental update

# Search with fuzzy matching
agents-exe session-search "database error"

# Search with auto-update flag
agents-exe session-search "migration" --auto

# Include tool outputs (disabled by default)
agents-exe session-search "config.yaml" --include-tool-outputs

# Combined text + metadata filtering
agents-exe session-search "auth" --after 2024-01-01 --tool write-file
agents-exe session-search "refactor" --agent my-coder --before "2 weeks ago"

# Output options
agents-exe session-search "TODO" --json           # machine-readable
agents-exe session-search "deploy" --preview 5    # show 5 lines context
```

## Technical Design

### Index Structure (SQLite)

```sql
-- Session metadata cache
CREATE TABLE session_index (
    session_id TEXT PRIMARY KEY,
    file_path TEXT NOT NULL,
    mtime INTEGER NOT NULL,          -- for incremental updates
    agent_slug TEXT,
    turn_count INTEGER,
    first_turn_at TIMESTAMP,
    last_turn_at TIMESTAMP
);

-- Trigram index for fuzzy search
CREATE VIRTUAL TABLE search_content USING fts5(
    session_id,
    content,                         -- concatenated text content
    tokenize='trigram'               -- enables fuzzy matching
);

-- Tool call index for filtering
CREATE TABLE tool_index (
    session_id TEXT REFERENCES session_index(session_id),
    tool_name TEXT NOT NULL,
    call_count INTEGER
);
```

### Index Build Process

```haskell
-- Pseudocode for index building
buildIndex :: SessionStore -> IO ()
buildIndex store = do
    sessions <- listSessions store
    forEachSession $ \sess -> do
        content <- extractSearchableContent sess
        trigrams <- generateTrigrams content
        insertIntoIndex sess trigrams
        indexTools sess
```

### Fuzzy Search Algorithm

Use SQLite FTS5 with trigram tokenizer:

```sql
-- Find sessions with fuzzy match to "error"
SELECT session_id, rank 
FROM search_content 
WHERE content MATCH 'error'
ORDER BY rank;
```

This handles:
- "error" matches "errors", "erroring", "terror"
- Typo tolerance for small changes
- Fast prefix matching

## Trade-offs Documented

1. **Trigram vs Full-text**: Chose trigram for fuzzy matching over BM25 for pure relevance. Can add BM25 later.
2. **SQLite index vs External**: Chose embedded SQLite to avoid external dependencies (ripgrep, etc.).
3. **Index location**: `.agents-search.db` in sessions directory for portability and easy cleanup.
4. **Tool outputs excluded by default**: Keeps index size manageable; opt-in with flag.
