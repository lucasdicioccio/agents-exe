# Implement session context passing for Bash tools via environment variables

## Overview

Modify the Bash tool execution (`System.Agents.Tools.Bash`) to optionally pass session context via environment variables. This allows bash scripts to access session metadata without the LLM providing it as arguments.

## Use Case

A bash tool might need to:
- Log actions with session ID for audit trails
- Access session history for context-aware processing
- Correlate external API calls with the current session
- Access agent-specific configuration per session

## Implementation

## Design Decisions

**Tradeoffs:**

1. **Environment variables vs special arguments**: Environment variables are preferred because:
   - Don't pollute the script's argument parsing
   - Inherently optional - scripts can ignore them
   - Standard Unix practice for passing metadata
   - Don't affect LLM-visible schema

2. **Always pass vs opt-in**: Initially always pass when context available:
   - Simpler implementation
   - No schema changes needed
   - Scripts can simply ignore unused vars
   - Opt-in can be added later via ScriptInfo

3. **Full session JSON size**: Consider large sessions:
   - Only include if `ctxFullSession` is `Just`
   - Document size limitations
   - Future: consider streaming or file-based for very large sessions
