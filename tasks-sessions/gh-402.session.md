# Add activation and lifetimes to all toolboxes (unified progressive disclosure)

Depends-on: #380

Final: false

## Summary

Generalize the skills progressive disclosure pattern to all toolboxes by adding `lifetime` and `activation` configuration parameters. The existing skills system (agentskills.io) remains a first-class concept but will use this new generic infrastructure internally.

## Background

Issue #380 proposed adding lifetime and activation controls to toolboxes, inspired by the skills system's progressive disclosure mechanism. This issue implements a unified approach where all toolbox types can participate in progressive disclosure while maintaining skills as a supported standard.

## Goals

1. Unify progressive disclosure across all toolbox types (Bash, MCP, OpenAPI, PostgREST, SQLite, System, Developer, Lua, Skills)
2. Replace skills' internal progressive disclosure with this generic implementation
3. Maintain backward compatibility (default: `activate: always`)
4. Support ECS resource scoping for lifetime management

## Type Definitions

### Lifetime

```haskell
data Lifetime
    = ProgramLifetime        -- One instance for entire program execution
    | RootConversationLifetime  -- One per root conversation (shared across extra-agents)
    | ConversationLifetime   -- One per conversation (default for most toolboxes)
    | ToolCallLifetime       -- Fresh instance per tool call
    deriving (Show, Eq, Generic, FromJSON, ToJSON)

-- JSON: "program", "root-conversation", "conversation", "tool-call"
```

**Constraints:**
- MCP servers CANNOT use `ToolCallLifetime` (compile-time or runtime error)
- Default: `ConversationLifetime`

### Activation

```haskell
data Activation
    = AlwaysActivated                          -- Always visible (current behavior)
    | OnDemandActivated Text                   -- on-demand with toolgroup name
    | FirstNStepsActivated Int StickyMode      -- first-n-steps(k, sticky-mode)
    deriving (Show, Eq, Generic, FromJSON, ToJSON)

data StickyMode
    = StickyIfUsed      -- Stays visible after n steps if it was used
    | NonSticky         -- Always disappears after n steps
    deriving (Show, Eq, Generic, FromJSON, ToJSON)

-- JSON for OnDemand: {"tag": "on-demand", "toolgroup": "filesystem-tools"}
-- JSON for FirstNSteps: {"tag": "first-n-steps", "steps": 3, "sticky": "sticky-if-used"}
```

newtype ToolboxSessionState = ToolboxSessionState
    { tssActiveToolgroups :: Map ToolgroupName ActivationState
