# Developer Tool File Pinning with Scoped Read/Write Operations

Depends-on: 
Base-branch: gh-499
Final: false

## Overview
Implement scoped file operations for the DeveloperToolbox with configurable allow-lists, pattern matching, and line range read/write restrictions. This restricts which files an assistant can access, similar to the Lua sandbox file restrictions.

## Motivation
Security and control requirement for DeveloperToolbox: limit file system access to explicitly allowed paths. This enables:
- **Security**: Prevent access to sensitive files outside project scope
- **Safety**: Avoid accidental modifications to system files
- **Isolation**: Keep different projects/containers properly separated
- **Auditability**: Track which files are accessible per tool configuration

## Proposed Solution
Implement file operation scoping for DeveloperToolbox that validates all file accesses against a configurable allow-list, supporting patterns and line ranges.

### Key Features
1. **Scoped Read/Write**: File operations respect configured boundaries
2. **Pattern Matching**: Glob patterns for flexible path specification (e.g., `/project/src/**/*.hs`)
3. **Range Support**: Read/write specific line ranges within scoped files (existing feature preserved)
4. **Deny by Default**: Operations outside scope are rejected with clear error messages
5. **Runtime Validation**: Scope enforcement at operation time

### Configuration Example
```yaml
developer_toolbox:
  file_scope:
    allowed_paths:
      - "/project/src/**/*.hs"
      - "/project/docs/*.md"
      - "/project/config.yaml"
    allow_range_operations: true
    allow_create_new_files: false  # restrict to existing files only
```

### Implementation Sketch
```haskell
-- File scope configuration
data FileScope = FileScope {
    allowedPatterns :: [GlobPattern],
    allowRangeOps :: Bool,
    allowCreate :: Bool
}

-- Validate file operation against scope

-- Scoped read file with range support
scopedReadFile :: FileScope -> FilePath -> Maybe LineRange -> IO (Either ScopeError Text)

-- Scoped write file with range support
scopedWriteFile :: FileScope -> FilePath -> Maybe LineRange -> Text -> IO (Either ScopeError ())
```
