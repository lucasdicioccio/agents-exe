# Add `--alias <name>` command line argument for well-known prompts

Depends-on:

## Summary

Add support for command-line prompt aliases that map to predefined prompt templates. This allows users to quickly use common prompt patterns without typing them out.

## Motivation

Users often want to perform common tasks like translation, code review, or summarization. Instead of typing out full prompts like "Translate the following text to French:", users could use a simple alias like `--alias translate`.

## Proposed Solution

Add a `--alias <name>` argument that maps to predefined prompt templates. The aliases should be:
1. Configurable via the `agents-exe.cfg.json` configuration file
2. Overrideable per-project
3. Have sensible defaults for common use cases

## Usage Examples

```bash
# Translate text
agents-exe run --agent openai-assistant --alias translate -f document.txt

# Code review
agents-exe run --agent openai-assistant --alias code-review -f src/Main.hs

# Summarize content
cat long-article.txt | agents-exe run --agent openai-assistant --alias summarize

# Explain code
agents-exe run --agent openai-assistant --alias explain -f script.py
```

## Configuration Format

Aliases are defined in `agents-exe.cfg.json`:

```json
{
  "agentsConfigDir": "~/.config/agents-exe",
  "agentsDirectories": ["./agents"],
  "agentsFiles": ["agent.json"],
  "promptAliases": {
    "translate": {
      "description": "Translate text to English",
      "template": "Please translate the following text to English:\n\n{{content}}",
      "inputMode": "stdin"
    },
    "code-review": {
      "description": "Review code for issues",
      "template": "Please review the following code for potential bugs, security issues, and style improvements:\n\n```{{language}}\n{{content}}\n```",
      "inputMode": "file"
    },
    "summarize": {
      "description": "Summarize long text",
      "template": "Please provide a concise summary of the following text:\n\n{{content}}",
      "inputMode": "stdin"
    },
    "explain": {
      "description": "Explain code in plain English",
      "template": "Please explain what the following code does in plain English:\n\n```{{language}}\n{{content}}\n```",
      "inputMode": "file"
    }
  }
}
```

## Template Variables

Aliases support variable substitution:
- `{{content}}` - The actual content from stdin or file
- `{{language}}` - Auto-detected or specified language
- `{{filename}}` - Name of the input file

## Implementation Details

### Data Types

Add to `Main.hs`:

```haskell
-- Configuration type extension
data AgentsExeConfig = AgentsExeConfig
    { agentsConfigDir :: Maybe FilePath
    , agentsDirectories :: [FilePath]
    , agentsFiles :: [FilePath]
    , agentsLogs :: Maybe AgentsExeLogConfig
    , promptAliases :: Maybe (Map Text AliasDefinition)  -- NEW
    }

data AliasDefinition = AliasDefinition
    { aliasDescription :: Text
    , aliasTemplate :: Text
    , aliasInputMode :: AliasInputMode
    }

data AliasInputMode = AliasStdin | AliasFile
    deriving (Show, Eq)

-- CLI arguments
data Prog = Prog
    { -- ... existing fields ...
    , promptAlias :: Maybe Text  -- NEW: --alias argument
    , promptScript :: PromptScript  -- Modified to be optional when alias is used
    , mainCommand :: Command
    }
```

### CLI Parser Changes

```haskell
parseOneShotOptions :: Parser OneShotOptions
parseOneShotOptions =
    OneShotOptions
        <$> optional (strOption ...)
        <*> (parseAliasPrompt <|> parsePromptScript)  -- NEW: alias OR script
        <*> parseThinkingOption

parseAliasPrompt :: Parser PromptScript
parseAliasPrompt = do
    aliasName <- strOption
        ( long "alias"
            <> metavar "NAME"
            <> help "Use a predefined prompt alias"
        )
    -- This will be resolved at runtime with the alias configuration
    pure $ AliasPrompt aliasName

-- Extend PromptScriptDirective
data PromptScriptDirective
    = Str Text
    | FileContents FilePath
    | Separator Int Text
    | ShellOutput String
    | SessionContents FilePath SessionInject.SessionVerbosity
    | AliasPrompt Text  -- NEW
    deriving (Show)
```

### Resolution Logic

In `interpretPromptScript`, handle the new directive:

```haskell
interpretPromptScriptDirective :: Map Text AliasDefinition -> PromptScriptDirective -> IO Text
interpretPromptScriptDirective aliases x =
    case x of
        -- ... existing cases ...
        AliasPrompt aliasName ->
            case Map.lookup aliasName aliases of
                Just def -> resolveAlias def
                Nothing -> throwError $ "Unknown alias: " <> aliasName

resolveAlias :: AliasDefinition -> IO Text
resolveAlias def = do
    content <- case aliasInputMode def of
        AliasStdin -> Text.getContents
        AliasFile -> -- read from -f argument or error
    pure $ substituteTemplate (aliasTemplate def) content
```

### Default Aliases

When no config is found or aliases are not defined, provide sensible defaults:

```haskell
defaultAliases :: Map Text AliasDefinition
defaultAliases = Map.fromList
    [ ("translate", AliasDefinition
        "Translate to English"
        "Please translate the following text to English:\n\n{{content}}"
        AliasStdin)
    , ("summarize", AliasDefinition
        "Summarize text"
        "Please provide a concise summary of the following:\n\n{{content}}"
        AliasStdin)
    ]
```

## Acceptance Criteria

- [ ] `--alias <name>` uses the predefined prompt template
- [ ] Aliases are configurable in `agents-exe.cfg.json`
- [ ] Unknown aliases produce helpful error with available aliases list
- [ ] Works with both `-f <file>` and stdin input
- [ ] Template variables are substituted correctly
- [ ] Default aliases are available even without configuration
- [ ] Aliases compose with other prompt directives (e.g., can add `--prompt` after alias)

## Trade-offs

- **Complexity**: Adds another layer of configuration
- **Default aliases**: May not suit all users, but they're overridable
- **Template language**: Simple variable substitution is used instead of complex templating
- **Input mode**: Explicit input mode (stdin/file) helps avoid ambiguity

## Future Enhancements

- Support for custom template variables via `--var name=value`
- Language auto-detection for code aliases
- Alias chaining: `--alias code-review --alias security-focus`
