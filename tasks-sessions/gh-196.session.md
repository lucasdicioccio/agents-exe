# Fix empty conversation JSON files when running outside a project

Depends-on: 
Base-branch: gh-189
Final: false

## Summary

When running `agents-exe` outside a project directory (i.e., when falling back to `~/.config/agents-exe/default`), the conversation JSON session files appear empty or are not created properly.

## Bug Description

The session storage mechanism uses a `SessionStore` with a configurable prefix. When no `agents-exe.cfg.json` is found in the current directory hierarchy, the application falls back to the default configuration in `~/.config/agents-exe`. However, the session storage path may not be properly initialized in this case, leading to:

1. Empty JSON files being created
2. Sessions not being persisted
3. Users unable to resume previous conversations

## Root Cause Analysis

Looking at the code flow:

1. In `Main.hs`, `initArgParserArgs` determines configuration:
   ```haskell
   initArgParserArgs :: IO ArgParserArgs
   initArgParserArgs = do
       homedir <- getHomeDirectory
       agentsExecConfig <- locateAgentsExeConfig  -- Returns Nothing outside project
       case agentsExecConfig of
           Nothing -> initWithoutAgentsExeConfig defaultConfigDir
           Just cfgPath -> initFromAgentsExeConfig homedir cfgPath
   ```

2. The session store is created from `pargs.sessionsJsonPrefix`:
   ```haskell
   let sessionStore = maybe SessionStore.defaultSessionStore SessionStore.mkSessionStore pargs.sessionsJsonPrefix
   ```

3. When no config is found, `defaultLogSesionsJsonPrefix` is `Nothing`:
   ```haskell
   initWithoutAgentsExeConfig :: FilePath -> IO ArgParserArgs
   initWithoutAgentsExeConfig pconfigdir = do
       jsonPaths <- FileLoader.listJsonDirectory (pconfigdir </> "default")
       pure $ ArgParserArgs
           pconfigdir
           jsonPaths
           Nothing  -- No HTTP logging
           Nothing  -- No JSON file logging
           Nothing  -- No raw file logging
           Nothing  -- NO SESSION PREFIX!
   ```

4. This results in `SessionStore.defaultSessionStore` being used, which has an empty prefix:
   ```haskell
   defaultSessionStore :: SessionStore
   defaultSessionStore = SessionStore ""
   ```

5. When `storeSession` is called with an empty prefix, the file path becomes:
   ```haskell
   sessionFilePath store convId = store.storePrefix ++ conversationIdToFileName convId
   -- Results in: "" ++ "conv.<uuid>.json" = "conv.<uuid>.json"
   ```

6. This writes to the current working directory, which may have permission issues or the files may be overwritten/empty due to path resolution issues.

## Proposed Fix

When running outside a project (no `agents-exe.cfg.json` found), default the session storage prefix to a subdirectory within the default config directory:

```haskell
-- In initWithoutAgentsExeConfig
let defaultSessionsDir = pconfigdir </> "sessions"
createDirectoryIfMissing True defaultSessionsDir
```

And set `defaultLogSesionsJsonPrefix` to this directory.

## Implementation Details

### Files to Modify

- `app/Main.hs`: Update `initWithoutAgentsExeConfig` to:
  1. Create a `sessions` subdirectory in the default config dir
  2. Set `defaultLogSesionsJsonPrefix` to this directory

### Code Changes

```haskell
initWithoutAgentsExeConfig :: FilePath -> IO ArgParserArgs
initWithoutAgentsExeConfig pconfigdir = do
    jsonPaths <- FileLoader.listJsonDirectory (pconfigdir </> "default")
    
    -- Create sessions directory for storing conversations when outside projects
    let sessionsDir = pconfigdir </> "sessions"
    createDirectoryIfMissing True sessionsDir
    
    pure $ ArgParserArgs
        pconfigdir
        jsonPaths
        Nothing
        Nothing
        Nothing
        (Just sessionsDir)  -- Now sessions have a proper home
```

- [ ] Session files contain valid JSON with conversation data
- [ ] Sessions can be resumed using `--session-file` with the stored files
- [ ] No empty JSON files are created
- [ ] The `sessions` directory is created automatically if it doesn't exist
