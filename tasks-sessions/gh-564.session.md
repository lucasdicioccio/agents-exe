## Phase 3: Event Subscriptions

### Goal
Enable **reactive** event consumption via publish-subscribe pattern. Multiple subscribers can receive events in real-time without polling.

### Subscription Registry

```haskell
-- src/System/Agents/OS/Events/Subscription.hs (new module)

-- | Unique subscription identifier
newtype SubscriptionId = SubscriptionId UUID
    deriving (Show, Eq, Ord)

-- | Subscription record
data EventSubscription = EventSubscription
    { subId :: SubscriptionId
    , subTargetSession :: ConversationId
    -- ^ Which session to subscribe to
    , subSubscriber :: SubscriberId
    -- ^ Who is subscribing (AgentId, ToolCallId, etc.)
    , subFilter :: SessionEventFilter
    -- ^ Which events to receive
    , subDeliveryQueue :: TQueue SessionEvent
    -- ^ Where to deliver matching events
    , subCreatedAt :: UTCTime
    , subExpiresAt :: Maybe UTCTime
    -- ^ Optional TTL for auto-cleanup
    }

-- | Who can subscribe
data SubscriberId
    = SubscriberAgent AgentId
    | SubscriberToolCall ConversationId ToolCallId
    | SubscriberSystem Text  -- Named system component
    deriving (Show, Eq, Ord)

-- | Event filtering capabilities
data SessionEventFilter
    = FilterAll
    -- ^ Receive all events
    
    | FilterEventType SessionEventType
    -- ^ Filter by event type only
    
    | FilterToolCall (Maybe Text)
    -- ^ Just (specific tool) or Nothing (any tool call)
    
    | FilterToolCallWithArg Text Text Value
    -- ^ Tool name, arg name, arg value match
    
    | FilterLlmResponseContains Text
    -- ^ LLM response contains text pattern
    
    | FilterUserInputMatches Text
    -- ^ User input matches regex/pattern
    
    | FilterTurnStatus TurnStatusFilter
    -- ^ Filter by turn state changes
    
    | FilterCombinator FilterCombinator
    -- ^ AND/OR/NOT composition
    
    deriving (Show, Eq)

data SessionEventType
    = TypeTurnEvent
    | TypeToolCallEvent
    | TypeUserInputEvent
    | TypeLlmResponseEvent
    | TypeSessionControlEvent
    deriving (Show, Eq)

data TurnStatusFilter = FilterTurnStarted | FilterTurnCompleted | FilterTurnFailed
    deriving (Show, Eq)

data FilterCombinator
    = FilterAnd SessionEventFilter SessionEventFilter
    | FilterOr SessionEventFilter SessionEventFilter
    | FilterNot SessionEventFilter
    deriving (Show, Eq)

-- | Subscription registry stored in OS World
data SubscriptionRegistry = SubscriptionRegistry
    { registrySubscriptions :: TVar (Map SubscriptionId EventSubscription)
    , registryBySession :: TVar (Map ConversationId (Set SubscriptionId))
    -- ^ Index for fast lookup by session
    , registryNextId :: TVar Int
    -- ^ For generating unique IDs
    }
```

## Phase 4: Wait-For Tool

### Goal
Enable **blocking** wait operations: "Wait for session X to emit event Y (matching condition Z) within timeout T".

### Tool API

```haskell
-- src/System/Agents/Tools/SystemToolbox/Wait.hs (new module)

-- | Parameters for system_wait_for_event tool
data WaitForEventParams = WaitForEventParams
    { waitTargetSession :: ConversationId
    -- ^ Which session to watch
    , waitEventTypes :: [EventTypeSelector]
    -- ^ What types of events to wait for
    , waitCondition :: Maybe EventCondition
    -- ^ Optional condition on the event
    , waitTimeoutSeconds :: Maybe Int
    -- ^ Timeout (Nothing = wait forever)
    , waitBufferEvents :: Bool
    -- ^ If true, buffer events while waiting (don't miss any)
    }
    deriving (Show, Eq)

-- | Event type selector (simpler than full filter for tool API)
data EventTypeSelector
    = SelectAnyToolCall
    | SelectToolCall Named
    -- ^ Wait for specific tool
    | SelectAnyUserInput
    | SelectAnyLlmResponse
    | SelectTurnCompletion
    | SelectCombinator EventCombinator
    deriving (Show, Eq)

data Named = NamedExactly Text | NamedMatching Text  -- exact or regex
    deriving (Show, Eq)

data EventCombinator
    = EventOr EventTypeSelector EventTypeSelector
    | EventAnd EventTypeSelector EventTypeSelector
    deriving (Show, Eq)

-- | Conditions on event content
data EventCondition
    = ToolCallArgCondition Text Value
    -- ^ Arg name must equal value
    | ToolCallResultContains Text
    -- ^ Result contains text
    | LlmResponseContains Text
    -- ^ Response text contains substring
    | UserInputMatches Text
    -- ^ Input matches pattern
    | ConditionCombinator ConditionCombinator
    deriving (Show, Eq)

data ConditionCombinator
    = ConditionAnd EventCondition EventCondition
    | ConditionOr EventCondition EventCondition
    | ConditionNot EventCondition
    deriving (Show, Eq)

-- | Result of wait operation
data WaitForEventResult
    = WaitSucceeded 
        { waitEvent :: SessionEvent
        , waitElapsedMs :: Int
        }
    | WaitTimeout 
        { waitEventsSeen :: [SessionEvent]
        -- ^ Events that occurred during wait (if buffering enabled)
        , waitElapsedMs :: Int
        }
    | WaitCancelled
        { waitReason :: Text
        -- ^ Why cancelled (e.g., target session ended)
        }
    | WaitError 
        { waitError :: Text
        -- ^ Error description
        }
    deriving (Show, Eq)

-- | The actual tool implementation
waitForEvent ::
    Tracer IO Trace ->
    Toolbox ->
    WaitForEventParams ->
    IO WaitForEventResult
```

### Implementation Strategy

```haskell
-- Wait implementation using STM + async
waitForEventImpl :: WaitForEventParams -> IO WaitForEventResult
waitForEventImpl params = do
    -- Create temporary subscription
    subId <- subscribeToSessionEvents ...
    
    -- Get subscriber queue
    queue <- getSubscriptionQueue subId
    
    -- Race between timeout and event arrival
    let timeoutMicros = (fromMaybe 300 params.waitTimeoutSeconds) * 1000000
    
    result <- race 
        (threadDelay timeoutMicros)
        (waitForMatchingEvent queue params)
    
    -- Cleanup subscription
    unsubscribeFromSessionEvents subId
    
    -- Return result
    case result of
        Left () -> return WaitTimeout{...}
        Right event -> return WaitSucceeded{...}

-- Wait for matching event from queue
waitForMatchingEvent :: 
    TQueue SessionEvent -> 
    WaitForEventParams -> 
    IO SessionEvent
waitForMatchingEvent queue params = atomically $ do
    event <- readTQueue queue
    if eventMatchesParams event params
        then return event
        else retry  -- Wait for next event
```

## Multi-Agent Coordination Patterns

### Pattern 1: Monitor-Worker

```
Agent A (Monitor)          Agent B (Worker)
     |                           |
     │── subscribe ─────────────►│
     │                           │
     │◄── events ────────────────│
     │                           │
     │── wait_for ──────────────►│
     │   (blocks)                │ (processes task)
     │◄── completion ────────────│
```

### Pattern 2: Fan-Out / Fan-In

```
              Agent Coordination
                     │
        ┌────────────┼────────────┐
        ▼            ▼            ▼
    Worker 1    Worker 2     Worker 3
        │            │            │
        │            │            │
        └────────────┼────────────┘
                     ▼
              Coordinator
              (waits for all)
```

### Pattern 3: Pipeline

```
Agent A        Agent B         Agent C
(Extract) ───► (Transform) ───► (Load)
   │               │               │
   │── wait ──────►│── wait ──────►│
   │◄─ done        │◄─ done        │
```
