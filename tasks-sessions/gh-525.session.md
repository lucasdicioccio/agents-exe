### 💬 User Query

# Make write-file-range 'ranges' parameter error message more explicit

Depends-on: 
Base-branch: gh-file-range-error-msg
Final: true

## Problem

When using the `developer_dev_developer_tools` with `capability: "write-file-range"`, the tool fails with an unhelpful error message if the `ranges` parameter is not provided:

```
{"error":"Developer tool error: ValidationError \"Missing 'ranges' parameter\""}
```

This error message doesn't explain what valid range values look like, making it confusing for first-time users.

## Location of Issue

The error message appears in two files:

1. **src/System/Agents/ToolRegistration.hs** (~line 970) in the `executeDeveloperCapability` function
2. **src/System/Agents/Tools.hs** (~line 280) in the `executeDeveloperCapability` function

Both have:
```haskell
_ -> pure $ DeveloperToolError () (DeveloperTools.ValidationError "Missing 'ranges' parameter")
```

## Valid Range Formats

Based on the `parseRanges` function in `DeveloperToolbox.hs`, valid range formats include:
- Single line: `"5"` (selects line 5)
- Range: `"1-10"` (selects lines 1-10)
- Head: `"head"` (prepend to file)
- Tail: `"tail"` (append to file)
- Multiple: `"1-10,20-30"` (multiple ranges, separated by commas)

## Expected Fix

Replace the error message with something more explicit:

```haskell
_ -> pure $ DeveloperToolError () (DeveloperTools.ValidationError 
    "Missing 'ranges' parameter. Valid formats: single line '5', range '1-10', 'head' (prepend), 'tail' (append), or multiple '1-10,20-30'")
```

## Files to Modify

1. `src/System/Agents/ToolRegistration.hs` - Update error message in the "write-file-range" case
2. `src/System/Agents/Tools.hs` - Update error message in the "write-file-range" case

## Acceptance Criteria

- [ ] Both files have been updated with the more explicit error message
- [ ] The error message explains all valid range formats
- [ ] Code compiles successfully
- [ ] Existing tests pass
