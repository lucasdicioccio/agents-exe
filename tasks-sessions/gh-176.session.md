## Summary

I have successfully implemented session-edition commands for the agents-exe application. Here's what was added:

### New Module: `System.Agents.Session.Edit`

A new library module at `src/System/Agents/Session/Edit.hs` that provides pure `Session -> Session` transformation functions:

- **`sessionEditTake`** - Take the first N turns from a session (like `head`)
- **`sessionEditTakeTail`** - Take the last N turns from a session (like `tail -n`)
- **`sessionEditDrop`** - Drop the first N turns from a session (like dropping from head)
- **`sessionEditDropTail`** - Drop the last N turns from a session (like dropping from tail)
- **`sessionEditCensorToolCalls`** - Remove all tool calls and responses from the session
- **`sessionEditCensorThinking`** - Remove all thinking content from LLM responses
- **`applySessionEdits`** - Compose multiple edits in sequence

### CLI Command: `session-edit`

The new command reads JSON from STDIN and outputs JSON to STDOUT, enabling compositional patterns like:

```bash
cat session.json | agents-exe session-edit --take --count 10
cat session.json | agents-exe session-edit --take-tail --count 5
cat session.json | agents-exe session-edit --drop --count 2 --take-tail --count 5
cat session.json | agents-exe session-edit --censor-thinking
cat session.json | agents-exe session-edit --censor-tool-calls
```

### Tests: `SessionEditTests`

Comprehensive test coverage including:
- Individual operation tests (take, takeTail, drop, dropTail, censor)
- Composition tests showing how operations chain together
- Edge case tests for empty sessions, negative values, and large N values

### Usage Examples

```bash
# Take first 10 turns
agents-exe session-edit --take --count 10 < session.json

# Take last 5 turns  
agents-exe session-edit --take-tail --count 5 < session.json

# Drop first 2, then take 5 (get turns 3-7)
agents-exe session-edit --drop --count 2 --take --count 5 < session.json

# Remove thinking content for privacy
agents-exe session-edit --censor-thinking < session.json

# Remove all tool calls
agents-exe session-edit --censor-tool-calls < session.json

# Compose multiple operations
agents-exe session-edit --drop --count 2 --drop-tail --count 2 --censor-thinking < session.json
```

This implementation follows the Unix philosophy of composable tools that read from STDIN and write to STDOUT, making it easy to chain session editing operations together. Closes #176
