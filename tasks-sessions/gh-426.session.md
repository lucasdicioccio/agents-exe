### 💬 User Query

# Refactor Skills to use ProgressiveDisclosure Framework

Depends-on: 
Base-branch: gh-skills-pd
Final: false

## Summary

Migrate the Skills system from its custom enable/disable mechanism to the generic ProgressiveDisclosure framework. Each skill becomes a toolgroup, and skill management uses the standard meta tools (`meta_activate_tool`, `meta_deactivate_tool`, `meta_discover_tools`) instead of skill-specific tools.

## Background

Currently, Skills implements its own progressive disclosure:
- Custom tools: `skill_list`, `skill_enable_{name}`, `skill_disable_{name}`
- Custom state: `SkillsSessionState` with `sssActiveSkills :: Map SkillName SkillScriptsState`
- Separate from the generic `ProgressiveDisclosure` framework in `System.Agents.Combinators.ProgressiveDisclosure`

The ProgressiveDisclosure framework provides:
- Generic meta tools: `meta_activate_tool`, `meta_deactivate_tool`, `meta_discover_tools`
- Generic state: `ToolboxSessionState` (folded from session history)
- Activation via `OnDemandActivated "toolgroup"` in `ToolRegistration`

## Goals

1. **Remove custom skill management tools** - Delete `skill_list`, `skill_enable_{name}`, `skill_disable_{name}`
2. **Use toolgroup per skill** - Each skill is a separate toolgroup: `"skill:{name}"` (e.g., `"skill:pdf-processing"`)
3. **Use generic meta tools** - Skill enablement becomes `meta_activate_tool("skill:pdf-processing")`
4. **Remove Skills-specific state** - Use `ToolboxSessionState` instead of `SkillsSessionState`
5. **Simplify tool registration** - All skill tools marked with `OnDemandActivated "skill:{name}"`
6. **Single PR transition** - Remove compatibility layer, do it all at once

### 4. Tool Loader (`System.Agents.AgentTree.ToolLoader`)

**Modify `loadSkillsTools`:**
```haskell
-- BEFORE: Loads skills and registers list + meta tools
loadSkillsTools :: ... -> IO [ToolRegistration]

-- AFTER: Loads skills and returns all skill tool registrations
-- The list/discover tool is now added by ProgressiveDisclosure combinator
loadSkillsTools :: ... -> IO [ToolRegistration]
loadSkillsTools agent toolsTVar = do
    let sources = fromMaybe [] (skillSources agent)
    resolvedSources <- mapM resolveSkillSource sources
    result <- SkillsSource.loadSkillsFromSources resolvedSources
    case result of
        Left err -> pure $ [errorTool err]  -- Or handle differently
        Right skillsStore -> do
            -- Just compute all skill tool registrations with activation
            let allSkillTools = concatMap (skillToToolRegistrations skillsStore) (allSkills skillsStore)
            -- No more separate meta tools - they're part of activation framework
            pure allSkillTools

-- Helper: Convert a skill to its tool registrations
skillToToolRegistrations :: SkillsStore -> Skill -> [ToolRegistration]
skillToToolRegistrations store skill =
    let skillToolgroup = "skill:" <> skillNameToText skill.skillMetadata.smName
        -- Describe tool for the skill
        describeTool = (makeDescribeTool store skill) { toolActivation = Just $ OnDemandActivated skillToolgroup }
        -- Script tools (all share same toolgroup - enabling skill enables all scripts)
        scriptTools = map (\s -> (makeScriptTool skill s) { toolActivation = Just $ OnDemandActivated skillToolgroup }) skill.skillScripts
    in describeTool : scriptTools
```

## Migration Path for Sessions

Existing sessions may have `skill_enable_{name}` tool calls. These should be handled:

**Option A: Backward compatibility (parse old tool calls)**
Modify `System.Agents.Tools.Activation.Session.extractFromToolCall` to also recognize:
- `skill_enable_{name}` → treat as `meta_activate_tool` with `toolgroup="skill:{name}"`
- `skill_disable_{name}` → treat as `meta_deactivate_tool` with `toolgroup="skill:{name}"`

**Option B: Session migration (one-time)**
When loading old sessions, transform the tool calls. This is complex and fragile.

**Recommendation: Option A** - Parse old skill tool calls as equivalent to meta tool calls.

Add to `parseMetaToolCall` in `Activation.Session`:
```haskell
parseMetaToolCall funcName val
    | funcName == "meta_activate_tool" = ...
    | funcName == "meta_deactivate_tool" = ...
    | "skill_enable_" `Text.isPrefixOf` funcName = 
        let skillName = Text.drop (Text.length "skill_enable_") funcName
        in activateToolgroup ("skill:" <> skillName)
    | "skill_disable_" `Text.isPrefixOf` funcName = 
        let skillName = Text.drop (Text.length "skill_disable_") funcName
        in deactivateToolgroup ("skill:" <> skillName)
    | funcName == "skill_list" = mempty  -- No state change
    | otherwise = mempty
```
