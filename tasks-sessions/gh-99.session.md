### 1. Updated `ToolRegistration.hs`
- Added `registerOpenAPITool` - registers a single OpenAPI tool with the LLM system
- Added `registerOpenAPITools` - registers all tools from an OpenAPI toolbox (fail-fast behavior)
- Added `registerOpenAPIToolInLLM` - alias for backward compatibility
- Added `openapi2LLMName` - naming policy for OpenAPI tools
- Imported necessary types directly from submodules to avoid circular dependencies

### 2. Updated `Tools.hs`
- Added `openapiTool` builder function for creating OpenAPI-based tools
- Re-exports all types from `Tools.Base`

### 3. Updated `Tools/Base.hs`
- Added `Tool` and `ToolDef` types (moved from `Tools.hs` to break import cycles)
- Added `OpenAPITool` constructor to `ToolDef`
- Added `OpenAPIToolResult` and `OpenAPIToolError` constructors to `CallResult`
- Added `extractCall` function

### 4. Updated `OpenAPI/Types.hs`
- Added `ToolResult` type (moved from `OpenAPIToolbox` to break import cycles)

### 5. Updated `OpenAPIToolbox.hs`
- Added exports for `toolboxName`, `toolboxBaseUrl`, `toolboxTools` accessors
- Added export for `getOperationId` helper
- Re-exported `ToolResult` from `OpenAPI.Types`
- Removed local definition of `ToolResult` (now in `OpenAPI.Types`)

### 6. Updated `System/Agents.hs`
- Exports `System.Agents.Tools.OpenAPIToolbox` module
- Exports `System.Agents.ToolRegistration` module
- Exports `System.Agents.Tools` module
- Exports `System.Agents.ToolSchema` module

### 7. No changes needed to `agents.cabal`
- All necessary modules were already exposed
- `scientific` package not explicitly needed (functionality provided by `aeson`)
