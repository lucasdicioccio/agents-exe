### 💬 User Query

# File References via Prompt Parsing

Depends-on: 
Base-branch: gh-509
Final: false

## Overview
Auto-detect `[[file:path]]` syntax in prompts to automatically include file contents as context when sending messages.

## Motivation
Users frequently reference files in their prompts. Automatic file inclusion saves time and reduces errors:
- **Convenience**: No manual copy-paste of file contents
- **Accuracy**: Always include up-to-date file content
- **Brevity**: Reference files without cluttering the prompt

## Proposed Solution
Parse prompts for `[[file:path]]` syntax and automatically inline the referenced file contents.

### Key Features
1. **File Reference Syntax**: `[[file:path/to/file.hs]]`
2. **Auto-inclusion**: File contents automatically added to context
3. **Range Support**: Reference specific lines `[[file:path#L10-L20]]`
4. **Preview Mode**: Show what will be included before sending

### Usage Examples
```markdown
# Reference entire file
Please review [[file:src/Main.hs]] and suggest improvements.

# Reference specific lines
The bug is in [[file:src/Parser.hs#L45-L67]], can you fix it?

# Reference multiple files
Compare [[file:config/prod.yaml]] with [[file:config/dev.yaml]].
```

### Implementation Sketch
```haskell
-- File reference in prompt
data FileReference = FileReference {
    filePath :: FilePath,
    lineRange :: Maybe (Int, Int)
}

-- Parse file references from prompt
parseFileReferences :: Text -> [FileReference]

-- Resolve references to content
resolveReferences :: [FileReference] -> IO [(FileReference, Text)]

-- Inline references in prompt
inlineReferences :: Text -> [(FileReference, Text)] -> Text
```
