# Directory Listing Tool with Scoped Traversal

Depends-on: 
Base-branch: gh-511
Final: false

## Overview
Implement a directory listing and file traversal tool for DeveloperToolbox that lists files with metadata, supports filtering, and respects the same file scope boundaries as read/write operations.

## Motivation
Users need to explore and discover files within their project scope. A scoped directory tool provides:
- **Discovery**: List available files without reading contents
- **Navigation**: Traverse directory structures
- **Safety**: Operations respect configured file boundaries
- **Information**: File metadata (size, modified time, permissions)

## Proposed Solution
Add directory listing capabilities to DeveloperToolbox with scope enforcement matching the file scoping system.

### Key Features
1. **Directory Listing**: List files with metadata (size, modified time, permissions, type)
2. **File Traversal**: Navigate directory trees within scoped boundaries
3. **Filtering**: Filter by name pattern, file type, date range, size
4. **Scope Enforcement**: All operations respect DeveloperToolbox file scope configuration
5. **Path Traversal Protection**: Prevent escaping scoped directories

### Tool Operations
```haskell
-- List directory contents
data ListDirectoryOp = ListDirectoryOp {
    targetPath :: FilePath,
    filters :: [FileFilter],
    recursive :: Bool,
    includeHidden :: Bool
}

-- File entry with metadata
data FileEntry = FileEntry {
    entryPath :: FilePath,
    entryType :: FileType,  -- File | Directory | Symlink
    entrySize :: Maybe Int64,
    modifiedTime :: Maybe UTCTime,
    permissions :: Maybe FileMode
}

-- Execute scoped listing
scopedListDirectory :: FileScope -> ListDirectoryOp -> IO (Either ScopeError [FileEntry])

-- Traverse directory tree (respecting scope)
scopedTraverseDirectory :: FileScope -> FilePath -> IO (Either ScopeError [FileEntry])
```

### Usage Examples
```yaml
# Configuration (shared with DeveloperToolbox file scope)
developer_toolbox:
  file_scope:
    allowed_paths:
      - "/project/src/**/*.hs"
      - "/project/docs/*.md"
    allow_directory_traversal: true
    allow_directory_listing: true

# Tool usage
list_directory:
  path: "/project/src"
  recursive: true
  filters:
    - pattern: "*.hs"
    - min_size: 100
```

### Integration with File Scoping
- Uses the same `FileScope` configuration as read/write operations
- Directory traversal cannot escape allowed path boundaries
- Attempts to list outside scope return `ScopeError`
- Symlinks are resolved and validated against scope
