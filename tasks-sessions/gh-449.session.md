# Add file attachment support to TUI MessageEditorWidget

Depends-on: 
Base-branch: main
Final: false

## Overview
Add the ability to attach files (images, documents, etc.) to messages in the TUI's `MessageEditorWidget`. This enables multi-modal conversations with LLMs that support vision and document understanding.

## Background
The underlying infrastructure already supports media attachments:
- `UserQuery` has `queryMedia :: [MediaAttachment]` field
- OpenAI completion handler already processes media attachments
- The CLI `run` command already supports `--media` / `-m` flag for one-shot mode

This issue brings the same capability to the interactive TUI.

## Proposed Implementation

### Phase 1: Basic File Path Input
Support attaching files via direct path entry:
- Add `Ctrl+F` keyboard shortcut to open file path input dialog
- User types/pastes a file path
- System validates file exists, detects MIME type, base64 encodes content
- File appears in attachment list below message editor

### Phase 2: File Browser Integration
Add proper file browser for better UX:
- Integrate `brick-file-browser` library or similar
- Navigate filesystem with arrow keys, Enter to select, Space to multi-select
- Preview selected files (name, size, MIME type)
- Support for both absolute and relative paths

### MIME Type Detection

Reuse existing `inferMediaTypeFromPath` from `System.Agents.CLI.PromptScript`:
- Images: `.png`, `.jpg`, `.jpeg`, `.gif`, `.webp`, `.svg`
- Documents: `.pdf`, `.txt`, `.md`
- Audio/Video: `.mp3`, `.mp4`, `.wav`, `.webm`
- Application: `.json`, `.xml`, `.zip`

If type cannot be inferred, prompt user to specify (e.g., `image/png;/path/to/file`).

### Error Handling

- File not found: Show status error
- Cannot read file: Show status error  
- Unknown file type: Prompt for explicit MIME type
- File too large: Warn user (some LLMs have limits)

### Supported File Types

Following OpenAI's multimodal API support:
- **Images**: PNG, JPEG, GIF, WEBP (for vision models like GPT-4V)
- **Documents**: PDF (for document understanding)
- **Other**: Any file the underlying LLM supports

### Backward Compatibility

- Sessions without attachments work unchanged
- `UserQuery` JSON serialization already handles empty media list
- Non-multimodal agents ignore media (handled at LLM level)
