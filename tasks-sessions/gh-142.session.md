### User Query

# Add CLI `toolbox list` command to list available tools from command line

Depends-on: #135,#139
Base-branch: gh-117
Final: false

## Overview

Add a new CLI subcommand `agents-exe toolbox list` that allows users to directly list all available tools without loading a full agent runtime. This provides visibility into what tools are available in a toolbox directory or from an agent configuration.

## Motivation

Currently, the only way to see what tools are available is to:
1. Run `agents-exe check` which loads an entire agent tree
2. Inspect the tool directory manually
3. Look at agent configuration files

Users need a lightweight way to introspect tool availability for debugging, documentation, and discovery purposes.

## Proposed Interface

```bash
# List tools from an agent's tool directory
agents-exe toolbox list --agent-file agent.json

# List tools from a specific directory
agents-exe toolbox list --tool-dir ./tools

# List MCP tools from a running MCP server config
agents-exe toolbox list --mcp-config mcp-server.json

# Output formats
agents-exe toolbox list --agent-file agent.json --format table  # human-readable table (default)
agents-exe toolbox list --agent-file agent.json --format json   # JSON for scripting
agents-exe toolbox list --agent-file agent.json --format names  # Just tool names, one per line
```

## Expected Output

### Table Format (default)
```
NAME                TYPE        DESCRIPTION
bash_hello          bash        Says hello to the user
bash_fetch_url      bash        Fetches content from a URL
mcp_filesystem_read mcp         Read a file from the filesystem
openapi_pet_get     openapi     Get a pet by ID
```

```json
{
  "tools": [

}
```

```haskell
parseToolboxCommand :: Parser Command
parseToolboxCommand = subparser
    ( command "list" (info parseListToolsCommand (progDesc "List available tools"))

parseListToolsOptions :: Parser ListToolsOptions
parseListToolsOptions = ListToolsOptions
    <$> parseToolSource
    <*> parseListFormat
```

### Key Implementation Points

1. **Agent-based listing**: Load agent config, resolve tool directory, use `BashTools.loadDirectory` for bash tools
2. **Directory-based listing**: Directly scan a directory for executable scripts using `BashTools.loadDirectory`
3. **MCP config listing**: Parse MCP server config, initialize MCP client briefly to fetch tools list
