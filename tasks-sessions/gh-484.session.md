## Summary

Created a comprehensive issue for adding session introspection tools to the SystemToolbox. The issue includes:

**Core Capabilities:**
1. **`list_sessions`** - List sessions with scope filtering (parents, children, all)
2. **`search_sessions`** - Full-text search using the existing FTS5 index
3. **`read_session`** - Read session content with formatting options
4. **`get_session_stats`** - Quick statistics without full content retrieval

**Scope Configuration:**
- `ScopeParentsOnly` - Access ancestor sessions
- `ScopeChildrenOnly` - Access descendant sessions  
- `ScopeSubtree` - Parents + current + children (default)
- `ScopeAll` - All sessions (explicit opt-in required)

**Key Implementation Points:**
- Follows existing toolbox patterns (SystemToolbox, SqliteToolbox)
- Leverages existing infrastructure:
  - `SessionStore` for file operations
  - `Session.Search.*` for FTS5 search
  - `SessionPrint` for formatting and statistics
  - `forkedFromSessionId` for parent/child relationships
- Includes security considerations (scope enforcement, content filtering)
- Documents tradeoffs (performance, memory, complexity)

The issue is marked with `needs_arbitrage: true` as requested, since this is a significant feature addition that warrants review before implementation begins.
