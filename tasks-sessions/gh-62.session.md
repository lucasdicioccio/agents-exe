### User Query

# Simplify 'check' command output to show 'slug: announce (n-tools)' on a single line

## Problem

The current `check` command in `app/Main.hs` prints noisy output including:
- `@<slug>:` lines
- `reload tools...` messages

This is because it uses `traceUsefulPromptStdout` as the tracer, which outputs these diagnostic messages.

## Desired Behavior

The `check` command should output a single line per agent in the format:
```
slug: announce (n-tools)
```

For example:
```
main-agent: a helpful puppet-master capable of orchestrating other agents ensuring (5 tools)
```

## Changes Required

In `app/Main.hs`, modify the `Check` command handler:

1. Use `Prod.silent` (or a silent tracer) instead of `Prod.traceBoth baseTracer traceUsefulPromptStdout` to suppress unwanted diagnostic output during agent loading.

2. After loading the agent tree with `withAgentTreeRuntime`, extract and print the required information from the `AgentTree`:
   - `agentSlug` from `Runtime` (or from `Agent` via `agentBase.slug`)
   - `agentAnnounce` from `Runtime` (or `announce` from `Agent` via `agentBase`)
   - Count of tools from `Runtime.agentTools` (or from children in `AgentTree`)

### Data Types Reference

```haskell
    , ...
    }
```
