# Partially-Applied Tools (Progressive Disclosure)

Depends-on: 
Base-branch: gh-500
Final: false

## Overview
Implement progressive disclosure for tool parameters, hiding advanced parameters until needed to reduce cognitive load and simplify tool interfaces.

## Motivation
Many tools have numerous parameters, but most use cases only need a few common ones. Progressive disclosure helps:
- **Reduce Complexity**: Show only essential parameters initially
- **Lower Barrier**: Easier for new users to understand tools
- **Discoverability**: Advanced options available when needed
- **Cleaner UI**: Less cluttered interface

## Proposed Solution
Add metadata to tool parameters indicating their level (basic/advanced) and UI logic to show/hide advanced parameters.

### Key Features
1. **Parameter Tiers**: Mark parameters as `basic`, `advanced`, or `expert`
2. **Progressive Disclosure UI**: Show/hide parameters based on tier
3. **User Preference**: Remember user's disclosure preference
4. **Tool Definition**: Declare parameter tiers in tool schema

### Tool Schema Extension
```yaml
tools:
  - name: complex_tool
    params:
      - name: input_file
        type: string
        tier: basic  # Always shown
      - name: output_format
        type: enum
        tier: basic
      - name: compression_level
        type: int
        tier: advanced  # Hidden by default
      - name: custom_headers
        type: object
        tier: expert  # Deeply hidden
```

### UI Behavior
```
[Basic View]              [Advanced View]
Input File: [______]      Input File: [______]
Format: [______ ▼]        Format: [______ ▼]
[Show Advanced ▼]         Compression: [___]
                          [Show Expert ▼]
```
