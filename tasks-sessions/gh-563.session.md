## Background

### Current State

The existing `SystemToolbox` session capabilities (`list-sessions`, `read-session`, `search-sessions`) only access **persisted** sessions via `SessionStore` (disk-based JSON files). Ongoing conversations in the TUI or OS ECS exist only in-memory and are not interactable.

```

-- Add event emission to runStepM
runStepMWithEvents ::
    forall r.
    ConversationId ->
    Agent r ->
    Session ->
    Maybe (TBQueue SessionEvent) ->  -- Event history queue
    IO (Agent r, Either r Session)
    
-- Emit function
emitEvent :: Maybe (TBQueue SessionEvent) -> SessionEvent -> IO ()
emitEvent Nothing _ = pure ()  -- No event queue configured
emitEvent (Just queue) event = atomically $ do
    -- Non-blocking write; drop oldest if full
    full <- isFullTBQueue queue
    when full $ void $ tryReadTBQueue queue
    writeTBQueue queue event
```
