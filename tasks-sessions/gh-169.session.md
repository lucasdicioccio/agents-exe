## Summary

Closes #169 - PostgREST toolbox mapping for post/put/get

### Changes Made

1. **`src/System/Agents/Tools/PostgREST/Types.hs`**:
   - Added `HttpMethod` data type with all HTTP verbs: `GET`, `HEAD`, `POST`, `PUT`, `PATCH`, `DELETE`, `OPTIONS`
   - Added `methodToText` and `textToMethod` conversion functions
   - Added `isReadOnlyMethod` function to distinguish read-only operations
   - Added `defaultAllowedMethods` constant (read-only methods: `[GET, HEAD, OPTIONS]`)
   - Updated `Config` type to include `configAllowedMethods :: [HttpMethod]`
   - Updated `PostgRESTool` to use `HttpMethod` instead of `Text` for `prtMethod`
   - Added `prtRequestBodySchema` and `prtIsReadOnly` fields to `PostgRESTool`
   - Updated `ToolParameters` to include optional `tpRequestBody` for write operations

2. **`src/System/Agents/Tools/PostgREST/Converter.hs`**:
   - Updated `convertPostgRESToTools` to accept a list of allowed HTTP methods
   - Added support for extracting request body schemas for `POST`, `PUT`, `PATCH` operations
   - Updated `deriveToolName` to include the HTTP method in the tool name (e.g., `postgrest_mydb_get_users`, `postgrest_mydb_post_users`)
   - Updated `buildToolDescription` to include method information
   - Updated `buildToolParameters` to conditionally include request body for write operations

3. **`src/System/Agents/Tools/PostgRESToolbox.hs`**:
   - Updated `initializeToolbox` to use `configAllowedMethods` from config
   - Updated `handleToolCall` to support request bodies for write operations
   - Added `executeRequest` and `buildRequest` functions to handle different HTTP methods
   - Updated `handleResponse` to handle HEAD requests specially (no body)

4. **`src/System/Agents/Base.hs`**:
   - Added `postgrestAllowedMethods :: Maybe [HttpMethod]` field to `PostgRESTServerDescription`
   - Added JSON serialization/deserialization for `HttpMethod`

5. **`src/System/Agents/AgentTree.hs`**:
   - Updated `postgrestDescToConfig` to handle the new `allowedMethods` field, defaulting to `defaultAllowedMethods` when not specified

6. **`src/System/Agents/ToolRegistration.hs`**:
   - Updated `postgrest2LLMName` to use `methodToText` for the method part of the tool name

7. **`test/PostgRESTParseTest.hs`**:
   - Updated test to use the new `convertPostgRESToTools` signature with `defaultAllowedMethods`
