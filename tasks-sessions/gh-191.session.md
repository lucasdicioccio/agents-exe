## Summary

The changes implement improved readability for session prints as requested in issue #191:

### 1. New Data Types in `SessionPrint.hs`

**`PrintAmount`** - Controls how much content to display:
- `Lines Int` - Number of lines to keep
- `Chars Int` - Number of characters to keep

**`PrintVisibility`** - Controls visibility with elision support:
- `Hidden` - Don't show content at all
- `Elided PrintAmount PrintAmount` - Show leading and trailing portions, eliding the middle
- `ShownFull` - Show complete content

### 2. The `elideDocument` Function

Intelligently handles overlapping regions:
- For lines: If a document is 7 lines and we request `Elided (Lines 5) (Lines 5)`, all 7 lines display (no duplication)
- For characters: Similar overlap detection prevents duplicate content

### 3. Pretty-printing Tool Arguments

- Tool arguments (JSON from LLM) are now pretty-printed using `aeson-pretty`
- Arguments displayed in `json` markdown code blocks
- Byte count shown in parentheses below the formatted output

### 4. CLI Changes in `Main.hs`

- Changed from boolean `--show-tool-call-results` and `--show-tool-call-arguments` flags to options accepting `hidden|shown|elided`
- `hidden` → `Hidden`
- `shown` → `ShownFull`  
- `elided` → `Elided (Lines 10) (Lines 10)`

### 5. TUI Changes in `Event.hs`

Uses `ShownFull` for both tool call results and arguments when exporting sessions.

### 6. Tests

Added comprehensive tests for `elideDocument` covering:
- Line-based elision with overlap detection
- Character-based elision
- Mixed units (Lines + Chars)
- Edge cases (empty strings, single lines/chars, exact fits)
