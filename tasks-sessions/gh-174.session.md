## Summary

Closes #174 - Support reading another session as input prompt

This PR adds support for injecting session content into prompts with varying verbosity levels via t-shirt sized flags.

### New CLI Options for `run` and `echo-prompt` commands:

- `--session-xs <FILE>` - Only queries and responses (skips tool-only turns)
- `--session-s <FILE>` - Adds thinking process and tool names called  
- `--session-m <FILE>` - Adds statistics about the session
- `--session-l <FILE>` - Adds tool-call results
- `--session-xl <FILE>` - Same as L (complete session)

### Usage Example:

```console
agents-exe echo-prompt -p "roast this session:" -S --session-xs conv.sess-file.json
```

   - Defines `SessionVerbosity` type with 5 levels (XS, S, M, L, XL)

   - Helper functions for filtering tool-only turns and formatting content

2. **`app/Main.hs`**
   - Added `SessionContents FilePath SessionVerbosity` to `PromptScriptDirective`
   - Added 5 new CLI parsers for the session verbosity flags
