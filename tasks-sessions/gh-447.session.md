# Implement Media Attachments and Multi-Modal Responses for LLM Sessions

## Overview

Implements full media attachment support for multi-modal LLM interactions. This supersedes #405 with updated design reflecting current codebase architecture post-OS migration.

**Key Design Principles:**
1. **Backwards compatible** - Only add optional fields, use session versioning for new features
2. **Explicit media types first** - Tools declare `output-media-type` in their schema
3. **Full multi-modal** - Support `MixedResponse` with mixed text + media content parts
4. **LLM-agnostic core** - Session types know nothing about OpenAI/Claude formats
5. **OpenAI format first** - But architecture supports adding other LLM providers later

## Background

Phase 1 (#265) implemented safe UTF-8 decoding with `lenientDecode`, preventing crashes on binary output but rendering binary data as replacement characters (�). This issue implements Phase 2: proper media handling with base64 encoding for LLM consumption.

## Design Decisions

### 1. Core Media Types (LLM-Agnostic)

```haskell
-- System.Agents.Media.Types
module System.Agents.Media.Types where

data MediaType
    = MediaImage ImageType
    | MediaAudio AudioType
    | MediaVideo VideoType
    | MediaApplication ApplicationType
    | MediaText TextSubtype
    deriving (Show, Eq, Generic)

data ImageType = ImagePNG | ImageJPEG | ImageGIF | ImageWebP | ImageSVG
    deriving (Show, Eq, Generic)

-- Media attachment for base64-encoded content
data MediaAttachment = MediaAttachment
    { mediaMimeType :: Text        -- e.g., "image/png"
    , mediaBase64Data :: Text      -- Base64-encoded content (already UTF-8 safe)
    , mediaFilename :: Maybe Text  -- Optional filename
    } deriving (Show, Eq, Generic)

-- Individual part of a mixed response
data ContentPart
    = TextPart Text
    | MediaPart MediaAttachment
    deriving (Show, Eq, Generic)
```

### 2. Extended CallResult

Extend `BlobToolSuccess` with optional media type hint:

```haskell
-- System.Agents.Tools.Base
data CallResult call
    = -- | Successful execution returning raw bytes with optional media type hint
      BlobToolSuccess call ByteString (Maybe MediaType)
    | -- ... rest unchanged
```

The `Maybe MediaType` is **optional** - existing tools continue to work with `Nothing`.

### 3. LLM-Agnostic UserToolResponse

Replace the opaque `newtype` with a proper sum type:

```haskell
-- System.Agents.Session.Types
data UserToolResponse
    = TextResponse Text                          -- Plain UTF-8 text
    | JsonResponse Aeson.Value                   -- Structured JSON data
    | MediaResponse MediaAttachment              -- Single binary media (base64-encoded)
    | MixedResponse [ContentPart]                -- Multi-modal: alternating text and media
    deriving (Show, Eq, Generic)
```

**Why this design:**
- `TextResponse`: Simple text output (most common case)
- `JsonResponse`: Structured data from API tools, SQLite, etc.
- `MediaResponse`: Single image/audio/video (convenience for simple cases)
- `MixedResponse`: Full multi-modal (e.g., "Here's the chart:" + image + "As you can see...")

### 4. Session Versioning (Backwards Compatibility)

Add optional version field to Session:

```haskell
-- System.Agents.Session.Types
data Session = Session
    { turns :: [Turn]
    , sessionId :: SessionId
    , forkedFromSessionId :: Maybe SessionId
    , turnId :: TurnId
    , sessionVersion :: Maybe Int  -- NEW: Nothing = legacy (pre-media), Just 1 = media support
    }
    deriving (Show, Ord, Eq, Generic)
```

Custom JSON instances handle both formats:
- Missing `sessionVersion` → `Nothing` (legacy session)
- Present `sessionVersion` → `Just n`

### 5. Tool Output Declaration

Add optional `output-media-type` to ScriptInfo:

```haskell
-- System.Agents.Tools.ScriptTypes
data ScriptInfo = ScriptInfo
    { scriptArgs :: [ScriptArg]
    , scriptSlug :: Text
    , scriptDescription :: Text
    , scriptEmptyResultBehavior :: Maybe ScriptEmptyResultBehavior
    , scriptOutputMediaType :: Maybe Text  -- NEW: e.g., "image/png", "application/pdf"
    }
    deriving (Show, Eq, Ord)
```

JSON format:
```json
{
  "slug": "generate_chart",
  "description": "Generates a chart image",
  "args": [...],
  "output-media-type": "image/png"
}
```

### 6. LLM-Specific Conversion (OpenAI First)

Conversion from `UserToolResponse` to LLM-specific format happens in `System.Agents.Session.OpenAI`:

```haskell
-- System.Agents.Session.OpenAI
toolResponseToMessages :: (LlmToolCall, UserToolResponse) -> [Aeson.Value]
toolResponseToMessages (call, response) =
    case response of
        TextResponse txt -> 
            -- Standard OpenAI text tool response
            [Aeson.object ["role" .= ("tool" :: Text), "tool_call_id" .= getToolCallId call, "content" .= txt]]
        
        JsonResponse val -> 
            -- Encode JSON as text
            [Aeson.object ["role" .= ("tool" :: Text), "tool_call_id" .= getToolCallId call, "content" .= encodeJson val]]
        
        MediaResponse media -> 
            -- OpenAI image_url format with base64 data URL
            [Aeson.object ["role" .= ("tool" :: Text), "tool_call_id" .= getToolCallId call, "content" .= mediaToOpenAI media]]
        
        MixedResponse parts -> 
            -- OpenAI content array format
            [Aeson.object ["role" .= ("tool" :: Text), "tool_call_id" .= getToolCallId call, "content" .= partsToOpenAI parts]]
```

```haskell
callResultToUserToolResponse :: forall toolcall. toolcall -> CallResult toolcall -> UserToolResponse
callResultToUserToolResponse _ result =
    case result of
        BlobToolSuccess _ bytes (Just mediaType) ->
            -- Known media type - encode as base64 MediaResponse
            MediaResponse $ MediaAttachment
                { mediaMimeType = mediaTypeToMime mediaType
                , mediaBase64Data = Text.decodeUtf8 (B64.encode bytes)
                , mediaFilename = Nothing
                }
        
        BlobToolSuccess _ bytes Nothing ->
            -- Unknown type - try UTF-8 first
            case Text.decodeUtf8' bytes of
                Right txt -> TextResponse txt  -- Valid UTF-8
                Left _ -> 
                    -- Binary data without media type - encode as generic binary
                    MediaResponse $ MediaAttachment
                        { mediaMimeType = "application/octet-stream"
                        , mediaBase64Data = Text.decodeUtf8 (B64.encode bytes)
                        , mediaFilename = Nothing
                        }
        
        -- Structured results become JsonResponse
        OpenAPIToolResult _ toolResult -> JsonResponse $ Aeson.toJSON toolResult
        PostgRESToolResult _ toolResult -> JsonResponse $ Aeson.toJSON toolResult
        McpToolResult _ res -> JsonResponse $ Aeson.toJSON res
        SqliteToolResult _ toolResult -> JsonResponse $ Aeson.toJSON toolResult
        SystemToolResult _ toolResult -> JsonResponse $ Aeson.toJSON toolResult
        DeveloperToolResult _ valResult -> JsonResponse $ Aeson.toJSON valResult
        -- ... etc for other structured results
        
        -- Error cases become TextResponse
        ToolNotFound _ -> TextResponse "Tool not found"
        BashToolError _ err -> TextResponse $ Text.pack $ show err
        -- ... etc
```

- [ ] `BlobToolSuccess` with `Just MediaType` → `MediaResponse`

-- Helper to convert MediaAttachment to OpenAI format
mediaToOpenAI :: MediaAttachment -> Aeson.Value
mediaToOpenAI media = 
    Aeson.object

        , "image_url" .= Aeson.object 
            [ "url" .= ("data:" <> media.mediaMimeType <> ";base64," <> media.mediaBase64Data)
            ]

-- Helper for ContentPart
contentPartToOpenAI :: ContentPart -> Aeson.Value
contentPartToOpenAI (TextPart t) = 
    Aeson.object ["type" .= ("text" :: Text), "text" .= t]
contentPartToOpenAI (MediaPart m) = mediaToOpenAI m

-- Update main conversion
toolResponseToMessages :: (LlmToolCall, UserToolResponse) -> [Aeson.Value]
toolResponseToMessages (LlmToolCall callVal, response) =
    let toolCallId = extractToolCallId callVal

            TextResponse txt -> Aeson.String txt

        [ "role" .= ("tool" :: Text)
        , "tool_call_id" .= toolCallId
        , "content" .= content
        ]]
