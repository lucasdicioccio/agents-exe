## Key Design Decisions

1. **Metadata Format**: Embedded as HTML comments in issue bodies to be hidden from normal rendering but machine-parseable:
   ```markdown
   <!-- AGENTS_METADATA_START
   {"feature_branch":"gh-123","dependencies":["#125"],"final_tag":"not-final"}
   AGENTS_METADATA_END -->
   ```

2. **Branch Naming Convention**: `gh-{original_issue_number}` (e.g., `gh-123`)

3. **Label Strategy**: 
   - No dependencies → `agents/to-be-taken`
   - Has dependencies → `agents/wait`

4. **Final Merge Issue**: Special issue with `final_tag: "final"` that merges the feature branch into main

## Workflow Flow

```
issue-123 (complex feature)
    ↓ (architect agent)
    ├── issue-125: task A (branch: gh-123, deps: "", tag: not-final)
    ├── issue-126: task B (branch: gh-123, deps: "#125", tag: not-final)  
    ├── issue-127: task C (branch: gh-123, deps: "#126", tag: not-final)
    └── issue-128: final merge (branch: gh-123, deps: "#125,#126,#127", tag: final)
```

Each issue can be worked on independently (when dependencies are met), with all PRs targeting the `gh-123` branch, and the final issue merging `gh-123` into `main`.
