Implement a status bar notification system that allows:

## Tradeoffs Considered

1. **Timeout vs Manual Dismiss**: Using timeout-based auto-clear (5s) keeps the UI uncluttered but may miss important errors. Consider adding a key binding (e.g., `Ctrl+l` or `Esc`) to manually clear or view message history.

2. **Single vs Multiple Messages**: This design shows only one message at a time. A queue-based system could show multiple messages but adds complexity.

3. **Position**: Bottom placement is standard for status bars but could conflict with future footers. Consider making position configurable or using a popup-style notification area.
