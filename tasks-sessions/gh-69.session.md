# Implement Archive Export/Import (Tar/Zip)

## Overview
Implement export and import functionality for tar and zip archive formats.

## Background
Building on the types defined in the Export/Import Types issue, this implements the actual archive creation and extraction.

## Requirements

### 1. Create Archive Operations Module
Create `src/System/Agents/ExportImport/Archive.hs`:

```haskell
module System.Agents.ExportImport.Archive where

import qualified Codec.Archive.Tar as Tar
import qualified Codec.Archive.Zip as Zip
import System.Agents.ExportImport.Types

-- | Supported archive formats
data ArchiveFormat = TarFormat | TarGzFormat | ZipFormat
    deriving (Show, Eq)

-- | Export a package to an archive file
exportToArchive :: ExportPackage -> ArchiveFormat -> FilePath -> IO ()

-- | Import a package from an archive file
importFromArchive :: FilePath -> IO (Either ImportError ExportPackage)

-- | Detect archive format from file extension
detectArchiveFormat :: FilePath -> Maybe ArchiveFormat

-- | Errors that can occur during import
data ImportError
    = ArchiveReadError String
    | ArchiveFormatError String
    | PackageParseError String
    | NamespaceConflictError Text
    deriving (Show)
```

### 2. Archive Structure
The archive should have this layout:
```
agents-export/
├── manifest.json          # PackageMetadata + list of contents
├── agents/
│   ├── agent1.json        # Agent config (with namespace prefix if applicable)
│   └── team1.agent2.json  # Namespaced agent
├── tools/
│   ├── tool1/
│   │   ├── script         # The actual executable
│   │   └── metadata.json  # Tool metadata (slug, description, args)
│   └── team1.tool2/
│       ├── script
│       └── metadata.json
└── mcp-servers/
    └── mcp1.json          # MCP server configurations
```

### 3. Export Implementation Details
- Create temporary directory structure
- Write all agents as individual JSON files
- Copy tool scripts with permissions preserved
- Write manifest.json with metadata
- Pack into selected archive format

### 4. Import Implementation Details

- Extract to temporary directory
- Parse manifest.json

- Verify no namespace conflicts (for namespaced imports)
- Return ExportPackage

## Trade-offs

**Archive Format Support**
- Tar: Unix-native, preserves permissions well, standard for git bundles
- Zip: Cross-platform, Windows-friendly, widely supported
- Tar.gz: Compressed tar, good for distribution
- **Decision**: Support all three, with Tar as default for Unix systems

**Temporary Directory Handling**
- System temp dir: Auto-cleanup on reboot, shared space
- Project-local .tmp: Survives reboot, easier debugging
- **Decision**: Use system temp with explicit cleanup, option for local temp with --debug flag
