### 💬 User Query

# Add command to inspect the tool schemas of the converted tools for toolboxes

Today debugging invalid tools is a bit difficult.

We'd benefit from a command that dumps (in markdown format preferably) the configurations of agents along with the tools.
Tool schemas sent to LLMs should be visible in details tags.

Ideally we would repurpose the `check` command to dump more information about the tools of every loaded agent (or errors otherwise).


This way we can run an optional `--tools` argument.

```
`check` (unchanged)
`check --tools=[none|list|agents-exe|openai]`
```

- `none` => unchanged, default
- `list` => give the tool list in textual form
- `agents-exe` => give the JSON description of the internal representation of the tooldescription
- `openai` => give the OpenAI.LLM JSON representation that is serialized in the LLM completion calls

## Summary

Added a `--tools` argument to the `check` command that allows users to inspect tool schemas in different formats.

### Changes Made

**1. `src/System/Agents/CLI/Check.hs`**
- Added `ToolsOutputMode` data type with four modes: `ToolsNone`, `ToolsList`, `ToolsAgentsExe`, `ToolsOpenAI`
- Added `CheckOptions` type to hold the tools output mode
- Modified `handleCheck` to accept `CheckOptions` parameter
- Modified `printAgentCheck` to handle the different output modes:
  - `ToolsNone`: Default behavior (unchanged)
  - `ToolsList`: Print markdown list of tool names and descriptions
  - `ToolsAgentsExe`: Print internal representation as JSON in `<details>` tags
  - `ToolsOpenAI`: Print OpenAI LLM schema as JSON in `<details>` tags

**2. `app/Main.hs`**
- Updated `Command` type to include `Check CheckOptions` instead of plain `Check`
- Added `parseToolsOption` parser with `--tools` flag
- Added `parseCheckOptions` parser
- Updated `parseCheckCommand` to use the new options parser
- Updated `runCommand` to pass `CheckOptions` to `handleCheck`
- Added program description for the check command

**3. `docs/binary-tool.md`**
- Added documentation for the new `--tools` option explaining all modes

### Usage

```bash
# Default behavior - unchanged
agents-exe check

# List tools in textual form
agents-exe check --tools=list

# Show internal agents-exe JSON representation
agents-exe check --tools=agents-exe

# Show OpenAI LLM JSON representation
agents-exe check --tools=openai
```

The output for `agents-exe` and `openai` modes is wrapped in markdown `<details>` tags, making it suitable for inclusion in documentation or issue reports.
