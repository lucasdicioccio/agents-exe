# OS Model: Conversation and Lineage Tracking

Depends-on: #1
Base-branch: gh-os-architecture
Final: false

## Overview

Define types for conversation/thread tracking and lineage tracking for nested tool calls. This enables thorough tracing of agent interactions and tool call hierarchies.

## Background

The system needs to track:
- **Conversations**: Top-level interactions between users and agents
- **Turns**: Individual steps within a conversation
- **Tool Calls**: Individual tool invocations, potentially nested
- **Lineage**: Full call chain for debugging and accounting

This supports:
- Recursive/nested agent calls with proper depth tracking
- Conversation forking (exploring multiple responses)
- Complete audit trails for debugging

1. **Branching Conversations**: Tree-structured conversations add complexity but enable exploring multiple responses from the same point.

2. **Nested Tool Calls**: Maximum depth limits should be enforced to prevent infinite recursion. Default: 10 levels.

### Key Design Features

- **ECS Pattern**: All types implement the `Component` typeclass with unique IDs (30-38)
- **Phantom Types**: Entity IDs are phantom-typed for type safety
- **Thread Safety**: `ConversationState` uses TVar for lastActivity updates
- **Branching Support**: Turns have `turnParentTurnId` for tree-structured conversations
- **Nested Calls**: Tool calls have `tcParentCallId` for recursive agent tracking
- **Complete Audit Trail**: Lineage tracks full call chain for debugging
