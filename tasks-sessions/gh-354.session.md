# OS Model: TUI and OneShot Interface Adaptation

Depends-on: #353
Base-branch: gh-os-architecture
Final: false

## Overview

Adapt the TUI and OneShot interfaces to work with the new OS model through the compatibility layer. This ensures existing user interfaces continue to work during the migration.

## Background

Current interfaces:
- **OneShot**: Single conversation execution, batch mode
- **TUI**: Interactive multi-agent terminal UI

Both currently use `Runtime` directly. We need to:
1. Make them work with the compatibility layer
2. Eventually refactor to use OS natively for better features
