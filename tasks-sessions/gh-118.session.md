## Summary: Create QA agent for empirical testing of agents-exe changes

The issue details the creation of a QA agent at `qa-agent/kimi-qa.json` with three tools:

1. **`cabal_build`** - Runs `cabal build` to verify code compiles
2. **`cabal_test`** - Runs `cabal test` to verify the test suite passes
3. **`run_agents_exe`** - Runs `cabal run -- agents-exe <args>` for testing various CLI commands

### Two QA Categories Supported:

**Code updates to agents-exe itself:**
- Build verification via `cabal_build`
- Test verification via `cabal_test`
- CLI command testing via `run_agents_exe`

**Prompt/tool updates:**
- Test agents with specific prompts using `run_agents_exe run --prompt ...`
- Validate configurations with `run_agents_exe check`
- Verify tool discovery with `run_agents_exe describe`
- Test project initialization with `run_agents_exe init`

### Key Tradeoffs Mentioned:
- Coverage vs. speed (full builds/tests are slow)
- No automatic isolation/cleanup for stateful commands
- Requires valid API keys for testing agent execution
- Runs in current directory context (not containerized)

Relates to #118.
