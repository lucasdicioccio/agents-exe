# Cross-Conversation References

Depends-on: 
Base-branch: gh-508
Final: false

## Overview
Enable citing and referencing previous conversations, creating a linked knowledge graph across multiple conversation sessions.

## Motivation
Users often have related conversations across different sessions. Cross-referencing enables:
- **Context Preservation**: Reference previous discussions
- **Knowledge Graph**: Build connections between related topics
- **Continuity**: Pick up threads from earlier conversations
- **Search**: Find related conversations via references

## Proposed Solution
Implement a referencing system that allows messages to cite other conversations and messages.

### Key Features
1. **Reference Syntax**: Special syntax to cite other conversations
2. **Inline Display**: Show referenced content inline or as preview
3. **Bidirectional Links**: Navigate between linked conversations
4. **Reference Graph**: Visualize conversation relationships

### Reference Syntax
```markdown
# In a message, reference another conversation
See our previous discussion [[conv:uuid#msg-id]] about this topic.

# Reference a specific message
As we established in [[conv:abc-123#msg-456]], the approach is...

# Auto-complete suggestions when typing [[
```

### Implementation Sketch
```haskell
-- Reference to another conversation/message
data ConversationRef = ConversationRef {
    targetConversation :: ConversationId,
    targetMessage :: Maybe MessageId,
    refType :: RefType  -- Cite | Reply | Related
}

-- Render reference inline
renderReference :: ConversationRef -> IO ReferenceView

-- Get related conversations
getRelatedConversations :: ConversationId -> IO [ConversationRef]

-- Search conversations by reference
searchByReference :: Text -> IO [Conversation]
```
