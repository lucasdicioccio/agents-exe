# Add comprehensive json.encode and json.decode tests to LuaToolboxTests

---
## Problem Statement

The current `LuaToolboxTests.hs` only verifies that the `json` module exists (`testJsonAvailable`), but does not test the actual `json.encode` and `json.decode` functions. We need to add tests that verify these functions work correctly for various data types.

## Proposed Solution

Add new test cases to `moduleTests` in `test/LuaToolboxTests.hs` to comprehensively test JSON encoding and decoding operations.

## Test Cases to Implement

### json.encode tests:
1. **Encode a simple table (object)** - `{name = "test", value = 42}` should encode to valid JSON
2. **Encode an array** - `{1, 2, 3}` should encode to `[1,2,3]`
3. **Encode nested tables** - `{person = {name = "John", age = 30}}`
4. **Encode with mixed types** - strings, numbers, booleans, nil
5. **Encode empty table** - `{}` should encode appropriately

### json.decode tests:
1. **Decode JSON object to table** - `{"key": "value"}` should decode correctly
2. **Decode JSON array** - `[1, 2, 3]` should decode to Lua array
3. **Decode nested JSON** - complex nested objects
4. **Decode with error handling** - invalid JSON should return nil and error message

### Round-trip tests (encode then decode):
1. **Encode and decode should preserve data** - verify `json.decode(json.encode(table))` equals original table
2. **Test with various data types** - strings, numbers, booleans, arrays, objects

## Implementation Notes

The tests should use the existing `withTestToolbox` helper and `LuaToolbox.executeScript` function. Use patterns like:

```haskell
testJsonEncodeObject :: Assertion
testJsonEncodeObject = withTestToolbox $ \box -> do
    result <- LuaToolbox.executeScript box $
        "local data = {name = \"test\", count = 42}\n" ++
        "return json.encode(data)"
    case result of
        Left err -> assertFailure $ show err
        Right execResult ->
            -- Verify result is a valid JSON string
            execResult.resultValues @?= [Aeson.String "{\"name\":\"test\",\"count\":42}"]
```
