# Add `agents-exe paths` command to show important configuration paths

Depends-on: 
Base-branch: gh-189
Final: false

## Summary

Add a new `agents-exe paths` command that displays all important paths used during configuration loading. This helps users debug configuration issues, especially when running outside a project.

## Motivation

When running `agents-exe` outside a project directory, configuration is loaded from `~/.config/agents-exe/default`. Users currently have no easy way to verify:
- Which configuration file was loaded
- Where agent files are being loaded from
- Where the API keys file is located
- Where session files would be stored

## Proposed Solution

Add a new `paths` subcommand that outputs structured information about:

1. **Configuration paths**
   - The `agents-exe.cfg.json` path (if found via directory traversal)
   - The default config directory (`~/.config/agents-exe`)
   - The active config directory being used

2. **Agent file paths**
   - All agent JSON files that were loaded
   - Their source directories

3. **Key file path**
   - Location of the API keys file

4. **Session storage path**
   - The session file prefix/directory (if configured)

## Output Format

The command should output either:
- Human-readable text (default)
- JSON format (with `--json` flag for programmatic use)

Example output:
```
Configuration:
  Config file found: /home/user/myproject/agents-exe.cfg.json
  Config directory: /home/user/myproject
  Default config dir: /home/user/.config/agents-exe

Agents:
  - /home/user/myproject/agent.json (slug: main-agent)
  - /home/user/myproject/agents/helper.json (slug: helper-agent)

API Keys:
  /home/user/myproject/secret-keys

Session Storage:
  Prefix: /home/user/myproject/sessions/
```
