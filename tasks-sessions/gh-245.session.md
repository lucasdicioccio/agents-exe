## Key Design Decisions

1. **Capability-based configuration**: Users enable specific capabilities (`date`, `operating-system`, `env-vars`, `running-user`, `hostname`, `working-directory`, `process-info`, `uptime`) via the config for security control.

2. **Single tool interface**: Exposes one LLM tool `system_{name}_system_info` with a `capability` parameter rather than cluttering the tool namespace with 8 separate tools.

3. **Linux-focused**: Initial implementation targets Linux systems using `/proc`, `/etc/os-release`, and POSIX APIs.

4. **Read-only**: System tools gather information but cannot modify the system.

5. **Env var filtering**: Optional substring filter for environment variables to help prevent accidental secret exposure.
