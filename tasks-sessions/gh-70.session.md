# Implement Git-Based Export/Import with Namespacing

## Overview
Implement export to and import from git repositories, supporting multiple agent configurations within a single repository using namespacing.

## Background
Unlike archives, git repositories allow:
- Version history of configurations
- Multiple teams to share a single "agent registry" repository
- Namespacing: `team-a/project-x/agent-name` or `org/department/agent`

## Requirements

### 1. Create Git Operations Module
Create `src/System/Agents/ExportImport/Git.hs`:

```haskell
module System.Agents.ExportImport.Git where

import System.Agents.ExportImport.Types

data GitUrl = GitUrl
    { gitRemote :: Text
    , gitBranch :: Maybe Text
    , gitPath :: Maybe Namespace  -- optional subpath within repo
    }

data GitExportOptions = GitExportOptions
    { gitCommitMessage :: Text
    , gitTag :: Maybe Text
    , gitPush :: Bool
    }

data GitImportOptions = GitImportOptions
    { gitRef :: Maybe Text  -- branch, tag, or commit
    , gitSparsePaths :: [Namespace]  -- import only specific namespaces
    }

-- | Export package to a git repository
exportToGit :: ExportPackage -> GitUrl -> GitExportOptions -> IO (Either GitError ())

-- | Import from a git repository
importFromGit :: GitUrl -> GitImportOptions -> IO (Either GitError ExportPackage)

-- | List available namespaces in a git repository
listGitNamespaces :: GitUrl -> IO (Either GitError [Namespace])

data GitError
    = GitCloneError String
    | GitCheckoutError String
    | GitCommitError String
    | GitPushError String
    | GitParseError String
    | GitNamespaceError String
    deriving (Show)
```

### 3. Index File Format
`.agents-index.json` at repository root:


### 5. Git Operations Strategy
1. Clone repository to temp directory (or use existing local clone)
2. Checkout specified ref (branch/tag/commit)
3. Parse `.agents-index.json` if present, or scan directory structure
4. For export: write files, commit, optionally push
5. For import: read files, build ExportPackage

### 6. Sparse Import Support
Allow importing only specific namespaces:
```bash
# Import only team-a agents
agents-exe import git --url https://github.com/org/agents.git --namespace team-a

# Import specific agent
agents-exe import git --url https://github.com/org/agents.git --namespace team-a.project-x.my-agent
```

## Trade-offs

**Git Library vs. CLI**
- Haskell `git` library: Pure Haskell, dependency-heavy
- Shell out to `git` CLI: Requires git installed, simpler
- **Decision**: Shell out to git CLI (git is standard tool for target users)

**Index File vs. Directory Scanning**
- Index file: Faster lookups, explicit registration, metadata
- Directory scanning: No extra file, automatic discovery
- **Decision**: Support both - use index if present, scan if not

**Single vs. Multiple Commits**
- Single commit: Atomic operation, simpler history
- Multiple commits: One per agent/tool, granular history
- **Decision**: Single commit per export operation, detailed commit message
