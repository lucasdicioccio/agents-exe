## Specification Content

The documentation should cover the bash-tools protocol:

### Tool Script Protocol

A bash-tool is an executable script that implements two commands:

1. **`describe`** - Returns JSON metadata about the tool
2. **`run`** - Executes the tool with provided arguments

### Example Tool Script

```bash
#!/bin/bash
# Example: my-tool

case "$1" in
  describe)
    cat <<'EOF'
{
  "args": [
    {
      "name": "input",
      "description": "The input to process",
      "type": "string",
      "backing_type": "string",
      "arity": "single",
      "mode": "dashdashspace"
    }
  ],
  "slug": "my-tool",
  "description": "Processes input and returns result",
  "empty-result": {"tag": "AddMessage", "contents": "--no output--"}
}
EOF
    ;;
  run)
    # Process arguments and produce output
    echo "Processing: $2"
    ;;
  *)
    echo "Unknown command: $1" >&2
    exit 1
    ;;
esac
```

### ScriptArg Fields

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Argument identifier (used in JSON) |
| `description` | string | Human-readable description |
| `type` | string | LLM-facing type (e.g., "string", "integer") |
| `backing_type` | string | Internal type (e.g., "string", "number") |
| `arity` | string | "single" or "optional" |
| `mode` | string | "stdin", "positional", "dashdashspace", or "dashdashequal" |

### Environment Variables

When `run` is called, the script receives session context via environment variables:

- `AGENT_SESSION_ID` - Current session UUID
- `AGENT_CONVERSATION_ID` - Conversation UUID  
- `AGENT_TURN_ID` - Current turn UUID
- `AGENT_AGENT_ID` - Agent UUID (if available)
- `AGENT_SESSION_JSON` - Full session as JSON (if requested)

```haskell
bashToolsDocumentation :: Text
bashToolsDocumentation = Text.unlines
    [ "# Bash Tools Specification"
    , ""
    , "A bash-tool is an executable script that implements two commands:"
    , "..."
    ]

handleSpec :: SpecOptions -> IO ()
handleSpec opts = case specTopic opts of
    BashToolsSpec -> Text.putStrLn bashToolsDocumentation
```

### CLI Parser Addition

```haskell
parseSpecCommand :: Parser Command
parseSpecCommand = Spec <$> parseSpecOptions

parseSpecOptions :: Parser SpecOptions
parseSpecOptions = SpecOptions <$> argument parseTopic (metavar "TOPIC")
  where
    parseTopic "bash-tools" = Just BashToolsSpec
    parseTopic _ = Nothing
```

Add to command subparser:
