# Support a keymap for the TUI event bindings

# context and need
Today TUI events are bound using a set of hardcoded pattern matches in the TUI Event.hs file.
We should allow configure a key-map to overload the bindings.

# requirements
The key-map should be a json file mapping event shorthand names with key-names and modifiers.
The agents-exe.cfg.json then should be allowed to reference an alternate keymap.

# tasks
Implement a KeyMapping data-type in TUI.Types for the key-map with to/from json instances.
Add a defaultKeyMapping using current logic.
Modify the TUI State and Events handler to use the KeyMapping.
Modify the agents-exe.cfg.json format to allow an optional alternate KeyMapping to be loaded.
