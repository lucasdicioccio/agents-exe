## Feature Request

Add builtin system tools for reading and writing specific line ranges in files, based on the existing Rust implementations in `resources/read_file_range.rs` and `resources/write_file_range.rs`.

## Motivation

The current Rust tools (`read_file_range` and `write_file_range`) provide powerful file editing capabilities that are essential for development workflows. Making these available as builtin tools will:
- Provide consistent file editing capabilities across all agents
- Eliminate the need to bundle external binaries
- Enable better integration with the agent framework (tracing, error handling, etc.)

## Proposed Design

Extend the **DeveloperToolbox** with two new capabilities:

### New Capabilities

| Capability | Description |
|------------|-------------|
| `DevToolReadFileRange` | Read specific line ranges from a file |
| `DevToolWriteFileRange` | Replace line ranges in a file with new content |

### Configuration Example

```json
{
  "tag": "DeveloperToolbox",
  "contents": {
    "name": "developer",
    "description": "Development utilities with file editing",
    "capabilities": [
      "validate-tool",
      "scaffold-agent",
      "read-file-range",
      "write-file-range"
    ]
  }
}
```

### Tool Interfaces

#### read_file_range

```json
{
  "capability": "read-file-range",
  "path": "/path/to/file",
  "ranges": "1-10,20-30"
}
```

**Parameters:**
- `path` (string, required): Path to the file to read
- `ranges` (string, optional): Comma-separated line ranges (e.g., `"1-10"`, `"1-5,20-30"`, `"5"`). Omit to read entire file.

**Returns:** Lines with line numbers prepended as `{line_num}\t{line_content}`

**Example output:**
```
1	def hello():
2	    print("Hello, World!")
3	    return True
```

#### write_file_range

```json
{
  "capability": "write-file-range",
  "path": "/path/to/file",
  "ranges": "head,10-12,tail"
}
```

Content provided via the `content` parameter (or stdin-equivalent in the tool interface).

**Parameters:**
- `path` (string, required): Path to the file to modify
- `ranges` (string, required): Comma-separated ranges. Supports:
  - Numeric ranges: `"5"`, `"1-10"`, `"1-5,20-30"`
  - Special locations: `"head"` (prepend before line 1), `"tail"` (append after last line)
- `content` (string, required): Replacement text. For multiple ranges, separate blocks with a line containing only `"---"`.

**Behavior:**
- Multiple ranges are applied bottom-to-top so earlier line numbers stay valid
- Preserves trailing newline if original file had one
- Creates file if it doesn't exist (for `head`/`tail` operations)

### Data Types

```haskell
-- Range specification (mirrors Rust RangeSpec)
data RangeSpec
    = Lines (Int, Int)  -- 1-based, inclusive line range
    | Head              -- Before line 1
    | Tail              -- After last line
    deriving (Show, Eq)

-- Result types
data ReadFileRangeResult = ReadFileRangeResult
    { readFilePath :: FilePath
    , readFileContent :: Text
    , readFileLinesRead :: Int
    }

data WriteFileRangeResult = WriteFileRangeResult
    { writeFilePath :: FilePath
    , writeFileRangesModified :: Int
    , writeFileLinesWritten :: Int
    }
```

### Range Parsing

Parse `"1-10,20-30"` or `"5"` or `"head,5-10,tail"` into `[RangeSpec]`:
- Single number `n` → `Lines (n, n)`
- Range `a-b` → `Lines (a, b)`
- `"head"` → `Head`
- `"tail"` → `Tail`

### Error Handling

| Error Condition | Result |
|-----------------|--------|
| File not found | Return error with file path |
| Invalid range format | Return error with offending range |
| Range out of bounds | Return error (for write) or truncate (for read) |
| Permission denied | Return error with details |
