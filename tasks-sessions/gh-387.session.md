# Update TUI to display nested/recursive agent conversations

Depends-on: #385,#386

## Overview

Building on the session persistence (#385) and tracing (#386) work, this issue updates the TUI to visually display nested/recursive agent conversations. Users should be able to see when an agent calls another agent, view the sub-agent's conversation, and navigate the parent-child hierarchy.

## Current TUI Structure

From `src/System/Agents/TUI/Render.hs`:

```haskell
-- Conversations are rendered as a flat list
render_conversationItem :: TuiState -> Bool -> Conversation -> Widget N
render_conversationItem st _ conv =
    let indicator = case conversationStatus conv of
            ConversationStatus_Active -> "⟳ "
            ConversationStatus_WaitingForInput -> "● "
            ConversationStatus_Paused -> "⏸ "
        baseText = indicator <> Text.take 18 (conversationName conv)
        -- Only shows turn count, no child info
        turnCount = case conversationSession conv of
            Nothing -> 0
            Just session -> length session.turns
     in txt $ " " <> baseText
```

## Proposed UI Changes

### 1. Conversation Tree View

Display conversations hierarchically:

```
Conversations
  ⟳ main-agent (12 turns)
    ● sub-agent:architect (5 turns)  
    ● sub-agent:documenter (3 turns)
  ● other-conversation (8 turns)
```

### 2. Enhanced Conversation List Item

```haskell
render_conversationItem :: TuiState -> Bool -> Conversation -> Widget N
render_conversationItem st isSelected conv =
    let -- Get child count from SessionStore
        childCount = getChildConversationCount conv
        
        -- Indentation based on depth
        depth = getConversationDepth conv
        indent = Text.replicate (depth * 2) " "
        
        -- Indicator with child marker
        indicator = case conversationStatus conv of
            ConversationStatus_Active -> "⟳ "
            ConversationStatus_WaitingForInput -> 
                if childCount > 0 then "▼ " else "● "
            ConversationStatus_Paused -> "⏸ "
        
        -- Show child indicator
        childInfo = if childCount > 0
            then " [" <> Text.pack (show childCount) <> " sub]"
            else ""
            
        baseText = indent <> indicator <> conversationName conv <> childInfo
     in txt $ " " <> baseText
```

## Tradeoffs

1. **Lazy vs Eager Loading**: Load child conversations on demand vs. all at once
2. **Inline vs Separate View**: Show sub-agents inline vs. separate panel
3. **Tree vs Flat List**: Tree is clearer but harder to navigate with many items

## Performance Considerations

- Child conversation counts should be cached to avoid repeated queries
- Tree structure should be built incrementally as data arrives
- Consider virtual scrolling for large conversation trees
