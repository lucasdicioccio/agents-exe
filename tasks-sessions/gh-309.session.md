## Summary

Added a pause conversation mechanism to the TUI that prevents consuming too many tokens by blocking step iteration until unpaused.

### Changes Made:

**1. Types.hs (`ConversationStatus`)**:
- Added `ConversationStatus_Paused` variant to track paused state
- Added `corePausedConversations :: Set ConversationId` to the `Core` state for shared pause tracking across threads

**2. Event.hs (Pause Handling)**:
- Added `Ctrl+Space` keyboard shortcut to toggle pause/unpause
- `handleTogglePauseConversation`: Toggles pause state in Core and updates conversation status
- Pause mechanism in `runConversation`: Before each agent step, checks if conversation is paused and blocks with polling loop (100ms intervals) until unpaused

**3. Render.hs (Visual Indicator)**:
- Added "⏸ " pause indicator prefix for paused conversations in the conversation list
- Added `pausedAttr` with yellow/bold styling for visibility
- Updated shortcuts help bar to show `Ctrl+Space: pause`

### How It Works:

1. **Pause**: User presses `Ctrl+Space` on a conversation
   - Conversation ID added to `corePausedConversations` set
   - Status set to `ConversationStatus_Paused`
   - Status message shows "Paused: <conversation name>"

2. **Block**: Before each agent step, checks if conversation is in paused set
   - If paused, blocks in polling loop until removed from set

3. **Unpause**: User presses `Ctrl+Space` again on paused conversation
   - Conversation ID removed from `corePausedConversations`
   - Status set to `ConversationStatus_WaitingForInput`
   - Agent loop resumes from blocked step

4. **Visual**: Paused conversations show "⏸ " prefix in yellow/bold in the Conversations box
