### 💬 User Query

# TUI Draft Messages

Depends-on: 
Base-branch: gh-505
Final: false

## Overview
Add ability to compose and save message drafts in the TUI without sending immediately, allowing users to prepare messages across multiple sessions.

## Motivation
Users often need to:
- Compose long messages over time
- Research information before sending
- Review and edit before committing
- Maintain multiple pending responses

Draft support improves the user experience for complex interactions.

## Proposed Solution
Implement a draft message system in the TUI with save/load capabilities.

### Key Features
1. **Draft Composition**: Write messages without immediate send
2. **Auto-save**: Periodically save draft to prevent loss
3. **Draft List**: View and manage multiple drafts
4. **Resume Editing**: Continue from where you left off
5. **Send or Discard**: Choose to send or delete drafts

### UI Flow
```
[Conversation View]
┌─────────────────────────────────────┐
│ Agent: How can I help?              │
│                                     │
│ [Drafts (3)] [Compose]              │
└─────────────────────────────────────┘

[Draft List]
┌─────────────────────────────────────┐
│ 📄 Draft 1 - "Project ideas..."     │
│ 📄 Draft 2 - "Question about..."    │
│ 📄 Draft 3 - "Follow up on..."      │
│                                     │
│ [New Draft] [Delete All]            │
└─────────────────────────────────────┘
```

### Implementation Sketch
```haskell
-- Draft message
data DraftMessage = DraftMessage {
    draftId :: UUID,
    conversationId :: ConversationId,
    content :: Text,
    createdAt :: UTCTime,
    updatedAt :: UTCTime
}

-- Draft operations
saveDraft :: DraftMessage -> IO ()
listDrafts :: ConversationId -> IO [DraftMessage]
loadDraft :: DraftId -> IO (Maybe DraftMessage)
sendDraft :: DraftId -> IO ()
deleteDraft :: DraftId -> IO ()
```

- [ ] Can view list of saved drafts

- [ ] Can delete drafts
