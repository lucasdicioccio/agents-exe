# Fix extraAgents self-reference confusion: documentation, error messages, and infinite loop protection

Depends-on: 
Base-branch: main
Final: false

## Problem

When defining an agent with `extraAgents` that references itself (or another agent), users encounter confusing errors or infinite loops.

### Example problematic configuration:

```json
{
  "tag": "OpenAIAgentDescription",
  "contents": {
    "slug": "local:lrm",
    ...
    "extraAgents": [
      {
        "slug": "self-reference",
        "path": "kimi-07.err1.json"
      }
    ]
  }
}
```

### Current behavior:

1. **ReferenceError**: `MissingAgentReference "local:lrm" "localdev-agents/kimi-07.err1.json" "self-reference"`
2. OR **Infinite loop** when the slug happens to match

### Root Cause

The `extraAgentSlug` field is NOT an alias - it must match the actual `slug` defined in the target file. The documentation doesn't clearly explain this, and the error message is confusing.

## Solution

This issue addresses three aspects:

### 1. Documentation Improvements

Update `docs/advanced-configuration.md` to clarify:
- The `slug` in `extraAgents` must match the target agent's actual `slug` (not an arbitrary alias)
- Add a clear warning box explaining this relationship
- Fix the self-reference example to explicitly show this requirement

**Key clarification to add:**
```markdown
> **Important**: The `slug` field in `extraAgents` must match the target agent's 
> actual `slug` as defined in its JSON file. It is NOT an alias. The system uses 
> this slug to look up the agent in the registry after all configurations are loaded.
```

### 2. Improved Error Messages

In `src/System/Agents/AgentTree.hs`, enhance the `MissingAgentReference` error to:
- Detect when the missing slug differs from any loaded agent's slug
- Suggest checking if the `extraAgentSlug` matches the target file's `slug`
- Provide helpful hints for common mistakes

**New error format:**
```
ReferenceError: Missing agent reference 'self-reference'
  Referrer: 'local:lrm' in file: localdev-agents/kimi-07.err1.json
  
  Hint: The slug 'self-reference' was not found in any loaded agent configuration.
  
  Did you mean to reference one of these agents?
    - 'local:lrm' (defined in localdev-agents/kimi-07.err1.json)
  
  Remember: The 'slug' in extraAgents must match the target agent's actual slug,
  not an arbitrary name. Check that your extraAgents configuration uses the 
  correct slug from the target file.
```

### 3. Infinite Loop Protection

In `src/System/Agents/AgentTree.hs`, in the `wireAgentTools` function:
- Partition extraRuntimes to skip infinite-looping on self-references (when `slug == childSlug`)
- However an agent must be able to add itself as its own tool

## Files to Modify

1. `docs/advanced-configuration.md` - Documentation improvements
2. `src/System/Agents/AgentTree.hs` - Better error messages and infinite loop protection

## Trade-offs

- **Documentation-only fix** would still allow users to make this mistake
- **Alias system** (allowing arbitrary extraAgentSlugs) would require significant changes to the registry lookup mechanism and is out of scope
- This solution balances clarity (better docs/errors) with safety (loop protection)

## Testing
