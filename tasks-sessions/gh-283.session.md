## Summary

Fixed `agq clean` to properly delete branches that have suffixed names (e.g., `gh-132-c7823.2` instead of just `gh-132`).

### Changes to `agq/Agq/Commands.hs`:

1. **Added `WorktreeEntry` data type** to hold both the worktree path and its associated branch:
   ```haskell
   data WorktreeEntry = WorktreeEntry
       { wtPath :: Text
       , wtBranch :: Maybe Text
       }
   ```

2. **Rewrote `parseWorktrees`** to parse the full porcelain output including the `branch` line, extracting the branch name (with `refs/heads/` prefix stripped) for each worktree.

3. **Updated `cmdClean`** to:
   - Use the new `WorktreeEntry` type instead of just `Text` paths
   - After removing a worktree, delete the specific branch associated with it (from `wtBranch`) rather than guessing the branch name from the worktree name

### How it works:

Before this fix:
- `cmdClean` would try to delete branch `gh-132` 
- But the actual branch was `gh-132-c7823.2` (created with SHA + tries suffix in `execTask`)
- The branch deletion would silently fail or delete the wrong branch

After this fix:
- `cmdClean` parses the branch from `git worktree list --porcelain` (e.g., `refs/heads/gh-132-c7823.2`)
- It removes the worktree first
- Then deletes the exact branch that was associated with that worktree
- If no branch is found (detached HEAD), it logs a warning instead of failing
