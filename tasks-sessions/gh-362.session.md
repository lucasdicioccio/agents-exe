## Background

This issue is part of implementing proper LLM-to-LLM recursive conversation capture (#347).

Depends on: #349 (Callback mechanism for `turnAgentRuntimeIntoIOTool`)

Currently, when an agent calls a sub-agent, the traces are emitted separately without correlation information. We need to enhance the tracing system to:
1. Link sub-agent traces to parent conversation traces
2. Emit special trace events for sub-agent session lifecycle
3. Enable trace visualization tools to show the full call tree

## Task

Enhance the tracing system to correlate sub-agent traces with their parent conversations.

```haskell
runSubAgent :: 
    SubAgentSessionConfig 
    -> Tracer IO Trace  -- ^ Parent tracer for emitting correlation traces
    -> ToolExecutionContext 
    -> PromptOtherAgent 
    -> IO CByteString.ByteString
runSubAgent cfg parentTracer ctx (PromptOtherAgent query) = do
    -- Extract parent context
    let parentSessionId = ctx.ctxSessionId
    let parentConversationId = ctx.ctxConversationId
    let parentAgentSlug = case ctx.ctxCallStack of
            (entry:_) -> callAgentSlug entry
            [] -> "unknown"
    
    -- Create child session
    session0 <- mkChildSession parentSessionId parentConversationId parentAgentSlug
    
    -- Create the agent
    agent <- runtimeToAgentForToolInIOScriptExecution 
                cfg.subAgentStore 
                cfg.subAgentOnProgress 
                rt 
                callerSlug 
                callerId 
                parentConversationId
    
    -- Generate sub-agent conversation ID
    convId <- newConversationId
    
    -- Emit SubAgentStarted trace
    runTracer parentTracer $ 
        AgentTrace_SubAgentStarted 
            parentAgentSlug
            callerId
            parentConversationId
            rt.agentSlug
            convId
            (sessionId session0)
    
    -- Notify callback
    case cfg.subAgentOnProgress of
        Just onProgress -> onProgress (SessionStarted session0)
        Nothing -> pure ()
    
    -- Run the agent with nested tracing
    result <- try $ run convId agent (agentSetQuery (UserQuery query) agent) session0
    
    case result of
        Left (e :: SomeException) -> do
            -- Emit SubAgentFailed trace
            runTracer parentTracer $ 
                AgentTrace_SubAgentFailed 
                    parentAgentSlug
                    callerId
                    parentConversationId
                    rt.agentSlug
                    convId
                    (sessionId session0)
                    (Text.pack $ show e)
            throwIO e
        
        Right (finalTurnContent, finalSession) -> do
            -- Emit SubAgentCompleted trace
            runTracer parentTracer $ 
                AgentTrace_SubAgentCompleted 
                    parentAgentSlug
                    callerId
                    parentConversationId
                    rt.agentSlug
                    convId
                    (sessionId session0)
            
            -- Notify callback
            case cfg.subAgentOnProgress of
                Just onProgress -> onProgress (SessionCompleted finalSession)
                Nothing -> pure ()
            
            pure $ Text.encodeUtf8 $ extractResponseText finalTurnContent.llmResponse
```

## Tradeoffs

1. **Trace Volume**: Adding sub-agent traces increases trace volume. However, these are high-value events that occur relatively infrequently (only when agents call other agents).

2. **Tracer Propagation**: Passing the tracer through the call chain adds parameter complexity. The alternative (global tracer registry) would add coupling.

3. **Error Handling**: Failed sub-agent calls emit failure traces. This provides visibility into sub-agent failures from the parent's perspective.
