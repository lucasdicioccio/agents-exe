## Overview

Currently, the TUI queues messages when the LLM conversation is not waiting for user inputs. These messages are stored in `coreBufferedMessages :: TVar (Map ConversationId [Text])` but are not visible to the user until they are consumed.

This feature makes the message queue explicit by:
1. Exposing the buffered messages in the Core data type for UI access
2. Displaying the list of queued messages in the main Conversation text (above total usage counts)
3. Showing the number of queued messages in the Conversations list (next to turn count)

## Current Implementation

In `src/System/Agents/TUI/Types.hs`:
```haskell
data Core = Core
    { coreAgents :: [TuiAgent]
    , coreConversations :: [Conversation]
    , corePausedConversations :: Set ConversationId
    , coreBufferedMessages :: TVar (Map ConversationId [Text])  -- Hidden in TVar
    }
```

Messages are buffered via `addBufferedMessage` in `Event.hs` and consumed via `readAndClearBufferedMessages` when the agent needs input.
