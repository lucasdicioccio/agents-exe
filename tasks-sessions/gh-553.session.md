## Summary

The Developer Toolbox provides a comprehensive `help` capability that returns detailed documentation for all enabled capabilities. However, the LLM is not explicitly informed about this `help` capability in the tool's registration description, making it less discoverable.

## Current Behavior

The developer toolbox is registered in `src/System/Agents/ToolRegistration.hs` with the following description pattern:

```haskell
toolDescriptionText = "Developer tools for writing and validating agents and tools: " <> box.toolboxDescription
```

While the `help` capability is listed in the `capability` parameter description along with other capabilities, there is no explicit mention that calling `help` will return comprehensive documentation about all available capabilities.

## Desired Behavior

Update the ToolRegistration for the developer toolbox to explicitly mention the `help` capability in the tool description, informing the LLM that this is the way to learn more about the toolbox's usage.

## Proposed Change

Update the `registerDeveloperTool` function in `src/System/Agents/ToolRegistration.hs` to include a mention of the `help` capability in the tool description. For example:

```haskell
toolDescriptionText = 
    "Developer tools for writing and validating agents and tools. "
    <> "Use the 'help' capability to get detailed documentation for all available capabilities. "
    <> box.toolboxDescription
```

## Rationale

1. **Discoverability**: LLMs often don't know about capabilities unless explicitly told
2. **Self-documentation**: The `help` capability provides comprehensive usage examples and best practices
3. **User experience**: Users benefit when LLMs proactively use the help capability to understand tool usage
4. **Consistency**: Other toolboxes (like SystemToolbox) could follow the same pattern
