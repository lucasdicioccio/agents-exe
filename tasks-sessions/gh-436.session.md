### 💬 User Query

# TUI: Add message queue management for paused conversations

## Overview

The TUI already has a message buffering mechanism that allows users to queue messages while an agent is processing tool calls. Currently, users can see queued messages (shown in yellow) but have no way to manage them. This feature adds the ability to **review**, **clear**, and **delete individual** queued messages when a conversation is paused.

## Background

### Current Implementation

The TUI uses two data structures for message buffering:

```haskell
-- In Core (System/Agents/TUI/Types.hs)
data Core = Core
    { ...
    , coreBufferedMessages :: TVar (Map ConversationId [Text])
    -- ^ Buffered messages per conversation - allows users to send messages
    -- while the agent is processing tool calls.
    }

-- In UIState
data UIState = UIState
    { ...
    , _uiBufferedMessages :: Map ConversationId [Text]
    -- ^ Copy of buffered messages from Core for UI rendering
    }
```

### Current Display

Queued messages are currently displayed via:
- `render_queued_messages` in `Render.hs` - shows messages in the conversation view
- `getQueuedMessageCount` - shows `[N queued]` indicator in the conversation list

### Paused Conversations

The system already supports pausing via `Ctrl+E`:
- `ConversationStatus_Paused` status blocks step iteration
- `corePausedConversations` tracks which conversations are paused
- When paused, the conversation shows `⏸` indicator

## Feature Requirements

### Functional Requirements

1. **Review Queued Messages**: Users can view all queued messages for the current conversation
2. **Clear All Messages**: Users can clear all queued messages with a single action
3. **Delete Individual Messages**: Users can select and delete specific messages from the queue

### Activation Condition

These features should **ONLY** be available when:
- The conversation status is `ConversationStatus_Paused`
- The Chats tab is active (where the conversation view is shown)

When the conversation is not paused, the queue management UI should be hidden or disabled.

-- | Render the queue management UI
render_queue_panel :: TuiState -> Conversation -> Widget N
render_queue_panel st conv = 
    let queuedMsgs = getQueuedMessages st conv
        count = length queuedMsgs
        selectedIdx = st ^. tuiUI . queuedMessagesFocus
     in if count == 0
            then emptyWidget
            else borderWithLabel (txt $ " Queued Messages (" <> show count <> ") ") $
                vBox 
                    [ txt "Ctrl+D: clear all  Del/Backspace: delete selected"
                    , render_queued_message_list selectedIdx queuedMsgs

render_queued_message_list :: Maybe Int -> [Text] -> Widget N
render_queued_message_list selectedIdx msgs =
    vBox $ zipWith (render_queued_item selectedIdx) [0..] msgs

render_queued_item :: Maybe Int -> Int -> Text -> Widget N
render_queued_item selectedIdx idx msg =
    let isSelected = selectedIdx == Just idx

        truncated = Text.take 60 msg <> if Text.length msg > 60 then "..." else ""
     in withAttr (if isSelected then queuedMessageSelectedAttr else queuedMessageAttr) $
            txt $ marker <> truncated

```haskell
-- | Clear all queued messages for the current conversation
handleClearQueuedMessages :: EventM N TuiState ()
handleClearQueuedMessages = do
    mConv <- getFocusedConversation
    case mConv of
        Nothing -> pure ()
        Just conv -> do
            coreRef <- use tuiCore
            let convId = conversationId conv
            -- Check if conversation is paused
            core <- liftIO $ readTVarIO coreRef
            if conversationStatus conv == ConversationStatus_Paused
                then do
                    -- Clear in Core
                    liftIO $ atomically $ 
                        modifyTVar (coreBufferedMessages core) $ Map.insert convId []
                    -- Clear in UI
                    tuiUI . uiBufferedMessages %= Map.insert convId []
                    tuiUI . queuedMessagesFocus .= Nothing
                    showStatus StatusInfo "All queued messages cleared"
                else showStatus StatusWarning "Can only clear when paused (Ctrl+E)"

-- | Delete the currently selected queued message
handleDeleteSelectedMessage :: EventM N TuiState ()
handleDeleteSelectedMessage = do
    mConv <- getFocusedConversation
    case mConv of
        Nothing -> pure ()
        Just conv -> do
            let convId = conversationId conv
            mSelectedIdx <- use (tuiUI . queuedMessagesFocus)
            case mSelectedIdx of
                Nothing -> showStatus StatusWarning "Select a message first (use Up/Down arrows)"
                Just idx -> do
                    -- Get current messages
                    buffered <- use (tuiUI . uiBufferedMessages)
                    case Map.lookup convId buffered of
                        Nothing -> pure ()
                        Just msgs -> 
                            if idx < 0 || idx >= length msgs
                                then pure ()
                                else do
                                    -- Remove message at index
                                    let newMsgs = deleteAt idx msgs
                                    -- Update Core
                                    coreRef <- use tuiCore
                                    liftIO $ atomically $ 
                                        modifyTVar (coreBufferedMessages $ unsafePerformIO $ readTVarIO coreRef) $ 
                                            Map.insert convId newMsgs
                                    -- Update UI
                                    tuiUI . uiBufferedMessages %= Map.insert convId newMsgs
                                    -- Adjust selection
                                    let newIdx = if null newMsgs then Nothing else Just (min idx (length newMsgs - 1))
                                    tuiUI . queuedMessagesFocus .= newIdx
                                    showStatus StatusInfo "Message deleted"

-- | Navigate through queued messages
handleQueueNavigation :: Int -> EventM N TuiState ()
handleQueueNavigation direction = do
    mConv <- getFocusedConversation
    case mConv of
        Nothing -> pure ()
        Just conv -> do
            let convId = conversationId conv
            buffered <- use (tuiUI . uiBufferedMessages)
            case Map.lookup convId buffered of
                Nothing -> pure ()
                Just msgs -> do
                    let count = length msgs
                    current <- use (tuiUI . queuedMessagesFocus)
                    let newIdx = case current of
                            Nothing -> if direction > 0 then 0 else count - 1
                            Just idx -> max 0 $ min (count - 1) (idx + direction)
                    tuiUI . queuedMessagesFocus .= Just newIdx
```

## Tradeoffs

1. **Scope restriction to paused state**: This intentionally limits the feature to when users explicitly want to change queued state, preventing accidental deletions during normal operation
2. **Keyboard shortcuts**: Using `Ctrl+D` for clear follows the pattern of `Ctrl+E` for pause, keeping related operations together
3. **No mouse support**: This keeps the implementation simpler and consistent with other TUI operations which are keyboard-driven
