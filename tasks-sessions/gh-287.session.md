### 💬 User Query

# Add mechanism, builtin toolbox, or tools and guidelines to write new agents/tools

Today, writing new agent configurations is still cumbersome.

- Import/Export commands: I've not really used them and they were vibe-coded so currently likely need some modification and they require some initial examples
- There is no init-agent or init-tool function
- We should have libraries or examples of tools in many programming languages
- We should have support of LLMs for writing tools (mainly when in bash, but not only)


To cover these:
- agents-exe should embed some documentation about the bash-tools spec so that we can do `agents-exe spec bash-tools` and get the equivalent of the documentation markdown
- agents-exe should allow to check the `describe` registration without needing to create an agent file
- agents-exe should have some form of `agents-exe new agent <slug> <file> <model>` command line
- agents-exe should have some form of `agents-exe new tool <slug> <proglang> <file>` command line
- we could support these features in a builtin toolbox so that writing an agent that "knows how to write, validate an agent or tool definition" is somewhat trivial
