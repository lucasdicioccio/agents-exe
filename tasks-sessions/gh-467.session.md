# Make helper agent subcall conversations visible in TUI via OS integration

Depends-on: 
Base-branch: feature/subcall-conversation-visibility
Final: false

## Problem Statement

When a helper agent (sub-agent) is called via `turnAgentRuntimeIntoIOTool` in `System.Agents.AgentTree.OneShotTool`, the conversation is created but remains invisible to the TUI. The subcall:

1. Generates a new `ConversationId` 
2. Creates a fresh `Session`
3. Runs the sub-agent
4. Returns the result

**However**, the conversation is:
- ❌ **NOT inserted into the OS World** (no `ConversationConfig`/`ConversationState` components)
- ❌ **NOT persisted to SessionStore**
- ❌ **NOT tracked in OS conversation lineage**
- ❌ **NEVER visible in the TUI**

Users cannot see when agents are calling other agents, track subcall progress, or review subcall history.

## Goals

1. **OS Integration**: Insert subcall conversations into the OS World as first-class entities
2. **Lineage Tracking**: Link subcalls to their parent conversations via OS Lineage components
3. **Event Emission**: Emit OS events for subcall lifecycle (start/progress/complete)
4. **TUI Visibility**: Display subcall conversations in the TUI with visual distinction
5. **Hierarchy**: Show parent/child relationships between conversations

## Migration Path

This change is **backward compatible** if implemented with `Maybe` fields:
- `ctxWorld = Nothing` → skips OS insertion (old behavior)
- `ctxEventQueue = Nothing` → skips event emission (old behavior)
- `ctxParentConversation = Nothing` → no parent link

Existing code continues to work; new fields are opt-in.
