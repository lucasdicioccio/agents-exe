# Suspend Session in Toolbox Lifecycle

Depends-on: 
Base-branch: gh-507
Final: false

## Overview
Implement proper resource cleanup and session suspension in the toolbox lifecycle, ensuring resources are released after tool calls or during pauses.

## Motivation
Long-running sessions can accumulate resources:
- Open file handles
- Database connections
- Memory allocations
- Temporary files

Proper lifecycle management prevents resource leaks and enables safe suspension/resumption.

## Proposed Solution
Add explicit lifecycle hooks for session suspension and resource cleanup in the toolbox system.

### Key Features
1. **Resource Tracking**: Track resources allocated during tool execution
2. **Cleanup on Suspend**: Release resources when session is suspended
3. **Resume Capability**: Restore necessary state on resume
4. **Timeout Cleanup**: Automatic cleanup for idle resources

### Lifecycle States
```
Active → Suspending → Suspended → Resuming → Active
   ↓         ↓            ↓           ↓
Cleanup   Persist      Wait       Restore
```

### Implementation Sketch
```haskell
-- Resource lifecycle management
data SessionLifecycle = Active | Suspending | Suspended | Resuming

-- Resource handle with cleanup
 data ManagedResource = ManagedResource {
     resource :: Resource,
     cleanup :: IO (),
     scope :: ResourceScope
 }

-- Suspend session
suspendSession :: Session -> IO SuspendedSession

-- Resume session
resumeSession :: SuspendedSession -> IO Session

-- Cleanup resources
cleanupResources :: ResourceScope -> IO ()
```

## Summary

### New Module: `src/System/Agents/OS/Resources/Lifecycle.hs`

This module implements the session lifecycle management system for the toolbox, providing:

#### Core Types
- **`SessionLifecycle`**: State machine with `Active`, `Suspending`, `Suspended`, `Resuming` states
- **`SessionState`**: Tracks session metadata including lifecycle, IDs, activity timestamp, and resource count
- **`ResourcePersistability`**: Classifies resources as `Persistent`, `Transient`, or `Recreatable`
- **`ManagedResource`**: Extended resource handle with cleanup, persistability, and scope tracking
- **`ResourceScope`**: Session-level scoping (`ScopeProgram`, `ScopeSession`, `ScopeConversation`, `ScopeTurn`, `ScopeToolCall`)
- **`SuspendedSession`**: Serializable state for session suspension/resumption

#### Session Manager
- **`SessionManager`**: Central manager for lifecycle and resource cleanup with STM-based concurrency
- **`newSessionManager`**: Create a new manager with a given registry and session IDs

#### Lifecycle Operations
- **`suspendSession`**: Transition to `Suspending`, cleanup transient resources, persist state
- **`resumeSession`**: Transition to `Resuming`, restore persistent resources, return to `Active`
- **`forceCleanupSession`**: Emergency cleanup of all resources

#### Resource Tracking
- **`withResourceTracking`**: Bracketed resource tracking for specific scopes
- **`registerCleanup`**: Register custom cleanup actions
- **`registerManagedResource`**: Track resources for lifecycle management

#### Timeout Management
- **`SessionTimeoutConfig`**: Configuration for idle detection
- **`startTimeoutWatcher`**: Background thread for idle detection
- **`stopTimeoutWatcher`**: Stop the watcher
- **`pingSessionActivity`**: Keep-alive ping to prevent timeout

#### Idle Resource Cleanup
- **`getIdleResourceInfo`**: Query idle resources
- **`cleanupIdleResources`**: Cleanup idle transient resources

#### Resource Snapshots
- **`ResourceSnapshot`**: Serializable resource state
- **`PersistedResource`**: Restorable resource with snapshot and restore action
- **`takeResourceSnapshot`**: Capture resource state
- **`restoreFromSnapshot`**: Recreate resource from snapshot
