# Implement session context passing for IO tools

## Overview

Modify the IO tool execution (`System.Agents.Tools.IO`) to receive `ToolExecutionContext`. Unlike bash tools which get context via environment variables, IO tools (Haskell functions) can receive context directly as a function argument.

## Current State

```haskell
data IOScript rtVal llmArg b = IOScript
    { description :: IOScriptDescription
    , ioRun :: rtVal -> llmArg -> IO b  -- rtVal is currently ConversationId
    }

-- Usage in OneShot.hs:
-- ToolRuntimeArg = ConversationId
type ToolRuntimeArg = ConversationId
```

## Required Changes

### 1. Update IOScript Type

Replace `rtVal` with `ToolExecutionContext`:

```haskell
data IOScript llmArg b = IOScript
    { description :: IOScriptDescription
    , ioRun :: ToolExecutionContext -> llmArg -> IO b
    }
```

### 2. Update runValue Function

```haskell
runValue :: 
    (FromJSON llmArg) 
    => Tracer IO (Trace llmArg b) 
    -> IOScript llmArg b 
    -> ToolExecutionContext  -- was runtimeValue (ConversationId)
    -> Aeson.Value 
    -> IO (Either RunError b)
```

### 3. Update ioTool Constructor

In `System.Agents.Tools`:

```haskell
ioTool ::
    (Aeson.FromJSON llmArg) =>
    IOScript llmArg ByteString ->
    Tool ()  -- no longer polymorphic in rtVal
ioTool script =
    Tool
        { toolDef = IOTool script.description
        , toolRun = run
        }
  where
    call = ()
    run tracer ctx v = do  -- ctx is now ToolExecutionContext, not ignored
        let adaptTrace = IOToolsTrace . IOTools.adaptTraceInput (const v)
        ret <- IOTools.runValue (contramap adaptTrace tracer) script ctx v
        ...
```

## Example IO Tool with Context

Before:
```haskell
myTool :: IOScript ConversationId MyArg ByteString
myTool = IOScript 
    { description = IOScriptDescription "my-tool" "Does something"
    , ioRun = \_convId arg -> do  -- conversation id ignored
        -- couldn't access session info
        pure $ result arg
    }
```

After:
```haskell
myTool :: IOScript MyArg ByteString
myTool = IOScript 
    { description = IOScriptDescription "my-tool" "Does something"
    , ioRun = \ctx arg -> do
        let sessionId = ctxSessionId ctx
        let fullSession = ctxFullSession ctx
        -- can now access complete session context
        logWithSession sessionId arg
        pure $ result arg
    }
```

## Design Decisions

**Tradeoffs:**

1. **Direct argument vs Reader pattern**: Direct argument is simpler:
   - Matches current IOScript structure
   - No additional monad transformers needed
   - Clear and explicit dependency
   - ReaderT could be added later if needed

2. **Breaking change vs backward compatibility**: This is a breaking change:
   - All existing IOScript definitions need update
   - Type system guides the migration
   - Better than maintaining two parallel APIs
   - Migration is mechanical (replace `_convId` with `ctx`)

## Migration Guide for Tool Authors

Existing IO tools need to update their `ioRun` signature:

```haskell
-- Before:
ioRun :: ConversationId -> MyArg -> IO ByteString
ioRun _convId arg = ...

-- After:
ioRun :: ToolExecutionContext -> MyArg -> IO ByteString  
ioRun ctx arg = ...
-- Use ctx when needed, or ignore with _ctx if not needed
```
