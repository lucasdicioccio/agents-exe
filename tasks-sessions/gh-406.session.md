# Create tmux-record.sh bash tool for capturing tmux window content

Depends-on: 
Base-branch: gh-tmux-record
Final: false

## Summary

Create a new bash tool at `tools/terminal/tmux-record.sh` that uses the "One-Line Shell Command" convention to record the currently-running tmux window using `tmux capture-pane -p -S`.

## Tool Specification

The tool should:

1. **Follow the standard bash tool pattern** with `describe` and `run` commands
2. **Use `tmux capture-pane -p -S`** to capture pane content to stdout
3. **Accept an optional argument** `start_line` (integer, optional) that specifies:
   - Which line to start capturing from (0-indexed, negative for relative to end)
   - Default: capture entire history (use `-` which means "start of history")
   - Common use: `-1000` for last 1000 lines
4. **Output format**: Follow the `----` separator convention like other read tools

## Implementation Details

### JSON Schema (describe mode)
```json
{
    "slug": "tmux-record",
    "description": "Record the content of the currently-running tmux window using tmux capture-pane",
    "args": [
        {
            "name": "start_line",
            "description": "Line number to start capturing from (negative for relative to end, default captures all history)",
            "type": "integer",
            "backing_type": "integer",
            "arity": "optional",
            "mode": "positional"
        }
    ]
}
```

### Behavior (run mode)
- Check if running inside tmux (`$TMUX` environment variable)
- Use `tmux capture-pane -p -S <start_line>` to capture content
- If no start_line provided, use `-S -` (start from beginning of history)
- Output with `----` separator prefix
- Handle errors gracefully (not in tmux, tmux command fails)

## File Location
`tools/terminal/tmux-record.sh`

## Acceptance Criteria
- [ ] Script follows the `describe|run` protocol
- [ ] Script outputs proper JSON schema in describe mode
- [ ] Script captures tmux pane content correctly in run mode
- [ ] Script handles the optional start_line argument
- [ ] Script checks for tmux environment and reports errors appropriately
- [ ] Script is executable

## Tradeoffs Considered

1. **Default behavior (capture all vs. limited lines)**:
   - Capturing all history may produce very large outputs for long-running sessions
   - Alternative: default to last 1000 lines, but this might miss important earlier content
   - Decision: Capture all by default (`-S -`) as the LLM can always truncate if needed

2. **Pane selection (current vs. specific)**:
   - Option: Always capture current pane (simplest)
   - Option: Allow specifying pane ID (more flexible but complex)
   - Decision: Capture current pane only - agents typically want to record their own session

3. **Target pane (pane vs. window)**:
   - `tmux capture-pane` captures a single pane
   - `tmux capture-pane` with `-a` captures all panes in a window
   - Decision: Capture single pane (current) - clearer semantics, agents can call multiple times if needed
