# Implement Session-Aware Skills.md Toolbox with Progressive Disclosure

---
## Summary

Implement **session-aware skills toolbox** supporting the [agentskills.io](https://agentskills.io) specification. Skills are discovered from configured sources (local directories, git repos) and exposed via **progressive disclosure**: only metadata tools are initially visible; script tools become available after explicit skill activation. Tool availability is computed as a **pure function of Session history**, making it deterministic, testable, and persistent.

---

## Motivation

Agents need contextual knowledge to perform complex tasks reliably. The skills.md format provides:
- **Procedural knowledge**: Step-by-step instructions for tasks
- **Executable capabilities**: Scripts for automation  
- **Progressive disclosure**: Load only what's needed (~100 tokens metadata vs ~5000 tokens full instructions)
- **Portability**: Skills work across compatible agents

### Current System Limitation
Toolboxes expose static `TVar [ToolRegistration]` - all tools are always available with no mechanism for progressive disclosure or historical tracking of tool enablement.

---

### Part B: Session State Monoid

**New File:**
- `src/System/Agents/Tools/Skills/State.hs` - Pure session folding

```haskell
-- Activation state for scripts within a skill
data ScriptState = Enabled | Disabled
    deriving (Show, Eq, Ord)

type SkillScriptsState = Map ScriptName ScriptState

data SkillsSessionState = SkillsSessionState
    { sssActiveSkills :: Map SkillName SkillScriptsState
    }

-- Later state overrides earlier (last enable/disable wins)
instance Semigroup SkillsSessionState where
    (SkillsSessionState s1) <> (SkillsSessionState s2) =
        SkillsSessionState (Map.unionWith (Map.unionWith const) s1 s2)

instance Monoid SkillsSessionState where
    mempty = SkillsSessionState Map.empty

-- Pure fold over session turns
foldSession :: Session -> SkillsSessionState
foldSession session = foldMap extractFromTurn session.turns

extractFromTurn :: Turn -> SkillsSessionState
extractFromTurn turn = case turn of
    LlmTurn llmTurn _ -> foldMap extractFromToolCall llmTurn.llmToolCalls
    UserTurn _ _ -> mempty
```

### Part C: Tool Registration Generation

**New File:**
- `src/System/Agents/Tools/Skills/Toolbox.hs` - Session-aware toolbox

```haskell
-- Main entry point: compute available tools from session
computeSkillTools :: SkillsStore -> Session -> [ToolRegistration]
computeSkillTools store session =
    let state = foldSession session
        -- Always available: skill discovery and enablement
        metaTools = concatMap (makeMetaTools store) (allSkills store)
        -- Available only when enabled: script execution
        scriptTools = concatMap (makeScriptTools state store)
            (Map.toList $ sssActiveSkills state)
    in metaTools ++ scriptTools

makeMetaTools :: SkillsStore -> Skill -> [ToolRegistration]
makeMetaTools store skill =
    [ -- Returns skill metadata + instructions
      makeDescribeTool skill
      -- Enables all scripts in skill
    , makeEnableTool skill
      -- Disables all scripts in skill
    , makeDisableTool skill
      -- Lists available skills
    , makeListSkillsTool store
    ]

makeScriptTools :: SkillsSessionState -> SkillsStore -> (SkillName, SkillScriptsState) -> [ToolRegistration]
makeScriptTools state store (skillName, scripts) =
    case lookupSkill store skillName of
        Nothing -> []
        Just skill ->
            [ makeScriptTool skill script
            | (scriptName, scriptState) <- Map.toList scripts
            , scriptState == Enabled
            , Just script <- [lookupScript skill scriptName]
            ]
```

**Tool naming convention:**
- `skill_describe_{skill_name}` - Get skill metadata and instructions
- `skill_enable_{skill_name}` - Enable skill scripts
- `skill_disable_{skill_name}` - Disable skill scripts
- `skill_list` - List all available skills
- `skill_{skill_name}_{script_name}` - Execute enabled skill script
